## 1.5. 摘要
智能体多模态大模型（如OpenAI o3、Gemini Agentic Vision）通过迭代调用视觉工具实现了极强的推理能力，但级联的感知-推理-工具调用循环引入了严重的串行开销（该开销被定义为**智能体深度**），导致延迟过高、系统并发能力严重受限。
为解决该问题，论文提出SpecEyes智能体级投机加速框架，核心思路是用轻量无工具的多模态大模型作为投机规划器预测执行轨迹，在不损失精度的前提下提前终止昂贵的工具链。为规范投机规划，论文提出基于答案可分性的认知门控机制，无需真值标注即可量化模型置信度实现自验证。此外设计了异构并行漏斗，利用小模型的无状态并发能力掩盖大模型的有状态串行执行开销，最大化系统吞吐。
在V* Bench、HR-Bench、POPE三个基准上的实验表明，SpecEyes相比智能体基线实现1.1~3.35倍的加速，同时精度保持不变甚至最高提升6.7%，大幅提升了并发负载下的服务吞吐量。

# 2. 整体概括
## 2.1. 研究背景与动机
### 核心问题
当前智能体多模态大模型依靠主动调用缩放、裁剪、OCR等视觉工具，形成迭代的感知-推理-工具调用循环，在细粒度视觉理解、多步组合推理等任务上表现出色，但该机制存在严重的效率瓶颈：
1. **延迟爆炸**：每个查询的端到端响应时间随智能体深度线性增长，每一轮推理和工具调用都必须完成后才能进入下一轮；
2. **并发崩溃**：每个查询的工具调用链会修改专属的状态，无法利用GPU的批量计算能力，大量硬件并行资源被浪费，最终智能体多模态大模型的速度比非智能体版本低几个数量级，无法落地实际场景。
### 现有研究的空白
现有高效推理方法都无法解决该瓶颈：
- 词元级投机解码仅能加速单步生成，还是需要走完完整的串行智能体流程，额外的草稿-验证交互甚至会引入 overhead 抵消加速收益；
- 多模态词元剪枝、时序压缩等方法仅能减少单步计算量，无法消除占延迟主导的重复工具调用。
  所有现有方法都在智能体循环内部做优化，从未质疑「是否每个查询都需要走完整的智能体循环」这一根本问题。
### 创新切入点
论文将投机范式从词元/语义级别提升到**智能体级别**：核心直觉是大部分发给智能体多模态大模型的查询并不需要深度工具辅助推理，轻量无工具的小多模态大模型仅看原始图像就能正确回答，只要能可靠识别这类查询，就能完全绕过昂贵的工具链，不损失精度。基于该直觉设计了「快思考-慢思考」的异构架构：轻量无工具小模型负责快速生成投机答案（快思考），大的智能体模型仅处理真正需要多步工具交互的查询（慢思考）。
## 2.2. 核心贡献/主要发现
### 核心贡献
1. **瓶颈形式化**：首次识别并形式化了智能体多模态大模型的有状态瓶颈，证明工具链固有的数据依赖是单查询延迟和系统级并发的根本障碍；
2. **范式创新**：提出SpecEyes，是首个将投机加速从词元级提升到智能体级的框架，为不需要工具的查询绕过整个工具链，同时保持精度不变；
3. **无标注门控**：提出基于top-K logit答案可分性的认知门控机制，提供了无标注、尺度不变的判断准则，让小模型自行决定是否信任自身输出，还是升级到智能体大模型；
4. **吞吐优化**：设计异构并行漏斗，利用小模型的无状态特性实现并发查询处理，吞吐量提升与投机接受率成正比。
### 主要发现
- 平均有80%的查询可以被筛选为不需要工具，其中71%可以被小模型正确回答，整体投机接受率可达56.8%；
- SpecEyes在三个基准上实现1.1~3.35倍的端到端加速，同时精度最高提升6.7%，并发场景下的吞吐量提升与投机接受率成正比；
- 绕过不必要的工具链还可以减少多步工具调用引入的累积误差，甚至降低幻觉风险。

# 3. 预备知识与相关工作
## 3.1. 基础概念
为方便初学者理解，首先解释本文涉及的核心专业术语：
1. <strong>多模态大语言模型（Multimodal Large Language Model, MLLM）</strong>：可以同时处理文本、图像等多模态输入的大语言模型，早期的MLLM仅对图像做一次编码，单轮生成回答，将视觉作为被动输入通道。
2. <strong>智能体多模态大语言模型（Agentic MLLM）</strong>：可以主动调用外部视觉工具（缩放、裁剪、OCR等），形成「感知-推理-工具调用」的迭代循环，逐步优化对图像的理解，适合需要细粒度检查、多步组合推理、主动信息检索的复杂视觉任务。
3. <strong>智能体深度（Agentic Depth, $D$）</strong>：单个查询触发的工具调用步骤数，每一步的操作都严格依赖上一步的观测结果，是串行开销的核心来源。
4. <strong>投机解码（Speculative Decoding）</strong>：传统的词元级加速方法，用小的草稿模型生成候选词元序列，大模型一次性验证所有候选，接受前缀匹配的词元，减少大模型的串行解码步数，但仅能加速单步生成，无法改变整体推理流程。
5. <strong>有状态/无状态推理（Stateful/Stateless Inference）</strong>：
   - 有状态推理：每一步的输入依赖上一步的输出，需要保存中间状态，无法并行处理多步，比如智能体MLLM的工具调用链；
   - 无状态推理：输入独立，不需要保存中间状态，可以批量并行处理，比如普通单轮MLLM的推理。
## 3.2. 前人工作
本文涉及的相关研究主要分为三类：
### 3.2.1. 智能体多模态大模型
智能体推理起源于工具增强的语言模型，典型工作如Toolformer，通过 interleaving 动作生成和外部反馈实现工具调用。后续该范式扩展到多模态领域：
- DeepEyes用强化学习训练模型在推理过程中调用感知工具；
- CodeDance、Thyme等工作通过代码生成和视觉操作实现可执行推理，通过多轮交互和自反思进一步提升智能体深度。
  但所有这类工作都依赖深度串行的感知-推理-工具循环，带来严重的延迟和并发限制，此前的研究几乎忽略了该系统级瓶颈。
### 3.2.2. 高效推理
- 词元级投机解码：典型工作如Medusa、SpecReason，其中SpecReason将简单步骤交给轻量模型，通过语义一致性验证，但所有这类方法都在固定的推理轨迹内加速步骤，智能体循环本身仍然是完全串行的，甚至会引入额外的交互开销。
- 自适应计算与早退出：通过跳过部分模型层实现加速，但仍然需要走完完整的智能体流程。
  传统词元级投机解码的核心公式如下：
$$
\text{Speedup} = \frac{1 - p^k}{1 - p}
$$
其中$p$是单个词元的接受率，$k$是草稿模型生成的候选词元长度，该方法仅能减少单步解码的串行步数，无法改变智能体工具链的串行本质。
### 3.2.3. 高效多模态感知
包括词元剪枝、词元合并、频域压缩、KV缓存压缩等方法，仅能减少单步多模态感知的计算量，仍然需要执行完整的串行智能体流程，无法消除工具调用带来的主导开销。
## 3.3. 技术演进
多模态大模型的效率优化经历了三个层级的演进：
1. **模型内优化**：通过剪枝、量化、早退出等方法减少单步推理的计算量，仅能实现线性加速；
2. **词元级推理优化**：通过投机解码等方法减少大模型的串行解码步数，加速比受限于词元接受率；
3. **智能体级优化**：也就是本文提出的范式，直接判断查询是否需要完整的智能体工具链，从根上消除不必要的串行步骤，可实现超线性的加速收益。
## 3.4. 差异化分析
本文与现有所有高效方法的核心区别在于：**跳出了「必须走完智能体循环」的固有假设，直接从智能体层级判断是否需要执行工具链**，而不是在智能体循环内部优化单步的效率。与最相关的词元级投机解码相比，本文的方法完全避免了工具调用的开销，也没有额外的草稿-验证交互 overhead，同时还能实现系统级的吞吐提升。

# 4. 方法论
## 4.1. 方法原理
### 核心直觉
大部分发给智能体MLLM的查询并不需要深度工具辅助推理，轻量无工具的小MLLM仅通过原始图像就能正确回答，只要能可靠识别这类查询，就能完全绕过昂贵的工具链，不损失精度。
### 理论基础
该设计借鉴了认知科学的**双系统理论**：
- 系统1（快思考）：直觉式、快速、无意识的推理，对应本文的轻量无工具小模型，无状态可并行，延迟低；
- 系统2（慢思考）：深思熟虑、慢速、需要多步推理的过程，对应本文的大智能体模型，有状态串行，精度高但延迟高。
## 4.2. 核心方法详解
### 4.2.1. 智能体MLLM的有状态瓶颈形式化
首先对智能体MLLM的串行瓶颈做数学建模：
将智能体多模态大模型形式化为有状态推理系统$\mathcal{A} = (\mathcal{S}, \mathcal{T}, \pi)$：
- $\boldsymbol{S}$：状态空间
- $\mathcal{T} = \{t_1, \dots, t_N\}$：有限的视觉工具集合，如缩放、裁剪、OCR等
- $\pi$：策略函数，同时选择工具调用和生成推理词元
  给定查询$q$和输入图像$I$，模型维护长度为$D$的状态轨迹$\{s_0, s_1, \dots, s_D\}$，初始状态$s_0 = (q, I)$。每一步$d$，策略生成动作`\boldsymbol{a}_d = \pi(\boldsymbol{s}_d)`，要么调用工具，要么输出最终答案。如果调用工具，状态转移公式为：
$$
\boldsymbol{s}_{d+1} = \boldsymbol{f}(\boldsymbol{s}_d, t_d(\boldsymbol{s}_d))
$$
符号解释：
- $t_d(s_d)$：将选中的工具$t_d$应用到当前视觉上下文的结果，比如从图像中裁剪感兴趣区域
- $f$：将工具返回的观测结果融合到下一个状态的函数
- $D$：该查询的智能体深度，即工具调用的总步数

  该状态转移存在严格的因果依赖：下一个工具的选择完全依赖上一步的观测结果，即$p(a_{d+1} \mid s_0, \dots, s_d) = p(a_{d+1} \mid s_d, t_d(s_d)) \neq p(a_{d+1} \mid s_0)$，因此整个智能体流程是完全串行的，单查询的端到端延迟为：
$$
L_{\mathrm{agent}}(q) = \sum_{d=0}^{D(q)} \left( \underbrace{c_{\mathrm{llm}}}_{\mathrm{推理延迟}} + \underbrace{c_{\mathrm{tool}}(t_d)}_{\mathrm{工具执行延迟}} \right)
$$
符号解释：
- $c_{\mathrm{llm}}$：每一步LLM推理的延迟
- $c_{\mathrm{tool}}(t_d)$：第$d$步调用工具$t_d$的执行延迟
  可见单查询延迟随智能体深度$D$线性增长。

从系统级吞吐来看，对于批量$B$个查询，由于每个查询的状态独立，大模型每次只能处理一个查询的一步，因此最大吞吐上限为：
$$
\Theta_{\mathrm{agent}} \leq \frac{B}{\sum_{i=1}^B L_{\mathrm{agent}}(q_i)}
$$
随着平均智能体深度$D$增大，该上限会急剧降低。

### 4.2.2. 四阶段投机推理Pipeline
SpecEyes通过四阶段的漏斗架构实现智能体级投机加速，流程如下图（原文Figure 2）所示：

![Fig. 2. Pipeline overview of SpecEyes. A batch of $B$ queries passes through a four-phase funnel. I: $\\mathcal { M } _ { L }$ screens tool necessity, splitting queries into tool-free and tool-required. II: A stateless $\\mathcal { M } _ { S }$ speculatively answers all tool-free queries with token-level logits. III: An answer separability score $S _ { \\mathrm { s e p } }$ gates each answer; those above $\\tau$ are accepted directly. IV: Remaining queries fall back to the full agentic loop. The funnel yields ${ \\approx } 1 / ( 1 { - } \\beta \\alpha ) \\times$ throughput speedup.](images/2.jpg)
*该图像是示意图，展示了SpecEyes的处理流程。图中描述了一个批量查询（B queries）的四个阶段：第一阶段为工具使用判断，第二阶段进行投机预测，第三阶段引入认知门控机制，最后则是回退至完整的智能代理环节。每个阶段的操作和关系通过图形和箭头清晰呈现，以提高系统的吞吐量。*

定义：
- $\mathcal{M}_S$：轻量无工具的小多模态大模型，无状态可并行
- $\mathcal{M}_L$：大的智能体多模态大模型，即上文中的$\mathcal{A}$

#### Phase I: 启发式工具必要性判断
首先用大智能体模型$\mathcal{M}_L$做轻量二分类，判断查询是否需要调用工具：
$$
g(q, I) = \mathcal{M}_L(q, I; \mathcal{P}_{\mathrm{judge}}) \in \{0, 1\}
$$
符号解释：
- $\mathcal{P}_{\mathrm{judge}}$：提示词， instruct 模型判断仅通过全局图像是否可以回答查询
- $g=0$：不需要工具，进入Phase II
- $g=1$：需要工具，直接进入Phase IV的智能体回退
  该步骤仅生成一个二元词元，无工具调用，开销可忽略，选用$\mathcal{M}_L$而非小模型是因为$\mathcal{M}_L$本身具备工具调用能力，对工具必要性的判断更准确。

#### Phase II: 投机预测
所有$g=0$的查询，交给无状态小模型$\mathcal{M}_S$直接生成答案和每个词元的logit分布：
$$
\hat{y}_S, \{\ell^{(n)}\}_{n=1}^{|\hat{y}_S|} = \mathcal{M}_S(q, I)
$$
符号解释：
- $\hat{y}_S$：小模型生成的投机答案
- $\ell^{(n)} \in \mathbb{R}^{|\mathcal{V}|}$：第$n$个生成词元在词表$\mathcal{V}$上的logit向量
  该推理是无状态的，不需要工具执行，可以对批量内的所有查询并行处理。

#### Phase III: 小模型置信度切换
通过认知门控函数$S_{\mathrm{sep}}$计算投机答案的置信度，无需真值标注，判断是否接受小模型的答案：
$$
\mathrm{decision} = \left\{
\begin{array}{ll}
\mathrm{accept} \ \hat{y}_S, & \mathrm{if} \ S_{\mathrm{sep}}(\hat{y}_S) \geq \tau, \\
\mathrm{fallback \ to} \ \mathcal{M}_L, & \mathrm{if} \ S_{\mathrm{sep}}(\hat{y}_S) < \tau,
\end{array}
\right.
$$
符号解释：
- $\tau$：在小型验证集上校准的阈值
- 接受的答案直接返回，完全绕过智能体流程；拒绝的查询进入Phase IV。

#### Phase IV: 智能体回退
所有$g=1$或者门控拒绝的查询，交给大智能体模型$\mathcal{M}_L$执行完整的有状态感知-推理循环：
$$
\hat{y}_L = \mathcal{M}_L(q, I) = \pi\left( s_0 \xrightarrow{t_0} s_1 \xrightarrow{t_1} \cdot\cdot\cdot \xrightarrow{t_{D-1}} s_D \right)
$$
该步骤作为安全网，保证不会出现明显的精度损失。

### 4.2.3. 端到端延迟建模
定义：
- $\beta \in [0,1]$：Phase I筛选出的不需要工具的查询比例
- $\alpha \in [0,1]$：Phase III认知门控的接受率
- $c_J$：Phase I工具必要性判断的开销
- $c_S$：小模型$\mathcal{M}_S$的推理开销
  则SpecEyes的单查询期望延迟为：
$$
\mathbb{E}[L_{\mathrm{SpecEyes}}] = c_J + \beta c_S + (1 - \beta\alpha) L_{\mathrm{agent}}
$$
由于$c_J + \beta c_S \ll L_{\mathrm{agent}}$，当$\beta\alpha$较大时（例如>0.6），期望延迟由轻量前端开销主导，可实现大幅加速。

### 4.2.4. 基于答案可分性的认知门控
认知门控是SpecEyes的核心组件，决定了投机答案的接受准确率，本文提出的答案可分性得分解决了传统概率置信度的缺陷。
#### 传统概率置信度的局限性
传统的序列生成置信度通过每个词元的最大softmax概率的几何平均计算：
首先定义第$n$个词元的最大softmax概率：
$$
p_{\mathrm{max}}^{(n)} = \max_{v \in \mathcal{V}} \sigma(\ell^{(n)})_v
$$
其中$\sigma(\cdot)$是softmax函数，$\mathcal{V}$是词表。整体置信度为：
$$
S_{\mathrm{log}}(\hat{y}_S) = \exp\left( \frac{1}{|\hat{y}_S|} \sum_{n=1}^{|\hat{y}_S|} \log p_{\mathrm{max}}^{(n)} \right)
$$
该方法存在两个明显缺陷：
1. softmax本身存在校准问题，大的logit幅度会导致过度自信的概率估计；
2. 标点、格式等低熵词元的$p_{\mathrm{max}}$本来就很高，几何平均没有显式衡量top预测与其他强竞争者的差距，容易误接受错误答案。

#### 答案可分性得分
本文设计的答案可分性得分直接基于logit的相对分布，而非softmax概率：
首先将第$n$个生成词元的logit降序排序：$\ell_{[1]}^{(n)} \geq \ell_{[2]}^{(n)} \geq \dots \geq \ell_{[|\mathcal{V}|]}^{(n)}$，其中$\ell_{[1]}^{(n)}$是最大的logit，对应预测的词元。token级可分性定义为：
$$
S_{\mathrm{sep}}^{(n)} = \frac{\ell_{[1]}^{(n)} - \mu_K^{(n)}}{\sigma_K^{(n)} + \epsilon}
$$
符号解释：
- $\mu_K^{(n)}$：前$K$个最大logit的均值
- $\sigma_K^{(n)}$：前$K$个最大logit的标准差
- $\epsilon > 0$：防止除零的小常数，默认取$10^{-6}$
  该得分的优势：
1. **尺度不变**：分子分母都随logit幅度线性缩放，消除了softmax的校准伪影；
2. **显式建模竞争关系**：通过方差项$\sigma_K^{(n)}$衡量top候选之间的区分度，得分越高说明预测词元与其他竞争者的差距越大，置信度越高。

#### 答案级聚合策略
需要将token级的可分性得分聚合为答案级的置信度，本文提出三种聚合策略：
$$
S_{\mathrm{sep}}^{\mathrm{mean}} = \frac{1}{|\hat{y}_S|} \sum_{n=1}^{|\hat{y}_S|} S_{\mathrm{sep}}^{(n)}, \quad S_{\mathrm{sep}}^{\mathrm{min}} = \min_{n \in [|\hat{y}_S|]} S_{\mathrm{sep}}^{(n)}, \quad S_{\mathrm{sep}}^{\mathrm{bottom}} = \frac{1}{|\mathcal{B}|} \sum_{n \in \mathcal{B}} S_{\mathrm{sep}}^{(n)}
$$
符号解释：
- $\mathcal{B}$：得分最低的$r$比例的token集合，$r$是经验值，默认取0.2
- 聚合后的得分通过sigmoid函数归一化到[0,1]区间

  本文默认采用$\boldsymbol{S_{\mathrm{sep}}^{\mathrm{min}}}$（最小得分聚合），基于风险理论的论证：
> 命题1：令$\hat{y}_S = (y_1, \dots, y_{|\hat{y}_S|})$为投机答案，定义答案错误事件$\mathcal{E} = \bigcup_n \mathcal{E}_n$，其中$\mathcal{E}_n$是第$n$个token错误的事件，则$P(\mathcal{E}) \leq \sum_n P(\mathcal{E}_n)$。如果每个$P(\mathcal{E}_n)$随$S_{\mathrm{sep}}^{(n)}$单调递减，那么对$\min_n S_{\mathrm{sep}}^{(n)}$做阈值判断，可以保证每个token的置信度都超过阈值，最紧地限制错误的并集概率。

最小得分聚合是保守设计，优先避免误接受错误答案，保证智能体基线的精度。

### 4.2.5. 异构并行漏斗提升吞吐
SpecEyes通过异构并行架构将单查询的延迟节省转化为系统级的吞吐提升，批量$B$个查询的处理流程如下：
1. Phase I 批量并行工具必要性判断，开销为$c_J$，得到$\beta B$个不需要工具的查询，$(1-\beta)B$个需要工具的查询；
2. Phase II 批量并行处理$\beta B$个查询，开销为$c_S$，得到$\alpha\beta B$个接受的答案直接返回，$(1-\alpha)\beta B$个被拒绝的查询；
3. 剩余的残差集合$\mathcal{R} = (1-\beta)B + (1-\alpha)\beta B = (1 - \beta\alpha)B$个查询，交给$\mathcal{M}_L$串行执行智能体流程。

   由于$c_J + c_S \ll B\bar{L}_{\mathrm{agent}}$（$\bar{L}_{\mathrm{agent}}$为平均单查询智能体延迟），批量处理时间由残差集合的串行执行时间主导，因此吞吐加速比为：
$$
\Theta_{\mathrm{SpecEyes}} / \Theta_{\mathrm{agent}} \approx 1/(1 - \beta\alpha)
$$
吞吐加速比由工具筛选比例$\beta$和门控接受率$\alpha$共同决定。

# 5. 实验设置
## 5.1. 数据集
实验选用三个覆盖不同场景的多模态基准，全面验证方法的泛化性：
1. **V* Bench**：细粒度视觉推理基准，包含两个多选子集：
   - Direct Attributes：115道题，测试属性识别能力
   - Relative Position：76道题，测试空间推理能力
2. **HR-Bench**：高分辨率感知基准，包含4K和8K两个子集，各800道题，测试高分辨率图像的理解能力，这类任务通常需要频繁调用缩放、裁剪工具。
3. **POPE**：幻觉探测基准，包含Adversarial、Popular、Random三个拆分，各3000道题，为yes/no二分类任务，测试模型的幻觉鲁棒性。
## 5.2. 评估指标
### 5.2.1. 准确率（Accuracy, Acc.）
1. **概念定义**：衡量模型回答正确的样本占总样本的比例，量化模型的推理正确性，越高越好。
2. **数学公式**：
   $$
\mathrm{Acc} = \frac{N_{\mathrm{correct}}}{N_{\mathrm{total}}}
$$
3. **符号解释**：
   - $N_{\mathrm{correct}}$：回答正确的样本数
   - $N_{\mathrm{total}}$：测试集总样本数
### 5.2.2. 加速比（Speedup, Spd.）
1. **概念定义**：基线智能体模型的平均端到端延迟与SpecEyes的平均端到端延迟的比值，量化效率提升，大于1表示比基线快。
2. **数学公式**：
   $$
\mathrm{Spd.} = \frac{L_{\mathrm{baseline}}}{L_{\mathrm{SpecEyes}}}
$$
3. **符号解释**：
   - $L_{\mathrm{baseline}}$：基线智能体模型的平均单查询延迟
   - $L_{\mathrm{SpecEyes}}$：SpecEyes的平均单查询延迟
## 5.3. 对比基线
论文选用三类代表性基线做对比：
1. **纯小模型基线**：Qwen3-VL-2B，无工具的轻量多模态大模型，作为速度上限参考，衡量SpecEyes能拿到多少潜在加速收益。
2. **智能体基线**：DeepEyes、Thyme，两个当前主流的开源智能体多模态大模型，每个查询最多执行5步工具调用，作为精度和延迟的基准。
3. **现有高效推理基线**：SpecReason，词元级投机推理的代表性工作，对比不同层级投机方法的效率和精度。
## 5.4. 实现细节
- 所有模型使用贪心解码（温度=0），延迟统计包含工具执行时间；
- 认知门控默认设置$K=64$，$\epsilon=10^{-6}$，采用min-token聚合，bottom聚合的底部比例$r=0.2$；
- 门控阈值通过在每个基准上运行小模型收集置信度分布后校准，离线校准仅需5~10分钟；
- 所有实验在单张NVIDIA A100 40GB GPU上运行。

### 结果分析
1. **基于DeepEyes backbone的结果**：
   - SpecEyes（min聚合）实现平均1.73倍加速，同时平均准确率从81.39%提升到84.26%；
   - V* Bench：Direct Attributes上与基线准确率一致（90.43%），1.53倍加速；Relative Position上准确率从82.89%提升到89.47%，1.90倍加速；
   - POPE：加速比最高（2.13~2.19倍），准确率均明显高于基线，比如Adversarial拆分从78.43%提升到85.13%，说明绕过不必要的工具链可以减少多步推理的累积误差，降低幻觉；
   - HR-Bench：加速比中等（1.08~1.13倍），因为高分辨率任务更多需要工具调用，$\beta$和$\alpha$都较低。
2. **基于Thyme backbone的结果**：
   - 与DeepEyes的趋势一致，平均1.42倍加速，平均准确率从82.29%提升到83.99%，证明方法的泛化性；
   - HR-Bench 8K子集的加速比略低于1，因为该场景下$\beta\alpha$极低，小模型的固定开销超过了节省的延迟，符合延迟公式的预期。
3. **与基线对比**：
   - SpecReason反而比基线慢（速度仅为基线的0.37~0.61倍），因为小模型没有结构化工具调用能力，引入了大量的token和轮次开销，且POPE上准确率严重下降；
   - 纯小模型平均4.13倍加速，但平均准确率仅78.93%，远低于基线，SpecEyes拿到了大部分的加速收益，同时精度甚至超过基线。

# 7. 总结与思考
## 7.1. 结论总结
本文提出的SpecEyes是首个智能体级别的投机加速框架，将投机范式从词元级提升到整个智能体流程级别：
- 用轻量无工具小模型处理不需要多步工具的查询，基于答案可分性的认知门控无需标注即可实现可靠的置信度判断；
- 异构并行漏斗架构将单查询的延迟节省转化为系统级的吞吐提升；
- 在三个基准上实现1.1~3.35倍的端到端加速，同时精度最高提升6.7%，解决了智能体多模态大模型落地的延迟和并发瓶颈。
## 7.2. 局限性与未来工作
作者指出的局限性：当前的投机模型仅能处理智能体深度$D=0$的场景（完全不需要工具），在HR-Bench这类大部分查询需要工具的基准上加速比有限。
未来工作方向：实现多深度投机，允许投机模型调用有限次轻量工具，在足够的推理深度就返回结果，进一步减少回退到大模型的比例。
## 7.3. 个人启发与批判
### 启发
1. **范式创新的价值**：效率优化不需要局限在现有流程内抠细节，跳出固有假设，从更高的层级（本文从词元级到智能体级）找优化空间，往往能带来量级的收益；
2. **双系统架构的通用性**：认知科学的双系统理论在AI系统设计中有很高的参考价值，将简单任务交给小模型，复杂任务交给大模型，可以很好地平衡效率和精度，该思路可以迁移到语音、自然语言处理等其他领域；
3. **无标注置信度的实用价值**：本文提出的答案可分性得分不需要额外标注，仅依赖模型输出的logit，落地成本低，可广泛应用于需要早退出、动态路由的场景。
### 潜在改进点
1. 当前Phase I的工具必要性判断用大模型$\mathcal{M}_L$实现，虽然开销低，但如果能将该判断交给小模型同时保持准确率，可以进一步降低前端开销；
2. 门控阈值需要在每个数据集上校准，未来可以设计自适应阈值机制，无需针对每个场景手动调参；
3. 多深度投机的思路很有前景，但需要解决轻量工具调用的开销问题，避免引入额外的延迟抵消收益；
4. 当前实验仅在单GPU上开展，分布式部署场景下的吞吐优化还需要进一步验证。
