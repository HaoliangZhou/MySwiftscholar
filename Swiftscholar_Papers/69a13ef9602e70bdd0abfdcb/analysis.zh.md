# 1. 论文基本信息

## 1.1. 标题
**WISER: Wider Search, Deeper Thinking, and Adaptive Fusion for Training-Free Zero-Shot Composed Image Retrieval**
（WISER：用于无需训练的零样本组合图像检索的更广搜索、更深思考与自适应融合）

## 1.2. 作者
Tianyue Wang, Leigang Qu, Tianyu Yang, Xiangzhao Hao, Yifan Xu, Haiyun Guo, Jinqiao Wang。
作者隶属于中国科学院自动化研究所（CASIA）、中国科学院大学（UCAS）、新加坡国立大学（NUS）以及中央民族大学。他们在计算机视觉和多模态学习领域具有深厚的研究背景。

该论文发表于 **arXiv**（预印本平台），日期为 2026年2月26日。考虑到其显著的性能提升和严谨的实验，该工作具有极高的学术参考价值。

## 1.4. 发表年份
2026年（UTC 时间 2026-02-26）。

## 1.5. 摘要
组合图像检索 (Composed Image Retrieval, CIR) 要求系统根据“参考图像 + 修改文本”的多模态查询来检索目标图像。现有的零样本 (Zero-Shot) 方法通常将其简化为单一模态任务（要么是文本到图像 T2I，要么是图像到图像 I2I），但这会导致细节丢失或语义理解不足。本文提出了 **WISER**，这是一个无需训练的框架，通过“检索—验证—细化”管道统一了 T2I 和 I2I。它通过并行搜索、基于验证器的置信度评估以及基于大语言模型的自我反思（更深思考）来动态融合两种范式的优势。实验表明，WISER 在多个基准测试上大幅超越了现有的无需训练方法，甚至优于许多需要训练的方法。

# 2. 整体概括

## 2.1. 研究背景与动机
*   **核心问题:** 组合图像检索 (Composed Image Retrieval, CIR) 的目标是找到一张既符合参考图像视觉特征，又满足修改文本语义要求的图像。传统的监督学习方法依赖昂贵的“图像对+文本”标注数据，难以扩展。
*   **现有挑战:**
    1.  <strong>文本到图像 (T2I) 范式:</strong> 将图像转为文字再修改，容易丢失细微的视觉特征（如衣服的纹理）。
    2.  <strong>图像到图像 (I2I) 范式:</strong> 利用扩散模型修改图像，在处理复杂的语义逻辑（如“把背景换成森林并增加一只猫”）时表现较差。
    3.  **缺乏意图与不确定性意识:** 现有的融合方法通常使用固定权重，无法根据用户的具体意图（是侧重视觉修改还是语义修改）动态调整。
*   **创新思路:** 既然 T2I 和 I2I 互补，能否建立一个闭环系统，先拓宽搜索范围，再通过一个“验证者”评估结果的可靠性，如果结果不理想，则让模型进行“自我反思”并重新搜索？

## 2.2. 核心贡献/主要发现
*   **提出了 WISER 框架:** 这是一个<strong>无需训练 (Training-Free)</strong> 的框架，意味着它可以直接利用现有的预训练模型（如 CLIP、大语言模型），具有极强的通用性。
*   <strong>检索—验证—细化 (Retrieve-Verify-Refine) 管道:</strong> 显式地对用户意图和结果不确定性进行建模。
*   <strong>更深思考 (Deeper Thinking):</strong> 引入结构化的自我反思机制，通过 LLM 分析检索失败的原因并提供改进建议。
*   **性能突破:** 在 CIRR 数据集上 Recall@1 提升了 57%，在 CIRCO 数据集上 mAP@5 提升了 45%，刷新了零样本组合图像检索的纪录。

# 3. 预备知识与相关工作

## 3.1. 基础概念
*   <strong>零样本组合图像检索 (Zero-Shot CIR, ZS-CIR):</strong> 在没有任何专门针对该任务标注的训练数据的情况下，实现根据图像和文本查询目标图像的任务。
*   <strong>对比语言-图像预训练模型 (Contrastive Language-Image Pre-training, CLIP):</strong> 一种将图像和文本映射到同一个向量空间的技术。其核心公式是计算图像向量 $v$ 和文本向量 $t$ 的余弦相似度：
    $$
    \text{Similarity}(v, t) = \frac{v \cdot t}{\|v\| \|t\|}
    $$
    WISER 利用 CLIP 强大的对齐能力进行最终的检索分发。
*   <strong>多模态大语言模型 (MLLM):</strong> 如 Qwen2-VL，能够同时理解图像和文本内容。本文将其用作“验证器”。

## 3.2. 前人工作
*   **Pic2Word / SEARLE:** 这些方法通过训练一个特殊的模块将图像转化为单词（Textual Inversion），属于 T2I 范式。
*   **CIReVL / CoTMR:** 利用大语言模型进行链式思考推理来生成目标描述，擅长语义理解但视觉细节捕捉能力有限。
*   **CompoDiff / IP-CIR:** 使用扩散模型生成编辑后的图像，属于 I2I 范式，视觉效果好但语义逻辑容易出错。

## 3.3. 技术演进与差异化分析
早期的工作试图寻找一种统一的表示，但 WISER 认为“没有一种表示能完美涵盖所有意图”。因此，WISER 不再纠结于寻找单一的最优特征，而是通过**并行双路检索 + 动态融合**的方式，让系统具备了根据输入自动切换侧重点的能力。

# 4. 方法论

WISER 的核心架构由三个关键模块组成：<strong>更广搜索 (Wider Search)</strong>、<strong>自适应融合 (Adaptive Fusion)</strong> 和 <strong>更深思考 (Deeper Thinking)</strong>。

*该图像是示意图，展示了WISER框架在零样本组合图像检索中的工作流程。图中包括三个主要步骤：首先通过文本和图像编码器生成候选图像，然后通过验证器分析结果，最后提供改进建议以优化后续检索流程。每个步骤都对应具体操作，强调了更深层次思考与自适应融合的重要性。*

## 4.1. 更广搜索 (Wider Search): 双路检索
为了不漏掉任何潜在目标，WISER 同时开启 T2I 和 I2I 两条检索路径。

1.  **T2I 路径:** 首先使用字幕生成器（Captioner）将参考图像 $I_{ref}$ 转为文字 $C_{ref}$，然后利用编辑器 $\mathcal{F}$ 生成编辑后的标题 $C_{edit}$：
    $$
    C_{edit} = \mathcal{F}_{txt}(C_{ref}, T_{mod})
    $$
    其中 $T_{mod}$ 是用户的修改指令。
2.  **I2I 路径:** 直接利用图像编辑模型生成编辑后的图像 $I_{edit}$：
    $$
    I_{edit} = \mathcal{F}_{img}(I_{ref}, T_{mod})
    $$
3.  **双路检索与候选池聚合:** 分别计算 $C_{edit}$ 和 $I_{edit}$ 的 CLIP 特征，在数据库中检索前 $K$ 个结果，形成并集候选池 $\mathcal{R}_{union} = \mathcal{R}_{T2I} \cup \mathcal{R}_{I2I}$。

## 4.2. 自适应融合 (Adaptive Fusion): 验证导向的整合
在得到候选池后，WISER 并不简单地相加得分，而是引入了一个<strong>验证器 (Verifier)</strong> $\Phi$（一个多模态大语言模型）。

### 4.2.1. 验证评分
对于候选池中的每个图像 $I_p^k$，验证器需要回答一个二分类问题：“候选图像是否符合修改指令？”。根据输出的 Logits（未归一化的概率得分）计算置信度得分 $c_p^k$：
$$
c_p^k = \frac{\exp(\ell_{p,k}^{(yes)})}{\exp(\ell_{p,k}^{(yes)}) + \exp(\ell_{p,k}^{(no)})}
$$
其中 $\ell_{p,k}^{(yes)}$ 和 $\ell_{p,k}^{(no)}$ 分别代表“是”和“否”的得分。$c_p^k$ 越高，说明该图像越符合用户意图。

### 4.2.2. 多级融合策略
*   **分支级不确定性感知:** 系统会找到每条路径中得分最高的“伪目标” $I_p^*$，并获取其可靠性得分 $r_p = \max_k c_p^k$。如果 $\min(r_{T2I}, r_{I2I}) < \tau$（$\tau$ 为预设阈值），说明检索结果不可靠，将触发**更深思考**模块。
*   **候选级意图感知:** 如果结果可靠，则计算融合得分 $c_{fused}^k = c_{T2I}^k + c_{I2I}^k$。为了进行最终排名，WISER 采用了一种**字典序排序函数** $\Psi$：
    $$
    \Psi(I^k) = (-c_{fused}^k, -\max(c_{T2I}^k, c_{I2I}^k), -c_{T2I}^k)
    $$
    这个公式的逻辑是：首先按融合得分排；如果融合得分一样，看谁单路得分最高；如果还一样，看 T2I 得分。这保证了排名不仅考虑了双路的共识，还兼顾了单路的突出表现。

## 4.3. 更深思考 (Deeper Thinking): 结构化自我反思
当验证器认为当前的检索结果不理想时，会调用 LLM 作为<strong>细化器 (Refiner)</strong> 执行以下步骤：
1.  **识别修改点:** 分析 $C_{ref}$ 和 $T_{mod}$，提炼出具体的“属性变化”或“实体增删”。
2.  **分析现有结果:** 将伪目标的描述与上述修改点对比，找出哪里没对上。
3.  **提供细化建议:** 生成针对性的提示词（例如：“在标题中强调红色的质感”或“在图像生成中修正猫的位置”）。
4.  **循环迭代:** 将建议反馈给编辑器 $\mathcal{F}$ 重新生成更准确的 $C_{edit}$ 或 $I_{edit}$，进入下一轮检索。

# 5. 实验设置

## 5.1. 数据集
*   **Fashion-IQ:** 专注于时尚领域，包含裙子、衬衫、上衣三类，要求对精细属性（如领口、颜色）进行检索。
*   **CIRR:** 首个自然场景图像 CIR 数据集，来源于真实生活，噪声较多。
*   **CIRCO:** 最新的基准测试，每个查询提供多个<strong>真实标注数据 (Ground Truth)</strong>，更符合实际检索场景。

## 5.2. 评估指标
论文使用了以下标准指标：
1.  **Recall@K (R@K):**
    *   **概念定义:** 衡量在前 $K$ 个检索结果中，包含正确目标图像的比例。
    *   **数学公式:** $R@K = \frac{1}{N} \sum_{i=1}^N \mathbb{I}(\text{rank}_i \le K)$
    *   **符号解释:** $N$ 是查询总数，$\text{rank}_i$ 是第 $i$ 个查询的正确目标在检索结果中的排名，$\mathbb{I}(\cdot)$ 是指示函数（成立为 1，不成立为 0）。
2.  **mAP@K (Mean Average Precision):**
    *   **概念定义:** 衡量检索结果的整体排序质量，特别是当存在多个正确目标时。
    *   **数学公式:** `mAP@K = \frac{1}{N} \sum_{i=1}^N \left( \frac{1}{M_i} \sum_{j=1}^{\min(K, R_i)} P(j) \cdot rel(j) \right)`
    *   **符号解释:** $M_i$ 是第 $i$ 个查询的目标总数，$R_i$ 是检索到的目标数，`P(j)` 是前 $j$ 个结果的精准率，`rel(j)` 是第 $j$ 个结果是否正确的二值标签。

## 5.3. 对比基线
WISER 与多类模型进行了对比：
*   **文本反转类:** SEARLE, LinCIR (通常需要少量数据微调)。
*   **训练免除类:** CIReVL, LDRE, AutoCIR, CoTMR (目前最先进的竞争对手)。
*   **I2I 增强类:** IP-CIR。

**分析点:**
1.  **巨大领先:** 在 ViT-B/32 下，WISER 在 CIRR 的 R@1 指标上从 31.50% 提升到了 49.45%，相对提升超过 50%。这说明“验证+细化”的逻辑极大减少了检索中的随机干扰。
2.  **超越有监督:** WISER（无需训练）甚至超过了许多需要训练的方法（如 LinCIR），证明了通用多模态模型蕴含的知识足以处理复杂的 CIR 任务，关键在于如何启发式地挖掘这些知识。

Table 3 证明了各个组件的有效性：

# 7. 总结与思考

## 7.1. 结论总结
WISER 证明了在零样本组合图像检索中，<strong>“博采众长”</strong>（T2I + I2I）以及<strong>“知错就改”</strong>（验证 + 反思）的重要性。它打破了以往研究中只关注如何表示图像和文本的局限，转而关注如何构建一个更智能、更具交互性的检索管道。

## 7.2. 局限性与未来工作
*   **计算开销:** 虽然不需要训练，但每一轮“检索—验证—细化”都需要调用 LLM 和 MLLM，这在实时检索系统中可能会带来较高的延迟。
*   **模型依赖:** 性能高度依赖于编辑器和验证器的质量。
*   **未来方向:** 作者提出可以将这种“检索—验证—细化”的范式扩展到视频检索或 3D 物体检索等更广阔的领域。

## 7.3. 个人启发与批判
*   **启发:** WISER 的成功很大程度上归功于它将复杂的感知问题转化为一个逻辑判断问题（由验证器完成）。这种“感知（搜索）+ 认知（验证/反思）”的结合是未来 AI 系统的重要趋势。
*   **批判:** 尽管论文声称是“训练免除 (Training-Free)”，但在实际部署中，调用 GPT-4o 或 Qwen2-VL 的推理成本可能比训练一个小型专用模型还要贵。未来的研究可以探索如何将这种多模态知识<strong>蒸馏 (Distillation)</strong> 到更小、更快的本地模型中，以实现工程上的落地。此外，字典序排序 $\Psi$ 虽然有效，但本质上还是基于启发式的硬编码逻辑，如果能学习一个动态的融合权重函数，可能会进一步提升鲁棒性。
