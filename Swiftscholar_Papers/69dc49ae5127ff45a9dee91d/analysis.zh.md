## 1.5. 摘要
现有的利用冻结的大型语言模型进行视觉理解的方法（通常通过图像描述或图像-文本检索任务）在处理复杂的多模态场景时面临挑战。为了增强多模态大型语言模型理解视觉和语言上下文的能力，本文引入了**多模态组合学习**，旨在映射或对齐视觉和语言输入。具体而言，本文提出了两个任务：**多模态上下文描述** 和 **多模态上下文检索**，以引导冻结的 LLM 理解视觉和语言上下文。这些专门设计的任务旨在提高 LLM 高效处理和利用多模态输入的能力，从而提升其生成更准确的文本或视觉表示的熟练度。在检索任务（如零样本组合图像检索、视觉故事讲述图像检索和视觉对话图像检索）和文本生成任务（如视觉问答）上的广泛实验证明了所提方法的有效性。

# 2. 整体概括

## 2.1. 研究背景与动机
**核心问题：**
现有的多模态大型语言模型通常通过简单的“模态翻译”任务（如图像描述生成或图像-文本检索）来训练视觉与语言之间的映射。虽然这些方法在标准任务上表现出色，但在需要深度理解多模态上下文的复杂场景中（例如，根据一张参考图片和一个文本修改指令来检索目标图片），它们往往表现不佳。这是因为简单的翻译任务未能充分促进跨模态之间的深度交互和组合能力。

**重要性：**
随着多模态应用（如视觉搜索、视觉问答、图像编辑）的普及，模型不仅需要“看懂”图片或“读懂”文字，更需要能够根据组合的指令（图片+文字）进行推理和检索。这是通向更智能、更通用的多模态智能体的关键一步。

**现有挑战：**
1.  **数据匮乏：** 训练多模态组合能力通常需要人工标注的三元组（参考图、文本条件、目标图），成本高昂且难以扩展。
2.  **能力局限：** 现有的基于 CLIP 文本编码器的方法在理解复杂的逻辑关系、词序和结构化指令方面存在局限，而 LLM 虽然语言能力强，但往往缺乏有效的视觉-语言组合训练机制。

**切入点：**
本文提出利用 LLM 本身强大的语言生成能力来自动构建大规模的训练数据（MMC 数据集），并设计新的训练目标（MCL），让冻结的 LLM 在其内部空间中学会如何“组合”视觉和语言信息，从而解决复杂的多模态上下文理解问题。

## 2.2. 核心贡献/主要发现
1.  <strong>提出了多模态组合学习（MCL）方法：</strong> 这是一种新的视觉-语言映射范式，不同于传统的模态翻译。MCL 能够有效地使冻结的 LLM 在各种多模态上下文中执行准确的图像检索和文本生成。
2.  **构建了 MMC 数据集：** 利用 LLM 自动增强现有的网络收集的图像-文本对，构建了包含 270 万个四元组（参考图、参考描述、文本条件、目标描述）的大规模数据集。
3.  **提出了堆叠检索机制：** 为了从 LLM 的多模态上下文中提取多样化的信息，提出了一种堆叠式的检索令牌机制，解决了传统顺序令牌因自回归特性导致的注意力聚焦单一问题。
4.  **实验验证：** 在四个零样本多模态上下文理解任务（组合图像检索、视觉故事讲述、视觉对话、视觉问答）上的广泛实验表明，MCL 显著优于现有的最先进方法。

# 3. 预备知识与相关工作

## 3.1. 基础概念
为了理解本文，读者需要掌握以下核心概念：

*   **冻结的大型语言模型：** 指在训练过程中参数保持不变的预训练语言模型（如 LLaMA, OPT）。这类模型拥有强大的通用语言理解能力，但通常只能处理文本输入。本文旨在通过添加适配器层使其具备处理视觉输入的能力，而不更新其内部权重。
*   <strong>CLIP (Contrastive Language-Image Pre-training)：</strong> 一种由 OpenAI 提出的视觉-语言模型，通过对比学习将图像和文本映射到共享的潜在空间。在本文中，CLIP 被用作图像编码器和视觉特征空间的基准。
*   **组合图像检索：** 一种检索任务，旨在根据一个“参考图像”和一个描述修改或相对关系的“文本条件”来检索“目标图像”。例如：给定一张“红色汽车”的图片和文本“换成蓝色的”，检索出一张“蓝色汽车”的图片。
*   **适配器/线性映射层：** 插入在冻结模型之间的可训练小模块，用于将不同模态的特征（如 CLIP 视觉特征）映射到目标模型（如 LLM）的特征空间中。

## 3.2. 前人工作
作者在相关工作部分主要回顾了以下三个方向：

1.  **视觉-语言映射：**
    *   **前人方法：** 早期工作主要利用图像描述任务将视觉特征映射到 LLM 空间（如 Frozen, Flamingo），或者利用检索任务将 LLM 表示映射回视觉特征空间（如 FROMAGe）。
    *   **本文区别：** 本文认为这些方法主要关注“翻译”（Image -> Text 或 Text -> Image），而忽略了“组合”。本文提出的 MCL 任务要求模型综合视觉和语言信息来生成目标，更强调跨模态的交互。

2.  <strong>组合图像检索 (CIR)：</strong>
    *   **前人方法：** 依赖人工标注的三元组进行监督训练；或者使用零样本方法，如将图像映射为伪文本词元，然后在 CLIP 文本编码器中进行组合。
    *   **本文区别：** 本文直接在冻结的 LLM 空间内进行组合，利用 LLM 强大的语义理解能力，避免了 CLIP 文本编码器在处理复杂逻辑时的不足。

3.  **利用 LLM 进行多模态数据增强：**
    *   **前人方法：** 使用 LLM 生成图像编辑指令或扩展数据集。
    *   **本文区别：** 本文利用 LLM 生成用于“多模态组合学习”的特定四元组数据（参考图、参考描述、文本条件、目标描述），专门服务于提升模型的上下文理解能力。

## 3.3. 技术演进
该领域从早期的简单特征拼接，发展到利用对比学习（如 CLIP）对齐视觉和语言空间。随着 LLM 的兴起，研究重点转向如何将视觉信号注入到 LLM 中。初期工作主要关注单模态输入的理解（如看图说话），近期趋势开始转向处理更复杂的、多模态交织的输入（如视觉对话、组合检索）。本文的工作处于这一技术脉络的前沿，试图解决更深层次的“组合”推理问题。

## 3.4. 差异化分析
本文方法与 FROMAGe 等代表性工作的核心区别在于训练目标。FROMAGe 使用标准的图像描述和图像-文本检索损失，这相当于教模型“看图说话”或“听声辨位”。而本文引入的 **MC-Cap** 和 **MC-Ret** 损失，则是教模型“根据图 A 和文本 B，描述或找到图 C”。这种基于“条件”和“参考”的生成/检索任务，迫使模型学习更精细的跨模态对齐和推理能力。

# 4. 方法论

## 4.1. 方法原理
本文提出的 <strong>多模态组合学习 (MCL)</strong> 框架旨在通过两个核心步骤增强冻结 LLM 的多模态上下文理解能力：
1.  **数据生成：** 利用 LLM 自动构建大规模的多模态组合数据（MMC 数据集）。
2.  **模型训练：** 设计新的训练目标（MC-Cap 和 MC-Ret），配合堆叠检索机制，使 LLM 学会在其内部空间中组合视觉和语言信息。

## 4.2. 核心方法详解

### 4.2.1. 数据生成
为了解决多模态组合数据稀缺的问题，本文提出利用现成的 LLM（如 Llama2）来自动生成数据。

**流程步骤：**
1.  **输入：** 给定一个从网络收集的图像-描述对 $\langle I_{ref}, T_{refc} \rangle$。
2.  **生成条件：** 将参考描述 $T_{refc}$ 输入到 LLM 中，并附带特定的提示词，要求 LLM 生成一个文本条件 $T_{con}$。这个条件通常是一个编辑指令（如“加上一个玩具鼠标”）或描述差异的短语。
3.  **生成目标描述：** 接着，要求 LLM 结合参考描述 $T_{refc}$ 和文本条件 $T_{con}$，生成一个目标描述 $T_{tgtc}$。
4.  **输出：** 最终得到一个四元组 $\langle I_{ref}, T_{refc}, T_{con}, T_{tgtc} \rangle$。

**关键点：**
*   我们不需要生成目标描述对应的真实图像，而是利用 CLIP 提取目标描述的文本特征作为视觉监督信号。这大大降低了数据收集的难度。

### 4.2.2. 映射视觉输入到 LLMs
为了使冻结的 LLM 能够处理图像，本文采用了一个线性映射层（适配器）。

**步骤 1：特征提取与映射**
给定输入图像 $I$，首先使用冻结的 CLIP 视觉编码器 $E_{image}$ 提取视觉特征。然后，通过一个可训练的线性映射层 $f_{map}$ 将其映射到 LLM 的词嵌入空间，得到 $n$ 个视觉向量 $\mathbf{V}$：
$$ \mathbf{V} = [v_0, v_1, ..., v_n] = f_{map}(E_{image}(I)) $$

**步骤 2：朴素图像描述目标**
作为基线，传统的训练目标是最大化生成正确描述的条件概率。给定视觉向量 $\mathbf{V}$ 和之前的词元，预测下一个词元 $t_i$。其损失函数 $\mathcal{L}_{Cap}$ 定义为：
$$ \mathcal{L}_{Cap}(\theta_m) = - \frac{1}{|t|} \sum_{i=1}^{|t|} \log P \Big( t_i | \mathbf{V}, t_{<i} \Big) $$
其中：
*   $\theta_m$：映射层 $f_{map}$ 的权重。
*   $t_i$：描述中的第 $i$ 个词元。
*   $P$：冻结的 LLM 模型。
*   $t_{<i}$：第 $i$ 个词元之前的所有词元序列。

<strong>步骤 3：多模态上下文描述 (MC-Cap) 目标</strong>
这是本文提出的核心目标之一。不同于朴素描述，这里模型需要根据**视觉向量**、**文本条件**和**已生成的目标描述词元**来预测下一个目标描述词元。
给定数据三元组 $\langle I_{ref}, T_{con}, T_{tgtc} \rangle$，损失函数 $\mathcal{L}_{MC-Cap}$ 定义为：
$$ \mathcal{L}_{MC-Cap}(\theta_m) = - \frac{1}{|t|} \sum_{i=1}^{|t|} \log P \Big( t_i | \mathbf{V}, c_1, ..., c_{|c|}, t_{<i} \Big) $$
其中：
*   $c_i$：文本条件 $T_{con}$ 中的第 $i$ 个词元。
*   $t_i$：目标描述 $T_{tgtc}$ 中的第 $i$ 个词元。
*   $\mathbf{V}$：参考图像 $I_{ref}$ 映射后的视觉向量。

**优势分析：**
相比于朴素描述，MC-Cap 强制模型在生成目标描述时，必须“关注”并“理解”输入的文本条件 $T_{con}$。这意味着映射层 $f_{map}$ 学习到的视觉特征必须能够与文本条件进行有效的交互，从而实现了“文本感知”的视觉映射。

### 4.2.3. 从 LLMs 提取视觉表示
为了执行检索任务，模型需要能够从 LLM 的内部状态中提取出视觉表示。本文使用可学习的特殊令牌 `[RET]` 来实现这一点。

**步骤 1：朴素图像-文本检索目标**
在 FROMAGe 等工作中，`[RET]` 令牌被附加在文本令牌之后。LLM 处理完输入后，`[RET]` 令牌的最后一个隐藏状态 $h([\text{RET}] | \mathbf{T})$ 被视为上下文的摘要。通过一个线性投影层 $f_{proj}$ 将其投影到 CLIP 的潜在空间，得到 $p_v$。然后使用 InfoNCE 损失与目标图像的 CLIP 特征对齐。
$$ \mathcal{L}_{Ret}([\text{RET}], \theta_p) = - \frac{1}{N} \sum_{i=1}^{N} \left( \log \frac{\exp(\sin(p_v, \mathbf{e}_i) / \tau)}{\sum_{j=1}^{N} \exp(\sin(p_v, \mathbf{e}_j) / \tau)} \right) $$
其中：
*   $\theta_p$：投影层 $f_{proj}$ 的权重。
*   $\sin(\cdot, \cdot)$：余弦相似度函数。
*   $\mathbf{e}_i$：第 $i$ 个图像的 CLIP 视觉特征。
*   $\tau$：温度参数，用于控制 softmax 的尖锐程度。

<strong>步骤 2：多模态上下文检索 (MC-Ret) 目标</strong>
本文发现，仅用朴素检索目标训练时，`[RET]` 令牌主要充当“文本摘要”的角色，缺乏根据多模态上下文提取特定信息的能力。因此，提出了 MC-Ret 目标。
在此任务中，输入包含参考图像 $I_{ref}$ 和文本条件 $T_{con}$，目标是匹配目标描述 $T_{tgtc}$ 的 CLIP 特征。
$$ \mathcal{L}_{MC-Ret}([\text{RET}], \theta_p, \theta_m) = - \frac{1}{N} \sum_{i=1}^{N} \left( \log \frac{\exp(\sin(p_v, \mathbf{e}_i) / \tau)}{\sum_{j=1}^{N} \exp(\sin(p_v, \mathbf{e}_j) / \tau)} \right) $$
其中：
*   `p_v = f_{proj}(h(\text{[RET]} | \mathbf{V}, \mathbf{T}))`。
*   $\mathbf{V}$：参考图像的视觉特征。
*   $\mathbf{T}$：文本条件的词元序列。
*   $\mathbf{e}$：目标描述 $T_{tgtc}$ 的 CLIP 文本特征。

**优势分析：**
MC-Ret 训练 `[RET]` 令牌去“组合”视觉和文本上下文，以匹配目标特征。这使得 `[RET]` 不再是简单的摘要，而是一个能够根据指令（文本条件）从图像中提取特定特征的查询器。

### 4.2.4. 堆叠检索机制
为了提取更多样化的信息，直观的想法是增加 `[RET]` 令牌的数量。然而，由于 LLM 的自回归特性，顺序排列的多个 `[RET]` 令牌会相互影响，导致它们关注相似的内容。

**顺序排列的问题：**
$$ h_i = h([\text{RET}]_i | \mathbf{V}, \mathbf{T}, [\text{RET}]_{<i}) $$
这里，第 $i$ 个令牌的隐藏状态依赖于之前的所有令牌，导致信息冗余。

**堆叠排列的解决方案：**
本文提出堆叠机制，使得每个 `[RET]` 令牌仅依赖于原始的多模态上下文 $\mathbf{V}$ 和 $\mathbf{T}$，而彼此独立。
$$ h_i = h([\text{RET}]_i | \mathbf{V}, \mathbf{T}) $$
最终的特征 $p_v$ 通过融合函数 $f_{fusion}$ 整合所有令牌的隐藏状态：
$$ p_v = f_{fusion}(h_1, ..., h_r) $$
其中 $r$ 是 `[RET]` 令牌的数量。实现上，通过在 `[RET]` 令牌之间添加额外的注意力掩码来阻断它们之间的相互注意力。

### 4.2.5. 模型训练
最终的训练损失是上述四个目标的加权和：
$$ \mathcal{L} = \lambda_{Cap} (\mathcal{L}_{Cap} + \mathcal{L}_{MC-Cap}) + \lambda_{Ret} (\mathcal{L}_{Ret} + \mathcal{L}_{MC-Ret}) $$
其中 $\lambda_{Cap}$ 和 $\lambda_{Ret}$ 是平衡生成任务和检索任务的权重超参数。

# 5. 实验设置

## 5.1. 数据集
实验使用了以下数据集：
1.  <strong>MMC (MultiModal Composition)：</strong> 本文提出的大规模生成数据集，包含 270 万个四元组，用于训练 MCL 模型。
2.  **CIRCO：** 一个开放域的零样本组合图像检索基准数据集，包含多个真实标注。
3.  **CIRR：** 经典的组合图像检索数据集。
4.  **GeneCIS：** 通用条件图像相似性基准数据集。
5.  **Visual Storytelling：** 视觉故事讲述数据集，包含按时间顺序排列的图像-文本对。
6.  **Visual Dialog：** 视觉对话数据集，包含关于图像的对话历史。
7.  **VQAv2：** 视觉问答数据集，用于评估文本生成能力。

**数据集示例说明：**
在 Visual Storytelling 任务中，输入可能是一系列描述（“一个人走在街上”，“他停下来看手机”...）和对应的图像，模型需要根据这些上下文检索最后一张图像。这测试了模型处理密集多模态上下文的能力。

## 5.2. 评估指标
论文中主要使用了以下指标：

1.  <strong>Recall@K (召回率@K)：</strong>
    *   **概念定义：** 衡量检索系统在前 $K$ 个结果返回相关项的能力。它关注的是“相关的目标是否出现在前 K 个结果中”。
    *   **数学公式：**
        $$ \text{Recall@K} = \frac{|\{ \text{relevant items} \} \cap \{ \text{top K items} \}|}{|\{ \text{relevant items} \}|} $$
    *   **符号解释：** 分子是前 $K$ 个结果中与查询相关的项目数量，分母是所有相关项目的总数。

2.  <strong>Mean Average Precision (mAP@K)：</strong>
    *   **概念定义：** 平均精度均值，是多个查询的平均精度（Average Precision, AP）的平均值。AP 综合考虑了排序的准确性，不仅看相关项是否出现，还看它们出现在第几位。mAP@K 通常计算前 K 个结果的 AP。
    *   **数学公式：**
        $$ \text{AP} = \frac{1}{R} \sum_{k=1}^{N} P(k) \times \text{rel}(k) $$
        $$ \text{mAP} = \frac{1}{Q} \sum_{q=1}^{Q} \text{AP}(q) $$
    *   **符号解释：**
        *   $R$：相关项目的总数。
        *   $N$：检索结果的总数。
        *   `P(k)`：前 $k$ 个结果的精度。
        *   $\text{rel}(k)$：指示函数，如果第 $k$ 个结果是相关的则为 1，否则为 0。
        *   $Q$：查询的总数。

3.  <strong>Accuracy (准确率)：</strong>
    *   **概念定义：** 在 VQA 任务中，预测答案与真实标注完全匹配的比例。
    *   **数学公式：**
        $$ \text{Accuracy} = \frac{\text{Number of correct answers}}{\text{Total number of questions}} $$

## 5.3. 对比基线
论文将 MCL 与以下基线进行了比较：
*   **Image-only / Text-only / Image+Text：** 基于 CLIP 的简单基线。
*   **Pic2Word / SEARLE：** 基于 CLIP 文本编码器的零样本组合检索方法。
*   **CompoDiff：** 基于潜在扩散模型的组合检索方法。
*   **Combiner-MMC：** 在本文生成的 MMC 数据集上训练的经典 Combiner 模型。
*   **FROMAGe：** 基于 LLM 的多模态模型，使用传统的描述和检索目标训练。

# 6. 实验结果与分析

## 6.1. 核心结果分析
<strong>组合图像检索 (CIR) 结果：</strong>
在 CIRCO、CIRR 和 GeneCIS 三个数据集上，MCL 显著优于所有基线方法。特别是与 FROMAGe 相比，MCL 展现出了巨大的性能提升。这证明了简单的模态翻译任务不足以训练出强大的组合能力，而本文提出的 MCL 任务有效地解决了这一问题。
此外，MCL 在使用更强的 LLM 主干网络（如 Llama2-7B）时性能进一步提升，表明 LLM 的语义理解能力对多模态组合至关重要。

**密集多模态上下文理解结果：**
在 Visual Storytelling 和 Visual Dialog 任务中，MCL 同样表现出色。随着输入上下文密度的增加（如从 1 个 caption 增加到 5 个 captions + 4 images），MCL 的性能提升明显，而 CLIP 和 FROMAGe 的性能往往下降或提升有限。这验证了 MCL 在处理复杂、密集的多模态信息流方面的优势。

<strong>视觉问答 (VQA) 结果：</strong>
在 VQAv2 上，MCL 优于同等参数规模的基线模型（如 FROMAGe），表明多模态组合学习带来的视觉-语言对齐提升具有泛化性，有助于其他生成任务。

## 6.2. 消融实验/参数分析
**堆叠检索机制的有效性：**
消融实验表明，使用顺序排列的多个 `[RET]` 令牌时，性能反而下降。这是因为 LLM 的自回归特性导致后续令牌过度关注前面的令牌，而非原始上下文。采用堆叠机制后，性能得到恢复并提升，证明了该机制在提取多样化信息方面的有效性。

下图（原文 Figure 4）可视化了检索令牌与上下文令牌之间的相关性。可以看出，堆叠机制使得不同的检索令牌关注上下文的不同部分（如颜色、背景、物体），而顺序令牌则倾向于关注相似的内容。

# 7. 总结与思考

## 7.1. 结论总结
本文提出了一种名为多模态组合学习（MCL）的新方法，旨在提升多模态大型语言模型在复杂上下文中的理解能力。通过利用 LLM 自动生成大规模的 MMC 数据集，并设计 MC-Cap 和 MC-Ret 两个核心训练任务，配合堆叠检索机制，MCL 在零样本组合图像检索、视觉故事讲述、视觉对话和视觉问答等多个任务上取得了显著优于现有方法的性能。研究证明了 LLM 在处理多模态组合推理方面具有巨大潜力，关键在于设计合适的训练目标和数据。

## 7.2. 局限性与未来工作
**局限性：**
1.  **继承 CLIP 的偏差：** 由于 MCL 依赖冻结的 CLIP 模型作为视觉编码器和检索基准，它不可避免地继承了 CLIP 模型本身的偏差和局限性（例如对某些视觉概念的识别错误）。
2.  **失败案例：** 论文展示了模型在处理极其复杂的关系或与图像内容不相关的文本查询时仍可能失败。
3.  **计算开销：** 使用多个堆叠的 `[RET]` 令牌以及 LLM 本身的高计算成本，使得推理过程相对较重。

**未来工作：**
作者希望 MCL 能启发未来关于将 LLM 与其他模态（如音频、视频）对齐的研究。此外，如何进一步优化数据生成质量以减少幻觉，以及如何降低计算成本，也是潜在的研究方向。

## 7.3. 个人启发与批判
**启发：**
1.  **数据自动化的力量：** 本文展示了如何巧妙地利用 LLM 本身来解决多模态训练数据稀缺的问题。这种“以模型治数据”的思路非常具有启发性。
2.  **训练目标的重要性：** 即使模型架构（LLM + Adapter）相似，不同的训练目标（简单的翻译 vs 复杂的组合）会导致模型能力的巨大差异。这提示我们在设计多模态模型时，应更关注任务的定义。
3.  **机制创新：** 堆叠检索机制是一个简单但有效的设计，它利用了对 LLM 内部机制（自回归注意力）的深刻理解，通过修改注意力掩码来改变模型的行为。

**批判与思考：**
1.  **数据质量把控：** 虽然利用 LLM 生成数据解决了数量问题，但生成数据的质量（尤其是 $T_{tgtc}$ 与真实图像的对应关系）是隐式的（通过 CLIP 特征监督）。如果 CLIP 本身对某些文本理解有误，这种错误可能会在训练中放大。未来可以考虑引入更精细的过滤机制。
2.  **泛化性验证：** 论文主要展示了检索和 VQA 任务。对于生成类任务（如根据文本条件直接生成图像），MCL 提升的视觉-语言对齐是否能带来显著收益，还有待验证。
3.  **基线对比的公平性：** 虽然 MCL 优于 FROMAGe，但部分原因可能也来自于使用了更大规模或更特定任务的数据（MMC）。如果能控制数据规模进行对比，更能凸显 MCL 目标函数本身的贡献。
