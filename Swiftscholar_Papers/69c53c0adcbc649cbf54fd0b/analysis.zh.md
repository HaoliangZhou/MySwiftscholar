# 1. 论文基本信息
## 1.1. 标题
论文标题为《Towards Effective Experiential Learning: Dual Guidance for Utilization and Internalization》（面向高效体验式学习：面向利用与内化的双引导优化），核心主题是通过模拟人类体验式学习的双引导机制，解决大语言模型在可验证奖励强化学习（RLVR）训练中的经验利用不足、内化不充分的问题，从而提升大模型的推理能力。
## 1.2. 作者与隶属机构
第一作者为Fei Bai（白飞）和Zhipeng Chen（陈志鹏），均来自中国人民大学高瓴人工智能学院；其他作者分别来自21Quest研究院、北京航空航天大学，通讯作者为中国人民大学赵鑫教授。
## 1.3. 发表状态与发表年份
本论文为2026年3月25日发布在arXiv的预印本，尚未正式发表在期刊或会议上。
## 1.4. 摘要
当前强化学习（RL）尤其是基于可验证奖励的强化学习（RLVR）已经成为提升大语言模型推理能力的重要范式，但现有RL训练方法和人类学习机制相差较大：人类会同时利用外部经验（如课本、他人经验）和内部已掌握的知识引导探索，还会将有用的推理轨迹内化成稳定的知识。
针对这一差距，论文提出了<strong>双引导优化（Dual Guidance Optimization, DGO）</strong> 统一框架，同时利用外部与内部经验提升训练效率：首先基于历史探索轨迹构建经验库，策略在经验库和模型内部知识的联合引导下进行探索，生成的轨迹再用来更新经验库和优化模型参数，形成经验利用与内化的闭环。实验显示DGO在多个基准上持续优于基线方法，证明更好的经验利用与内化可以有效提升推理能力。

# 2. 整体概括
## 2.1. 研究背景与动机
### 2.1.1. 核心问题
近年来，<strong>可验证奖励强化学习（Reinforcement Learning from Verifiable Rewards, RLVR）</strong> 逐渐成为大模型推理训练的主流范式：它不需要人工标注偏好，奖励完全可以客观验证（比如数学题答案正确给1、错误给0），训练成本低且适合推理类任务。但现有RLVR方法存在两个核心缺陷：
1. 仅依赖参数更新的方法（如GRPO、DAPO）把高质量轨迹只当成单次训练信号，用完即丢弃，没有作为可复用的经验存储，导致有用的推理模式没有被充分利用，还容易出现遗忘；
2. 依赖外部经验的方法（如经验检索推理）仅把经验作为推理时的外部提示，没有将经验内化成模型自身的参数能力，模型的独立推理能力没有得到提升。
   这一问题和人类的体验式学习机制形成了明显差距：人类学习会同时用外部经验拓展探索边界，用内部知识保证推理稳定性，还会将外部引导下学到的方法内化成自己的能力，最终脱离外部引导也能独立解题。
### 2.1.2. 研究重要性
推理能力是大语言模型的核心能力之一，现有RLVR方法的性能瓶颈很大程度上来自经验利用和内化的不足，解决这一问题可以在不增加模型参数、不增加训练数据规模的前提下，大幅提升大模型的推理效率和效果。
### 2.1.3. 创新思路
论文模拟人类的体验式学习机制，提出将非参数的外部经验库和参数化的模型更新结合成闭环：用外部经验库引导模型探索自身难以发现的高质量推理轨迹，再将这些轨迹处理后蒸馏到模型参数中实现内化，内化后的模型又能生成更高质量的轨迹反哺经验库，形成正向循环，同时提升经验利用效率和内化效果。
## 2.2. 核心贡献与主要发现
论文的核心贡献可以总结为三点：
1. 理论层面：证明了同时优化经验利用和内化可以有效提升大模型的推理能力，还能拓展模型可以稳定覆盖的推理行为范围；
2. 方法层面：提出了DGO双引导优化框架，首次将非参数的经验利用和参数化的经验内化整合到统一的迭代闭环中，为大模型的体验式学习提供了新的研究范式；
3. 实验层面：在Qwen3-4B/8B/14B三个尺度的模型上，DGO的内在推理（零样本）平均得分分别达到30.34%、32.41%、38.04%，测试时缩放（TTS）后进一步提升到36.51%、39.38%、44.78%，显著优于SFT、GRPO、DAPO等现有强基线，同时展现出良好的跨域泛化能力。

# 3. 预备知识与相关工作
## 3.1. 基础概念
理解本文需要掌握以下核心基础概念：
### 3.1.1. 可验证奖励强化学习（RLVR）
是专门针对推理任务的强化学习范式，**奖励信号完全可以客观验证、不需要人工标注偏好**，比如数学题的答案是否正确、代码是否可以运行、逻辑推导是否自洽等，相比依赖人类偏好的RLHF（基于人类反馈的强化学习），训练成本更低，更适合推理类任务的训练。
### 3.1.2. 参数学习与非参数学习
- <strong>参数学习（Parametric Learning）</strong>：知识存储在模型的参数中，通过更新模型参数来提升能力，比如监督微调（SFT）、强化学习（RL）都属于参数学习，优点是推理时不需要额外依赖，缺点是容易遗忘、记忆容量受参数规模限制；
- <strong>非参数学习（Non-Parametric Learning）</strong>：知识存储在外部的存储模块（如经验库、记忆库）中，模型推理时通过检索获取知识，一般不需要更新模型参数，优点是记忆容量大、知识可更新，缺点是推理时依赖外部检索，独立能力弱。
### 3.1.3. 近端策略优化（PPO）
是当前大模型RL训练最常用的基础算法，核心思想是通过**截断重要性采样系数**限制策略的更新步长，避免策略更新幅度过大导致训练崩溃，本文用到的GRPO（分组相对策略优化）就是PPO的改进版本，训练稳定性更高。
### 3.1.4. 体验式学习
原本是教育学中的概念，指学习者通过实践体验、反思总结、内化应用的循环来学习知识，而不是被动接收灌输，人类的大部分技能学习都属于体验式学习，本文将这一机制引入大模型的RL训练中。
## 3.2. 前人工作
现有和大模型体验式学习相关的工作可以分为两大类：
1. **非参数类方法**：主要聚焦于外部经验的构建和检索，比如构建分层经验仓库、设计经验的更新和验证机制，但是这类方法大多只关注经验的利用，没有将经验内化成模型自身的参数能力，模型的独立推理能力没有提升；
2. **参数类方法**：主要包括监督微调、强化学习、偏好优化等，直接更新模型参数来内化能力，但是这类方法把高质量轨迹只当成单次训练信号，不会作为可复用的经验存储，导致有用的推理模式没有被充分利用，训练效率低。
## 3.3. 技术演进脉络
大模型的推理训练范式经历了三次主要演进：
1. 第一阶段：纯监督微调（SFT），用标注的正确推理轨迹直接微调模型，优点是简单稳定，缺点是需要大量高质量标注数据，模型的探索能力弱；
2. 第二阶段：RLVR训练，用可验证的奖励信号让模型自主探索推理轨迹，不需要大量标注，探索能力强，缺点是训练不稳定，经验利用效率低；
3. 第三阶段：体验式学习训练，结合外部经验引导和参数内化，在RLVR的基础上进一步提升训练效率和效果，本文的DGO就是这一阶段的代表性工作。

# 4. 方法论
DGO的核心思想是模拟人类体验式学习的闭环：**经验构建→联合轨迹-策略优化→经验内化**，三个阶段反复迭代，同时提升经验利用效率和内化效果。框架的整体结构如下图（原文Figure 2）所示：

## 4.1. 方法原理
DGO的核心直觉非常符合人类的学习规律：
1. 外部经验可以引导模型探索到仅靠自身现有能力难以发现的高质量推理轨迹，拓展探索边界；
2. 内部已掌握的知识可以保证推理的稳定性，避免被无效经验误导；
3. 外部引导下找到的高质量轨迹经过处理后，可以内化成模型自身的参数能力，之后即使没有外部引导也能独立完成同类任务。
   形式化定义如下：
给定输入问题$x$，外部经验库$\mathcal{E}$，参数为$\theta$的模型策略$\pi_\theta$，模型在内外双引导下生成推理轨迹$s$：
$$
s \sim \pi _ { \theta } ( \cdot \mid x , { \mathcal { E } } ) .
$$
其中$\pi_\theta(\cdot|x,\mathcal{E})$表示给定问题$x$和经验库$\mathcal{E}$时策略生成轨迹的概率分布。
生成的轨迹经过转换（过滤、重写等）后变成监督信号，用来更新模型参数：
$$
\theta ^ { \prime } \gets \mathrm { U p d a t e } ( \theta ; \mathcal { T } ( s ) ) ,
$$
其中$\mathcal{T}(\cdot)$是轨迹转换函数，用来将原始轨迹转换成适合训练的监督信号，$\theta'$是更新后的模型参数。
通过这个过程，外部经验提升轨迹探索的质量，高质量轨迹又反过来优化经验库和模型的内部能力，形成正向反馈闭环。
## 4.2. 迭代双引导训练流程
DGO的训练是迭代式的，每一轮迭代包含三个核心阶段：经验构建、联合轨迹-策略优化、经验内化，下面分阶段详细解释。
### 4.2.1. 经验构建
经验构建阶段的目标是从历史推理轨迹中提炼出可迁移的、通用的推理经验，而不是直接存储完整的解题轨迹（避免模型背诵答案）。
每个经验被表示为三元组$\boldsymbol{e}=(c,\kappa,\tau)$：
- $c$：经验适用的领域或问题阶段，比如“代数因式分解”“排列组合计数”；
- $\kappa$：经验对应的具体场景、条件或潜在错误，比如“遇到乘积形式的单位根问题”“容易忽略对数的定义域限制”；
- $\tau$：对应的操作建议或避坑提示，比如“用多项式因式分解结合单位根乘积公式求解”“先验证对数的底数和真数满足定义域再计算”。
  经验通过专门训练的经验生成器$G$从轨迹中提取：`e=G(s)`，其中$s$是输入的推理轨迹。从正确轨迹中提取的经验是有效的解题策略，从错误轨迹中提取的经验是避坑提示。为了避免经验变成答案提示，经验库会严格过滤掉所有包含或暗示最终答案的内容。
### 4.2.2. 联合轨迹-策略优化
本阶段的目标是用RL强化模型利用经验引导推理的能力，同时生成更高质量的轨迹，为后续的经验内化提供数据。
#### （1）带经验退火的探索
为了让模型在训练初期多依赖外部经验探索高质量轨迹，后期逐渐减少对外部经验的依赖，真正掌握底层推理逻辑，训练采用**经验退火机制**：
在第$k$次迭代时，训练数据从经验引导分布$\mathcal{D}_e^{(k)}$（带外部经验提示的问题）和无经验引导分布$\mathcal{D}_i^{(k)}$（不带外部经验提示的问题）中混合采样，混合系数为$\alpha_k$：
$$
\mathcal{D} ^ { ( k ) } = \alpha _ { k } \mathcal { D } _ { e } ^ { ( k ) } + ( 1 - \alpha _ { k } ) \mathcal { D } _ { i } ^ { ( k ) } .
$$
$\alpha_k$随着训练迭代逐渐减小，本文中$\alpha_k$从1.0（第一次迭代全用经验引导）降到0.5，再降到0.25，和人类学习中“先跟着例题学，再慢慢自己独立做题”的过程一致。
训练实例表示为$\boldsymbol{z}=(x,\tilde{e})$，其中$\tilde{e}$是检索到的对应问题的经验，无经验引导的实例$\tilde{e}$为空。模型采用PPO类损失进行优化，损失函数原文如下：
$$
\begin{array} { l } { \displaystyle \mathcal { L } ( \theta ) = \mathbb { E } _ { z \sim D ^ { ( k ) } , s \sim \pi _ { \theta _ { \mathrm { o l d } } } ( \cdot | z ) } \left[ \frac { 1 } { | s | } \sum _ { t = 1 } ^ { | s | } \operatorname* { m i n } \left( \rho _ { t } ( \theta ) A _ { t } , \right. \right. } \\ { \displaystyle \left. \left. \mathrm { c l i p } ( \rho _ { t } ( \theta ) , 1 - \epsilon , 1 + \epsilon ) A _ { t } \right) \right] , } \end{array}
$$
其中重要性采样系数$\rho_t(\theta)$的定义为：
$$
\rho _ { t } ( \theta ) = \frac { \pi _ { \theta } \left( s _ { t } | z , s _ { < t } \right) } { \pi _ { \theta _ { \mathrm { o l d } } } \left( s _ { t } | z , s _ { < t } \right) }
$$
符号解释：
- $\mathbb{E}$：期望运算，对采样的训练实例和轨迹求平均；
- $z$：从混合分布$\mathcal{D}^{(k)}$中采样的训练实例；
- $s$：从旧策略$\pi_{\theta_{\text{old}}}$生成的推理轨迹；
- $|s|$：轨迹的词元长度；
- $t$：轨迹中的第$t$个词元，$s_{<t}$表示第$t$个词元之前的上下文；
- $A_t$：第$t$步的优势函数，衡量当前动作比平均动作的优劣程度，正的表示当前动作比平均好，负的表示差；
- $\text{clip}$：截断函数，将$\rho_t(\theta)$限制在$[1-\epsilon,1+\epsilon]$范围内，防止策略更新步长过大导致训练不稳定，$\epsilon$一般取0.2；
- $\rho_t(\theta)$：重要性采样系数，用来修正新旧策略的分布差异，让旧策略采样的轨迹可以用来更新新策略。
  这个损失函数的核心是：当优势函数为正时，鼓励模型增加当前动作的概率；当优势函数为负时，降低当前动作的概率，同时通过截断避免更新幅度过大。
#### （2）经验更新优化
每次RL迭代完成后，收集模型生成的所有推演轨迹$S^{(k)}=\{s_j^{(k)}\}$，用经验生成器$G$从中提取新的经验$\mathcal{E}_{\text{new}}^{(k)}=G(S^{(k)})$，然后更新经验库：
$$
\mathscr { E } ^ { ( k + 1 ) } = \mathrm { S e l e c t } \Big ( \mathscr { E } ^ { ( k ) } \cup \mathscr { E } _ { \mathrm { n e w } } ^ { ( k ) } \Big ) ,
$$
其中$\text{Select}(\cdot)$是经验选择函数，通过冗余检测和质量评估，只保留最有信息量的经验，删掉冗余、低质量的内容。
这个过程形成正向循环：模型能力越强，生成的轨迹质量越高，提取的经验越有用，反过来又能引导模型生成更好的轨迹。
### 4.2.3. 经验内化
本阶段的目标是将外部经验引导下获得的推理能力，迁移到模型的参数中，让模型即使没有外部经验提示也能独立解题，实现真正的能力内化。
#### （1）轨迹重写
经验引导下生成的轨迹通常会包含“根据经验XXX”这类明确引用外部经验的内容，如果直接用来训练，会让模型依赖这类外部提示，无法真正内化能力。因此需要对轨迹进行重写，去掉所有对外部经验的引用，保留核心推理逻辑，变成自包含的推理过程。
轨迹重写的形式化定义为：给定问题$x$，对应的经验$\mathcal{E}_x$，原始的经验引导轨迹$s_e^{(k)}$，重写提示$p$，用训练好的RL策略$\pi_{\theta_{\text{RL}}^{(k)}}$生成重写后的轨迹：
$$
\tilde { s } _ { e } ^ { ( k ) } \sim \pi _ { \theta _ { \mathrm { R L } } ^ { ( k ) } } \Big ( \cdot \mid x , s _ { e } ^ { ( k ) } , \mathcal { E } _ { x } , p \Big ) .
$$
重写后的轨迹会经过严格过滤：去掉答案错误的、仍然提到外部经验的、长度超过6K的、包含无意义内容的噪声轨迹，然后每个问题保留质量最高的$K$条，形成隐式轨迹集$S_{\text{imp}}^{(k)}$：
$$
S _ { \mathrm { i m p } } ^ { ( k ) } = \mathrm { T o p - K } \Big ( \mathcal { F } \big ( \tilde { S } _ { e } ^ { ( k ) } \big ) \Big ) ,
$$
其中$\mathcal{F}(\cdot)$是过滤函数，$\text{Top-K}$是按质量得分选最高的$K$条轨迹。
#### （2）策略蒸馏
轨迹重写完成后，通过监督蒸馏将轨迹中的推理模式内化到模型参数中。
第$k$次迭代时，积累之前所有迭代的隐式轨迹$S_{\text{imp}}^{(\leq k)}=\cup_{i=1}^k S_{\text{imp}}^{(i)}$，和所有的经验引导轨迹$S_e^{(\leq k)}=\cup_{i=1}^k S_e^{(i)}$，按比例混合形成蒸馏数据集，混合系数为$\beta_k$：
$$
\mathcal { D } _ { \mathrm { d i s t } } ^ { ( k ) } = \beta _ { k } S _ { \mathrm { i m p } } ^ { ( \leq k ) } + \left( 1 - \beta _ { k } \right) S _ { e } ^ { ( \leq k ) } ,
$$
本文中$\beta_k$固定为0.5，平衡内化后的独立能力和利用外部经验的能力。
蒸馏阶段用最大似然估计优化模型参数，注意参数从初始模型$\theta^{(0)}$初始化，避免过拟合：
$$
\boldsymbol { \theta } ^ { ( k + 1 ) } = \arg \operatorname* { m a x } _ { \boldsymbol { \theta } } \mathbb { E } _ { ( \boldsymbol { x } , \boldsymbol { s } ) \sim \mathcal { D } _ { \mathrm { d i s t } } ^ { ( k ) } } \log \pi _ { \boldsymbol { \theta } } ( \boldsymbol { s } \mid \boldsymbol { x } ) .
$$
蒸馏得到的模型作为下一次迭代的初始模型，完成整个闭环。
## 4.3. 推理范式
DGO支持两种推理模式，适配不同的场景：
### 4.3.1. 内在推理（Intrinsic Inference）
即零样本推理，完全不需要外部经验，只靠模型自身的参数知识生成轨迹，用来衡量模型的独立推理能力：
$$
s \sim \pi _ { \theta } ( \cdot \mid x ) .
$$
### 4.3.2. 迭代经验引导测试时缩放（Test-Time Scaling, TTS）
推理时也引入经验引导，进一步提升性能，适合对推理准确率要求高、可以接受一定计算量的场景：
模型进行$K$轮轨迹生成，每轮生成$N$条轨迹，从中提取在线经验，第一轮用离线经验库冷启动，后面的轮次用前一轮生成的在线经验引导：
$$
s ^ { ( k ) } \sim \left\{ \begin{array} { l l } { \pi _ { \theta } ( \cdot \mid x , \mathcal { E } _ { \mathrm { o f f l i n e } } ) , \quad k = 1 , } \\ { \pi _ { \theta } ( \cdot \mid x , \mathcal { E } _ { \mathrm { o n l i n e } } ^ { ( k - 1 ) } ) , \quad k > 1 . } \end{array} \right.
$$
其中$\mathcal{E}_{\text{offline}}$是训练得到的离线经验库，$\mathcal{E}_{\text{online}}^{(k-1)}$是第`k-1`轮生成轨迹中提取的在线经验。

# 5. 实验设置
## 5.1. 数据集
### 5.1.1. 训练集
训练数据从SkyWork开源数学题集中采样9600道题，覆盖中小学到竞赛级别的数学问题。
### 5.1.2. 测试集
测试集分为域内和域外两类，覆盖不同难度和领域：
- <strong>域内基准（数学推理）</strong>：
  1. AIME24、AIME25：美国数学邀请赛2024、2025年真题，难度很高，是衡量数学推理能力的黄金基准；
  2. HMMT25：2025年哈佛-麻省理工数学锦标赛真题，难度高于AIME；
  3. MATH500：常用的数学推理基准，包含500道不同类型的高中数学题。
- <strong>域外基准（跨域推理）</strong>：
  1. GPQA-Diamond：研究生级别的科学多选题，覆盖物理、化学、生物等领域，几乎无法通过搜索引擎得到答案，用来测试跨域知识推理能力；
  2. HLE-Verified：高难度多领域推理基准，涵盖人文、社科、自然科学等领域，用来测试通用推理能力的迁移性。
     选择这些数据集的原因是覆盖了从数学推理到跨域通用推理的不同场景，可以全面验证DGO的有效性和泛化能力。
## 5.2. 评估指标
论文用到两个核心评估指标：
### 5.2.1. avg@16
- **概念定义**：对于每个问题，模型采样生成16条推理轨迹，统计正确轨迹的比例，再对所有问题求平均，反映模型的平均推理准确率，采样16次可以消除随机性，更稳定地反映模型的真实能力。
- **数学公式**：
  $$
  \text{avg@16} = \frac{1}{|\mathcal{D}|} \sum_{x \in \mathcal{D}} \frac{\text{count of correct trajectories for } x \text{ in 16 samples}}{16}
  $$
- **符号解释**：$|\mathcal{D}|$是测试集的总问题数，$x$是单个测试问题。
### 5.2.2. Pass@K
- **概念定义**：对于每个问题，模型采样$K$条推理轨迹，只要至少有一条轨迹得到正确答案，就算这个问题做对，统计所有问题中做对的比例，反映模型在多次尝试下的解题能力，$K$越大，模型做对的概率越高。
- **数学公式**：
  $$
  \text{Pass@K} = \frac{1}{|\mathcal{D}|} \sum_{x \in \mathcal{D}} \mathbb{I}\left( \exists s \in S_K(x), s \text{ is correct} \right)
  $$
- **符号解释**：$S_K(x)$是模型对问题$x$生成的$K$条轨迹，$\mathbb{I}(\cdot)$是指示函数，条件满足时取1，否则取0。
## 5.3. 对比基线
论文选择了四类有代表性的基线方法：
1. <strong>EGI（经验引导推理）</strong>：训练时不更新模型参数，只在推理时给模型外部经验提示，属于训练免费的基线，用来衡量纯外部经验引导的效果；
2. <strong>SFT（监督微调）</strong>：用正确的推理轨迹做监督微调，是最常用的基础推理训练方法；
3. <strong>GRPO（分组相对策略优化）</strong>：当前最先进的大模型RL训练算法之一，仅用参数更新，不用外部经验；
4. <strong>DAPO（分布式异步策略优化）</strong>：另一个先进的大模型RL训练基线，同样仅用参数更新，不用外部经验。
   这些基线覆盖了从训练免费方法到先进RL方法的所有主流方案，对比结果具有说服力。

# 7. 总结与思考
## 7.1. 结论总结
本文提出的DGO双引导优化框架，首次将非参数的外部经验利用和参数化的经验内化整合到统一的迭代闭环中，完美模拟了人类的体验式学习机制：
1. 通过外部经验库引导模型探索高质量推理轨迹，拓展探索边界；
2. 通过轨迹重写和策略蒸馏将外部经验内化成模型自身的参数能力，提升独立推理能力；
3. 形成正向循环，迭代提升经验质量和模型能力。
   实验证明DGO在多个数学和跨域推理基准上都显著优于现有强基线，同时提升了模型的内在推理能力、经验利用效率和跨域泛化能力，为大模型的体验式学习提供了新的研究范式。
## 7.2. 局限性与未来工作
作者在论文中指出的局限性和未来研究方向包括：
1. 当前的经验管理（冗余检测、质量评估）还是基于人工设计的规则，未来可以让模型更自主地管理和更新自身的经验库；
2. 进一步优化外部经验和内部知识的协调机制，提升体验式学习的效率、鲁棒性和可持续性；
3. 将DGO扩展到更多任务场景，比如代码生成、工具调用、具身智能等。
## 7.3. 个人启发与批判
### 7.3.1. 启发
DGO的框架非常符合人类的学习规律，这种“外部引导探索→内化沉淀能力”的思路有很高的通用性：
1. 可以迁移到小模型场景：通过外部经验库弥补小模型参数不足的缺陷，用DGO训练的小模型有望达到更大参数模型的推理能力；
2. 可以扩展到多模态、具身智能等场景：经验库不仅可以存储文本推理经验，还可以存储多模态感知经验、具身操作经验，提升多模态模型和智能体的学习效率；
3. 可以支持持续学习：经验库可以持续更新，模型定期蒸馏新的经验，避免灾难性遗忘，实现持续的能力提升。
### 7.3.2. 潜在改进点
1. 当前的经验生成器需要单独训练，需要额外的标注数据，未来可以将经验生成也整合到DGO的闭环中，不需要单独训练，进一步降低训练成本；
2. 当前的经验是固定结构的三元组，形式不够灵活，可以探索更灵活的经验表示方式，比如自然语言形式的经验、结构化知识图谱形式的经验；
3. 蒸馏阶段用初始模型初始化来防止过拟合，虽然避免了遗忘，但也会丢失之前迭代学到的部分能力，未来可以探索更优的蒸馏初始化方式，兼顾防止过拟合和保留已有能力。
