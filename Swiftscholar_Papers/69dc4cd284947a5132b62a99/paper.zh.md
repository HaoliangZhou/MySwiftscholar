# 生成用于无训练零-shot组合图像检索的虚构世界

王彤 $^{1}$，赵云汉 $^{2}$，孔树 $^{1, 3}$ 1 澳门大学，2 加州大学欧文分校，3 澳门大学协同创新研究所 网址和代码: https://github.com/leowangtong/Paracosm/

摘要。组合图像检索（CIR）是通过多模态查询从数据库中检索目标图像的任务，该查询由参考图像和修改文本组成。文本指定如何改变参考图像以形成“心理图像”，CIR应基于此“心理图像”在数据库中找到目标图像。CIR的基本挑战在于，这个“心理图像”在物理上并不可用，仅通过查询隐式定义。当前文献追求零样本方法，并使用大型多模态模型（LMM）为给定的多模态查询生成文本描述，然后利用视觉语言模型（VLM）进行文本-视觉匹配以搜索目标图像。相较之下，我们从第一原则出发，直接生成“心理图像”以实现更准确的匹配。特别地，我们提示LMM为给定的多模态查询生成“心理图像”，并提出使用该“心理图像”来搜索目标图像。由于“心理图像”与真实图像之间存在合成到真实的域间差异，我们还为数据库中的每个真实图像生成一个合成对应物，以促进匹配。从这个意义上讲，我们的方法使用LMM构建一个“幻想世界”，在其中匹配多模态查询和数据库图像。因此，我们称这种方法为幻想世界。值得注意的是，幻想世界是一种无训练的零样本CIR方法。它在具有挑战性的基准测试中显著优于现有的零样本方法，实现了零样本CIR的最先进性能。 关键词：无训练零样本组合图像检索 · 基础模型 · 合成到真实域间差异

# 1 引言

组合图像检索 (CIR) 在互联网搜索 [10, 22] 和电子商务 [42, 59] 中提出了新的场景，其中根据用户提供的多模态查询从数据库中检索目标图像，该查询由参考图像和修改文本组成 [56]。修改文本描述了如何改变参考图像，CIR 算法应根据该描述从数据库中找到最佳匹配图像 [1, 32, 59]。

![](images/1.jpg)  
Fig. 1: Overview of our method and benchmarking results. Unlike existing training-free methods [21, 49, 62] generating descriptions for multimodal queries, which use an LMM to generate descriptions for multimodal queries, we use it to generate "mental images" for multimodal queries and synthetic counterparts of database images. Matching them effectively mitigates synthetic-to-real domain gaps and boosts CIR performance. Our final training-free zero-shot method Paracosm (Fig. 2) significantly outperforms existing zero-shot CIR methods, as summarized in the radar chart on standard benchmarks. Detailed results are provided in Section E.

现状。目前，通过监督学习 $[ 5 , 9 , 1 3 , 2 4 , 3 3 ]$ 针对标注三元组 <参考图像, 修改文本, 目标图像 $>$ 解决了 CIR 问题。由于策划大规模三元组数据的成本极高，近年来的研究开发了零-shot CIR (ZS-CIR) 方法 [4, 39, 49, 62]。值得注意的是，ZS-CIR 并不一定意味着不学习，而是强调不直接在数据三元组上训练 CIR 模型。相反，它们仍然以间接的方式训练模型 [16, 47, 48, 57, 64]。例如，许多方法 [4, 39] 在图像-文本对上训练文本逆向网络，以将图像映射到文本词元进行跨模态匹配；一些方法 [29, 57] 使用预训练的图像生成模型 [37, 40, 68] 合成伪图像，以辅助跨模态匹配。为了对比基于训练的 ZS-CIR 方法，非训练方法 [21, 49, 62] 提出利用大型多模态模型 (LMMs) [8, 35, 55] 为多模态查询生成描述，并用于文本到图像 (T2I) 匹配以实现 CIR。

洞察。CIR的基本挑战在于多模态查询仅隐含地定义了一个“心理图像”，该图像并不存在于物理世界中，因此无法用于检索相应的目标图像。我们希望从基本原理入手，通过为给定查询生成“心理图像”，来提高检索的准确性（见图1）。虽然现有研究利用大语言模型（LMMs）生成文本描述，但我们利用LMMs基于多模态查询生成图像[58]。然而，由于生成的“心理图像”与数据集中真实图像之间存在合成与现实的领域差距，我们提出为每个数据集图像生成一个合成对应物，以促进匹配。由于我们的方法本质上使用LMMs来定义一个合成或虚拟空间，如一个“虚构世界”，我们将其命名为Paracosm。值得注意的是，Paracosm是一种无训练的零样本CIR方法。为了验证Paracosm，我们遵循已建立的实验协议，使用预训练的视觉语言模型（VLMs）[19, 36]进行匹配。在具有挑战性的基准上进行的广泛实验表明，Paracosm显著优于现有的零样本CIR方法（参见图1中的总结）。

![](images/2.jpg)  
Fig. 2: Flowchart of our training-free zero-shot CIR method Paracosm. Given a multimodal query that consists of a reference image and a modification text, we feed it to an LMM to generate a "mental image". We further generate a brief description for it. 贡献。我们做出了三项重要贡献：我们通过无训练方法 Paracosm 从第一性原理解决了 CIR。它为多模态查询生成“心理图像”，以促进与数据库图像的匹配。- 我们通过生成数据库图像的合成对应物来缓解心理图像的合成到真实领域差距。将它们匹配在一起提升了 CIR 的性能。在标准基准测试中，我们证明 Paracosm 的表现显著优于现有的 ZS-CIR 方法，达到了最先进的性能。

# 2 相关工作

组合图像检索（CIR）通过允许用户在检索中使用多模态查询，扩展了传统的检索任务，如文本到图像检索和图像到图像检索 [56]。CIR最初采用监督学习方法 [5, 9, 13, 24, 33]，即在标注的数据三元组<参考图像，修改文本，目标图像$>$上训练模型。这些方法 [5, 18, 53, 60, 65, 66] 提出了在标注的数据三元组上训练一个Transformer或多层感知机（MLP），旨在将参考图像和修改文本融合在特征空间中，以便与数据库图像进行匹配。一些方法 [25, 33, 34] 微调了一个预训练的视觉-语言模型（VLM） [26, 36]，以更好地匹配多模态查询和数据库图像。然而，由于监督学习方法需要昂贵的三元组数据策划，最近的方法探索了零样本组合图像检索（ZS-CIR） [4, 16, 21, 29, 39, 49, 57, 62]。我们探索了无训练的ZS-CIR，并介绍了一种相对简单的方法，该方法与一些最近的监督方法具有竞争力。

![](images/3.jpg)  
Fig. 3: Comparison of qualitative results between OSrCIR [49] and our Paracosm. We show four examples from the CIRCO dataset [1] in the first column, followed by generated descriptions and top-4 retrievals by OSrCIR, and the mental images and top-4 retrievals by Paracosm. For each multimodal query, OSrCIR uses an LMM to generate a description, uses it to match database images, and returns top-ranked ones. Instead, Paracosm uses an LMM to generate a "mental image" for each query, which contains much richer information than a description, allowing image-to-image matching for better retrieval. Consequently, Paracosm yields better retrievals than OSrCIR.

零-shot复合图像检索（ZS-CIR）旨在解决无需直接在标注三元组数据上进行训练的CIR问题。它并不是指开发无训练方法，而是允许以间接方式进行训练。例如，许多方法在现有图像数据集上训练文本反演网络，将参考图像映射到伪词元。这个词元与修改文本结合后，用于与数据库图像进行跨模态匹配。 最近的一些方法使用预训练生成模型合成合成图像（类似于我们的“心理图像”），称为伪目标图像，用于给定的多模态查询。他们使用这些图像增强文本特征（即由文本反演网络输出），以计算与数据库图像的相似度分数。值得注意的是，出现了一些无训练的ZS-CIR方法，它们利用LMM为给定的多模态查询生成描述。这避免了训练单独的文本反演网络。值得一提的是，现有的ZS-CIR方法并没有考虑为数据库图像生成描述或合成图像。相反，我们的工作做到了这一点，旨在从第一原则出发解决ZS-CIR问题。具体而言，我们建议为查询生成“心理图像”和真实数据库图像的合成对应物，以减轻合成与真实领域之间的差距，从而提高性能。

基础模型（FMs）在各种格式的网络规模数据上进行了预训练，涵盖了多种模态，并在零样本任务中展现出前所未有的性能。因此，FMs 在 CIR 文献中得到广泛应用。首先，视觉-语言模型（VLMs）在网络规模的图像-文本对上进行预训练，常用于 CIR 中从修改文本和图像中提取特征，实现跨模态匹配。其次，大语言模型（LLMs）在大规模文本语料库上进行预训练，用于 CIR 中生成目标图像描述，结合参考图像的标题和修改文本。第三，大多模态模型（LMMs）在网络规模的多模态数据上进行预训练，通常在参数规模上大于 VLMs，用于 CIR 中为多模态查询生成图像标题。早期的 CIR 研究利用 LMMs 创建三元组数据以进行监督训练。相比之下，最近的大多数无训练 ZS-CIR 方法使用 LMMs 为多模态查询生成文本描述，并利用 VLM 将这些描述与数据库图像匹配，将 CIR 任务转变为文本到图像的检索问题。在此基础上，我们广泛利用 LMMs，不仅为多模态查询生成“心象”，还创建真实数据库图像的合成对应物。有效匹配这些合成图像显著缩小了合成与现实之间的领域差距，从而显著提升了 CIR 性能。

# 3 方法论

我们首先定义 ZS-CIR 问题及其开发协议。接着介绍我们无需训练的方法 Paracosm。尽管其概念简单，但 Paracosm 在零-shot 方法中取得了最先进的性能。图 2 展示了完整的流程。

# 3.1 零-shot CIR 的基础知识

任务定义。设 $\left( \mathbf { I } _ { r e f } , \mathbf { t } _ { m o d } \right)$ 表示给定多模态查询的参考图像和修改文本。 $\mathbf { t } _ { m o d }$ 描述如何修改 $\mathbf { I } _ { r e f }$ 以匹配用户想要搜索的内容，即从包含 $n$ 张图像 $\{ \mathbf { I } ^ { 1 } , \mathbf { I } ^ { 2 } , . . . , \mathbf { I } ^ { n } \}$ 的给定数据库中获取的目标图像 $\mathbf { I } _ { t a r g e t }$。 ZS-CIR 的目标是开发方法，以便在不直接训练标注三元组 $( \mathbf { I } _ { r e f } , \mathbf { t } _ { m o d } , \mathbf { I } _ { t a r g e t } )$ 的情况下，检索多模态查询的目标图像。

方法论开发。尽管 ZS-CIR 避免使用带注释的三元组数据训练集，但仍然允许利用开放世界中可用的数据和其中的预训练特征模型。遵循这一协议，现有的 ZS-CIR 方法以不同方式利用大语言模型，例如，使用视觉-语言模型进行跨模态匹配 [4, 16, 39]，以及利用大语言模型生成描述 [21, 49, 62]。通过利用开放世界数据 [38, 41]，一些方法训练必要的模型，以便将 ${ \bf { I } } _ { r e f }$ 和 $\mathbf { t } _ { m o d }$ 融合成特征 [4, 16, 39]，或生成用于多模态查询的伪图像 [29, 57]。在本研究中，我们希望通过广泛利用大语言模型来开发一种无训练的方法，而无需训练任何新模型。接下来，我们将详细介绍我们的方法 Paracosm。

# 3.2 提出的方法：虚构世界

Paracosm 是一种相对简单的方法，通过使用大语言模型（LMMs）处理多模态查询和数据库图像。以下，我们将描述它如何处理这些内容并进行匹配以便检索。

处理多模态查询。对于由参考图像和修改文本组成的多模态查询，Paracosm 首先使用 LMM 生成“心理图像”${ \mathbf { I } } _ { m e n t a l }$。基于该心理图像 $\mathbf { I } _ { m e n t a l }$，我们进一步生成简短的文本描述 $\mathbf { t } _ { q u e r y }$，可以将其视为目标图像 $\mathbf { I } _ { t a r g e t }$ 的描述。在此步骤中，我们使用一个提示来指示 LMM 只专注于视觉内容，同时尽量减少美学细节，生成单句描述。图 4 说明了用于处理多模态查询的提示模板。由于不同数据集提供的修改文本格式各异，我们相应地调整提示模板结构，以确保 LMM 的输入格式正确。给定一个多模态查询，Paracosm 使用生成的心理图像和简短描述进行检索。这不同于一些近期的方法 [21, 49, 62]，这些方法仅使用生成的描述进行检索。例如，这些方法首先为参考图像生成描述，然后使用 LLM 根据修改文本修订描述。图 3 演示了单靠生成的描述会遗漏关键的视觉信息，而我们的心理图像保留了丰富的信息，有助于 CIR。预处理数据库图像。由于生成的心理图像是合成的，并且存在从合成到真实的领域差距，直接将其与数据库中的真实图像进行匹配并不理想。为了减轻这一差距，我们为数据库图像创建合成副本，并利用这些副本辅助匹配 [15,17,67]。具体而言，Paracosm 首先利用 LMM 为每个数据库图像生成详细描述。我们使用一个描述提示，强调捕捉所有可见物体、属性、空间关系和细致的视觉元素，以确保最大保真度。图 5 说明了这个提示模板。详细描述接着作为每个数据库图像 ${ \bf \cal I } ^ { i }$ 的文本-$\mathbf { I } _ { s y n } ^ { i }$ 的提示。真实数据库图像和合成副本共同用于多模态查询的匹配。检索的匹配。在将多模态查询和数据库图像转换为虚拟的梦境后，我们使用一个经过预训练的 VLM 构建特征，包括视觉编码器 $V ( \cdot )$ 和文本编码器 $T ( \cdot )$。具体而言，查询特征 $\mathbf { q }$ 和第 $i ^ { t h }$ 数据库图像特征 $\phi ^ { i }$ 的计算方式如下：

$$
\begin{array} { l } { \mathbf { q } = \lambda ( V ( \mathbf { I } _ { m e n t a l } ) + T ( \mathbf { t } _ { q u e r y } ) ) + ( 1 - \lambda ) T ( \mathbf { t } _ { m o d } ) } \\ { \phi ^ { i } = V ( \mathbf { I } ^ { i } ) + V ( \mathbf { I } _ { s y n } ^ { i } ) } \end{array}
$$

# 3.3 备注

Paracosm 更好地利用了 LMM。目前的 ZS-CIR 方法 [21, 49, 62] 也利用了 LMM，但主要集中在为多模态查询生成描述，并将 CIR 问题转变为文本到图像检索问题。然而，仅有文本描述无法充分捕捉到对于 CIR 至关重要的丰富视觉信息，因此常常导致检索图像不准确，如图 3 所示。一些近期的研究 [29,57] 意识到结合伪目标图像以促进 CIR 的重要性，例如，通过利用预训练的文本到图像（T2I）生成模型 [68] 根据描述合成伪目标图像。与此不同，我们利用具有图像编辑能力的 LMM，即根据修改文本编辑参考图像为“心理图像”。如表 2 所示，编辑参考图像相比于 T2I 生成伪目标图像能够产生更好的 CIR 结果。此外，Paracosm 利用 LMM 为每个数据库图像生成详细的描述，并使用这些描述作为提示生成合成对照。

![](images/5.jpg)  
Fig. 5: Ilustration of processing database images. For a database image, we first prompt an LMM to generate a comprehensive description about its visual content, capturing all visible objects, attributes, and visual elements. Using this description, we prompt a text-to-image generation model to produce a synthetic counterpart for this database image. For a database image, we use both itself (i.e., the real photo) and its synthetic counterpart as representation for retrieval.

考虑到高质量编辑图像，CIR 是否仍然必要？这个问题在基于多模态查询生成的“心理图像”（图 3）高质量面前浮现。我们认为，CIR 在现实世界的视觉搜索应用中仍然重要。例如，在电子商务和时尚行业，用户可能希望从一张衣物照片开始，通过指定如何改变照片在电子商店中搜索不同类型或风格的商品。也就是说，无论心理图像多少逼真，图像生成都无法替代库存中的真实产品。

计算成本。由于Paracosm广泛利用大语言模型（LMM），其计算成本较高，尤其是在生成图像时，即为查询生成的心理图像和数据库图像的合成对应物。然而，这并不是Paracosm特有的缺陷，现有研究同样依赖生成模型进行检索。例如，流行的零样本类别识别（ZS-CIR）方法使用LMM生成描述 [21, 29, 49, 62]，还有一些方法转向生成模型以合成图像 [29, 57]。重要的是，像这些方法一样，Paracosm可以提前处理数据库图像，而不需要在推理时动态计算它们的特征。因此，主要的计算开销在于推理过程中给定多模态查询的处理，包括生成相应的心理图像及其描述。由于生成模型中的高效推理是一个重要课题，并已经通过模型优化 [20, 63] 和优化实现 [14, 23] 取得了很大进展，Paracosm在长期内不会受到计算成本的困扰。

比较方法。本研究将我们的Paracosm与代表性和最新的零样本交互式检索（ZS-CIR）方法进行比较，这些方法可以分为基于训练的方法和无训练的方法。我们还比较了基准方法。基准方法包括“仅图像”方法，该方法使用参考图像的特征与数据库图像进行匹配；“仅文本”方法，该方法使用修改文本进行跨模态匹配；以及“图像+文本”方法，该方法将图像和文本特征相加以进行匹配。基于训练的方法包括Pic2Word [39]、SEARLE [4]、LinCIR [16]、CIG [57]和IP-CIR [29]。这些方法都依赖于文本特征，例如，训练一个文本反演网络，将参考图像映射为伪词元，并将其与修改文本的文本词元合并。值得注意的是，IP-CIR [29]和CIG [57]使用预训练的生成模型生成多模态查询的伪目标图像。然而，它们与其他方法一起运行以确保良好的CIR性能。- 无训练方法。CIReVL [21]使用LMM生成给定参考图像的描述，然后使用LLM根据修改文本修改该描述。它使用修改后的描述在图像数据库中进行跨模态检索。LDRE [62]提出生成多个多样化描述，以提高CIR性能。AutoCIR [12]采用自动化多智能体协作，通过自我纠正迭代优化检索查询。CoTMR [44]利用LMM生成全局标题和细粒度的对象级描述，并结合多粒度评分，这显著提高了检索性能。OSrCIR [49]精心设计了LMM的提示，使用反思链式思维来提高多模态查询的输出描述质量。

# 5 影响与局限性

社会影响。CIR 是许多现实应用（如电子商务和时尚行业）中搜索服务的重要组成部分。所提出的 Paracosm 通过广泛利用 LMM，极大地推进了 CIR 领域，这在其在标准基准数据集上的最先进性能中得到了体现。值得注意的是，LMM 的大规模预训练数据集可能包含有毒、冒犯或偏见的数据。这可能导致 LMM 生成不准确、有偏见甚至有害的文本描述和合成图像。此外，Paracosm 并没有针对不当多模态查询的警报机制，这些查询可能包含恶意修改的文本或旨在检索不当目标图像。如果被迫根据此类查询检索目标图像，则中间的心理映像和检索输出可能导致负面的社会影响。

局限性与未来工作。Paracosm 依赖于 LMM 生成的用于查询和数据库图像的合成对应物的心理图像质量，以及它们的文本描述。这意味着 Paracosm 的性能本质上取决于基础 LMM 的生成能力。这些生成的图像在初看时可能视觉上似乎合理，但往往缺乏事实准确性和细致的细节。例如，在图 7 的第一行中，心理图像包含一只卡通风格的鸭子，这并不对应于现实世界中的任何实际物品，从而导致检索失败。未来的工作可以集中在改进图像生成或编辑方法，以及开发处理错误生成视觉内容的方法。此外，为了实现最先进的性能，Paracosm 在基准数据集中使用了略有不同的提示模板。这主要由于不同数据集间修改文本的格式和具体性的差异，如图 4 所示。例如，修改文本可以包含相对标题和共享概念（参考 CIRCO）。为此，未来的工作可以开发自适应提示生成方法，利用 LMM 自动构建针对多样化多模态查询的最佳提示，减少对手动调整的需求，并增强在不同数据集分布下的鲁棒性。

# 6 结论

我们提出了Paracosm，这是一种新颖且相对简单的无训练零-shot CIR 方法。与大多数现有的零-shot CIR 方法不同，后者依赖于对多模态查询的文本描述生成，Paracosm 利用大语言模型生成多模态查询的心理图像，提供丰富的视觉信息，有利于CIR。此外，由于合成的心理图像与真实数据库图像之间存在合成到真实的领域差距，它进一步生成数据库图像的合成对应物。通过结合生成的视觉内容和生成的描述，Paracosm 在流行的基准数据集上实现了最先进的性能。Paracosm 显著优于现有的零-shot CIR 方法，甚至与监督学习方法相媲美。

# A Additional Details of Benchmark Datasets

In this section, we provide additional details on the benchmark datasets used in our experiments, namely CIRR [32], CIRCO [1], and Fashion IQ [59]. We summarize these datasets used in Table 6.

- The Compose Image Retrieval on Real-life images (CIRR) dataset constructs negative samples by mining visually similar images from NLVR2 [43]. To mitigate the false negative issue, CIRR organizes visually similar images into subsets. Since each subset contains images that are visually close to the target, evaluating retrieval within this constrained subset places greater demands on the model's discriminative capability.   
Composed Image Retrieval on Common Objects in context (CIRCO) is based on open-domain real-world images [30]. It is the first dataset for CIR with multiple ground-truth targets per query and fine-grained annotations. - Fashion Interactive Queries (Fashion IQ) contains fashion images across three categories (Shirt, Dress, Toptee) from Amazon.com. Following prior work [4, 12, 29, 39, 62], we evaluate on the validation set, as the test set is not publicly released.

# B Qualitative Analysis of Mental and Synthetic Counterparts

To gain deeper insight into the behavior of our Paracosm, we visualize the generated mental images for multimodal queries and synthetic counterparts for database images across datasets. As shown in Figure 10, we leverage Qwen-Image-Edit [58] to generate mental images from reference images, conditioned on the provided modification texts. Since the generated mental images are synthetic, they exhibit synthetic-to-real domain gaps relative to real database images. To address this, we generate a synthetic counterpart for each database image based on its detailed textual description, which is generated by an LMM. These synthetic counterparts preserve fine-grained visual attributes of the original images while aligning both queries and database images into a unified synthetic space, thereby mitigating the domain gap and enabling more effective retrieval.

# C Further Analyses and Ablation Study

We further analyze the impact of different LMMs for generating textual descriptions of mental images. Specifically, we replace Qwen2.5-VL-7B-Instruct [52] used in Paracosm with GPT-4o, the LMM employed by OSrCIR [49]. As shown in Table 8, switching to GPT-4o yields only marginal improvements in retrieval performance. Notably, the best performance across all datasets is achieved when $\lambda = 0 . 3$ (Fig. 6), indicating that retrieval performance is relatively insensitive to the choice of LMM for mental image description generation. This indicates that the core strength of Paracosm lies not in the scale of LMMs but in the unified synthetic space ("paracosm") constructed by our method.

Additionally, we evaluate the effect of incorporating brief textual descriptions $( \mathbf { t } _ { b r i e f } ^ { i } )$ for database images into Paracosm, as illustrated in Fig. 8. However, as shown in Tab. 8, this strategy yields performance gains only on the CIRR and CIRCO datasets, with negligible improvements on Fashion IQ. Consequently, we opt to exclude $\mathbf { t } _ { b r i e f } ^ { \ i }$ from our final pipeline to maintain simplicity and efficiency. Beyond the ablation study in Table 5, we further analyze an additional variant in Table 10 that incorporates detailed textual descriptions of database images into the matching process. However, this variant yields degraded performance compared to our final design. We attribute this decline to the granularity mismatch between the detailed descriptions of database images (which capture comprehensive visual attributes) and the brief descriptions of mental images (which focus on core semantic content).
