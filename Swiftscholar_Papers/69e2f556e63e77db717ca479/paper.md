# Abstract

Zero-shot composed image retrieval (ZS-CIR) is a challenging task that aims to retrieve images similar to a composed query of a reference image and a description, without relying on training on triplet datasets. Existing methods for this task often rely on predefined, fixed retrieval processes that combine the image and the modified text through hand-crafted templates, which suffer from two main issues: non-adaptive retrieval queries and user-unfriendly retrieval processes. To address these limitations, we propose a novel framework  Automatic multi-agent collaboration for zero-shot Composed Image Retrieval (AutoCIR). AutoCIR consists of three training-free agents  a planner, a retriever, and a corrector  that work together to iteratively identify and rectify mismatches. The planner guides the retriever by generating a customized target caption for the composed query and further refines this caption to resolve any semantic discrepancies based on feedback. The corrector, equipped with a chain-of-thought reasoning mechanism, conducts an in-depth evaluation of the retrieved results and generates appropriate self-correction actions. Extensive experiments on three benchmarks demonstrate that AutoCIR consistently outperforms previous competitive methods for ZS-CIR.

# 1 Introduction

Composed Image Retrieval (CIR) is a complex vision-language task that aims to retrieve target images based on a composed query consisting of a reference image and modification text. The goal is to identify images that are visually similar to the reference image but incorporate the specified modifications [22, 33, 41, 42]. Previous supervised approaches [4, 13, 32] have relied on carefully curated triplets (reference image, modifying text, target image) as training data to build specialized CIR models. However, generating such annotated triplets is labor-intensive and resource-consuming. To address this challenge, recent studies [3, 39] have introduced Zero-Shot CIR (ZS-CIR), enabling the generalization of CIR models without annotated triplets while maintaining retrieval accuracy. To implement ZS-CIR, researchers [3, 12, 39] typically leverage large-scale pre-trained vision-language models (VLMs) (e.g., CLIP [38]) to design various textual inversion methods. These methods learn a mapping network to project the reference image into pseudo-text tokens. Subsequently, a text modifier then uses handcrafted templates to combine these tokens with modifying captions, generating target captions for retrieval through VLMs. Recent advancements have integrated large language models (LLMs) for more dynamic textual inversion by recombining image captions with modification texts through manually designed prompts [24].

As shown in Figure 1 (a), existing methods usually use a pretrained captioning model to generate a caption for the reference image. Then, an LLM-based text modifier recombines the image caption and modifies text using manually crafted prompts to facilitate CIR. However, they rely on predefined, fixed retrieval processes that project the reference image into a language form, recombining them through hand-crafted templates, and then retrieve once. Despite their promise, they encounter two primary limitations: (1) Non-adaptive Retrieval Queries. Fixed retrieval strategies fail to adapt to complex, evolving scenarios involving multiple objects or intricate modifications, such as object removal or attribute adjustments [24]. This rigidity hampers the model's ability to comprehend and verify user intentions. (2) User-unfriendly Retrieval Processes. The reliance on fixed templates often forces users, especially novices, to engage in time-consuming trial-and-error cycles of editing descriptions and generating new captions, resulting in a cumbersome retrieval experience [36].

Drawing inspiration from human problem-solving behaviors [23, 37], where individuals create an initial plan for a specific task and then continuously adapt it based on the task's current state, we propose AutoCIR  Automatic multi-agent collaboration framework for zero-shot Composed Image Retrieval. AutoCIR performs self-checks to ensure alignment between intended modifications and retrieved images. As shown in Figure 1 (b), AutoCIR comprises three LLM-based agents, each with a specialized role: a planner, a retriever, and a corrector. The planner crafts a strategic plan (i.e., the target image caption) that guides the retrieval and refines the plan by identifying semantic discrepancies between retrieved images and intended modifications. The retriever performs zero-shot textto-image retrieval by aligning the planner's target with relevant images within the VLM embedding space. Finally, the corrector uses chain-of-thought (COT) reasoning for an in-depth evaluation of retrieved images through three stages: see, think, and judge. These stages involve parsing essential editing operations, detecting potential mismatches between the query and retrieval outcomes, and suggesting appropriate self-correction adjustments. To evaluate the effectiveness of our AutoCIR, we conduct experiments across three datasets: CIRCO [3], CIRR [32], and FashionIQ [43]. These datasets assess AutoCIR's compositional ability, showcasing its ability to generalize across different tasks. The results demonstrate that AutoCIR outperforms all previous CIR methods, especially in the FashionIQ domain, where it achieves a significant improvement of $7 . 2 6 \%$ in Recall $@ 1 0$ on average. In summary, the key contributions of our work are as follows: We introduce AutoCIR, a novel closed-loop framework that iteratively identifies and rectifies mismatches in CIR. AutoCIR pioneers automatic multi-agent collaboration to improve the composability of the retrieval process, ensuring accurate retrieval without the need for additional training data. • We propose three plug-and-play agents—a planner, a retriever, and a corrector—that enable the accurate retrieval of a composed image based on a reference image and a modification text in a model-agnostic manner. AutoCIR is user-friendly and can be seamlessly integrated into various LLM architectures and retrieval backbones. •We conduct extensive experiments on real-world datasets to demonstrate the effectiveness of AutoCIR. AutoCIR advances the state-of-the-art ZS-CIR, offering a more adaptable and usercentric retrieval process. The code and datasets to reproduce the results are available at https://github.com/coloreyes/AutoCIR.

# 2 Related Work

We review relevant research from two perspectives: compositional image retrieval and vision-language models for CIR.

# 2.1 Compositional Image Retrieval

The task of compositional image retrieval (CIR) [17, 26, 41], which focuses on retrieving a target image that best aligns with a given reference image and a modification description, has gained substantial attention in the research community. Traditional methods [17, 41] typically employ supervised strategies, training specialized models on extensive sets of human-annotated triplets that include a reference image, a target image, and a textual description of their difference. However, these conventional CIR methods are heavily dependent on high-quality annotated data, which is labor-intensive and costly to produce.

To address this limitation, zero-shot CIR (ZS-CIR) approaches [3, 12, 39] primarily operate without the need for human-annotated triplets. Instead, ZS-CIR methods leverage noisy image-text pairs to learn a mapping function that translates images into words. These generated word tokens are then combined with modification text for retrieval through vision-language models (VLMs). Recently, with the advent of large language models (LLMs), CIReVL [24] has introduced LLMs into ZS-CIR, designing a straightforward, training-free method. This approach involves captioning the reference image with a pre-trained VLM, then using an LLM-based text modifier to adjust the caption based on the modification text for retrieval. However, these methods generally rely on a static prompt strategy to combine the text-image query for constructing the desired target caption. This approach struggles with scenarios involving multiple objects and complex changes, resulting in a tedious trialand-error process that diminishes user experience. To address this challenge, we explore automatic multi-agent collaboration-based retrieval, which incorporates self-checking mechanisms to ensure alignment between intended modifications and retrieved images, thereby enhancing retrieval performance.

# 2.2 Vision-Language Models for CIR

Vision-language models (VLMs) such as CLIP [38], BLIP [27], and ALIGN [28], CoCa [45] are pre-trained on large-scale image-caption datasets, allowing them to map images and text into a shared embedding space [9, 47]. Most VLMs demonstrate remarkable zero-shot performance across downstream tasks, including zero-shot classification [8, 25, 29, 38], visual question answering (VQA) [7, 34, 46], semantic segmentation [6], recommender systems [14, 19] and social network analysis [10, 11, 48].

For ZS-CIR, many methods leverage large pre-trained VLMs, such as CLIP [38], as a backbone for image retrieval, aiming to project images and texts into a unified embedding space. Specifically, PALAVRA [12] uses CLIP to design mapping networks that project image embeddings into the text embedding space through self-supervised learning to retrieve personalized concepts. Pic2word [39] designs a mapping network to transform image features into pseudo-word tokens, which are then fused with modified text features for retrieval using CLIP. SEARLE [3] leverages LLMs to extract visual deltas via captions of reference images to enhance retrieval effectiveness. While these models directly employ VLMs for ZS-CIR through specialized modules [13] or fine-tuning [15], they are often limited by static prompting strategies, which can negatively impact user experience. Our research investigates an alternative approach: combine the standalone VLMs with an LLM to effectively perform ZS-CIR without additional training.

# 3 Methodology

In this section, we present a detailed description of our proposed AutoCIR. The overall framework of AutoCIR is illustrated in Figure 2. Next, we detail the three core components of AutoCIR: the reflective planner (Sec. 3.2), the cross-modal retriever (Sec. 3.3), and the chain-of-thought corrector (Sec. 3.4).

# 3.1 Preliminary

Let $\{ \mathcal { T } _ { q } , \mathcal { T } _ { q } \}$ denote a composed query, where $\scriptstyle { \mathcal { T } } _ { q }$ represents the reference image, and $\mathcal { T } _ { q }$ is the associated modifying text. The goal of composed image retrieval (CIR) is to retrieve a target image $\mathcal { T } _ { t } \in \mathcal { D }$ from a large image corpus $\mathcal { D }$ that incorporates the specified modifications in $\mathcal { T } _ { q }$ while largely retaining other visual similarities to the reference image $\scriptstyle { \mathcal { T } } _ { q }$ In zero-shot settings, mainstream ZS-CIR methods primarily design a customized mapping module $\phi$ : $\mathcal { T } _ { q } \to \mathcal { Z } _ { q }$ ,which projects the image $\scriptstyle { \mathcal { T } } _ { q }$ into pseudo text tokens $\scriptstyle { \mathcal { Z } } _ { q }$ .

Practically, $\phi$ is trained over intermediate image representations from the specific image encoder $\Psi _ { i }$ within VLMs. A carefully crafted prompt (e.g., "a photo of $\{ \mathcal { Z } _ { q } \}$ that $\{ \mathcal { T } _ { q } \} ^ { * } )$ is then applied to the inverted embedding $\scriptstyle { \mathcal { Z } } _ { q }$ and the modifying text $\mathcal { T } _ { q } .$ generating a target caption $\mathcal { T } _ { t }$ This caption $\mathcal { T } _ { t }$ is subsequently used for target image retrieval through a VLM-based retriever, which encodes the caption $\mathcal { T } _ { t }$ using the text encoder $\Psi _ { t }$ and candidate images $\tau \in \mathcal { D }$ using the image encoder $\Psi _ { i }$ The matching score $s$ is calculated using cosine similarity cos_sim: $S = \cos \_ \sin ( \Psi _ { i } ( \mathcal { I } ) , \Psi _ { t } ( \mathcal { T } _ { t } ) )$ .

# 3.2 Reflective Planner

The reflective planner of AutoCIR is designed to plan the target image caption, denoted as $\mathcal { T } _ { t }$ , that ensures reliable alignment between the composed query (image and text) and the retrieved results. Equipped with reasoning and self-reflection capabilities, the planner first generates an initial caption by reasoning over the composed image and text. It then refines this caption by assessing and reflecting on any discrepancies between the retrieved images and the intended modifications. Stage 1: Visual Perception. Inspired by existing ZS-CIR methods [24], we adopt a language-for-vision strategy that transforms visual signals into linguistic form, thereby offering a rudimentary understanding of the reference image $\scriptstyle { \mathcal { T } } _ { q }$ without the need for complex model training as required by textual inversion methods. Specifically, we use a pre-trained captioning model $\varphi$ (e.g., BLIP) to generate a language description of the reference image $\begin{array} { r } { \bar { I } _ { q } \colon C _ { q } = \varphi ( \bar { I } _ { q } ) } \end{array}$ . This description, $C _ { q }$ , resides in the natural language text domain, making it more accessible for human reasoning. This approach provides users with clearer insights into the retrieval process and enables potential intervention.

Stage 2: Recaption. Using a pre-defined template to recombine text in a specific and fixed manner (e.g., "a photo of $C _ { q }$ that $\mathcal { T } _ { q } ^ { \mathfrak { s } }$ limits the flexibility of retrieval, making it less adaptive to complex scenarios. When the composed query involves multiple objects or complex modifications, such as object removal, this rigidity often results in inaccurate retrieval and negatively impacts user experience. Ideally, an intelligent planning agent should effectively interpret the user's intent for modification and adjust the reference image caption $C _ { q }$ accordingly. To address this, we introduce an LLM with advanced reasoning capabilities and design an LLM-based modifier $\Phi _ { p }$ to produce cohesive, unified target captions. Specifically, we develop a simple prompt template $\mathcal { P }$ , inspired by [24], to combine the dense caption $C _ { q }$ and the modification text $\mathcal { T } _ { q }$ generating an initial image caption: $\dot { C _ { q } ^ { t } } = \Phi _ { p } ( \mathcal { P } \circ C _ { q } \circ \mathcal { T } _ { q } )$ .These captions $C _ { q } ^ { t }$ are then fed into the LLM-based modifier to ensure the task's nature and input details are clearly represented, enabling model to accurately interpret the conditions and produce the edited captions $\mathcal { T } _ { t }$ .Notably, this prompt construction is a one-time setup requiring minimal annotation effort, and we found it generalize well across diverse tasks.

Stage 3: Refining. In practice, composed retrieval often involves complex scenarios with multiple objects and nuanced modifications, driven by diverse, user-specific intentions. Due to the limited information available in the composed query, the planner may initially generate a target caption, $\mathcal { T } _ { t }$ , that is semantically misaligned with the target image, $\mathcal { T } _ { t }$ , in a single retrieval attempt. To address this misalignment, the intelligent planner needs to assess the gap between the feedback results and the intended modifications, thereby enhancing planning and reducing semantic bias. Specifically, we refine the initial caption $C _ { q } ^ { t }$ by querying the modifier $\Phi _ { P }$ with a concatenation of the refining prompt $\mathcal { P } _ { r }$ , the initial caption $C _ { q } ^ { t }$ (prefixed by "Initial caption:"), and the edited suggestion $s _ { q }$ obtained from the corrector (detailed in Sec. 3.4) (prefixed by "Edited Suggesti:") Thi ps can  ua: $C _ { q } ^ { r e } = \Phi _ { p } ( \mathcal { P } _ { r } \circ C _ { q } ^ { t } \circ S _ { q } )$ where $C _ { q } ^ { r e }$ represents the refined caption. These captions $\bar { C } _ { q } ^ { r e }$ are then fed into the same LLM-based modifier to produce the edited captions $\mathcal { T } _ { t } ^ { r e }$ . The refinement stage terminates when no further feedback is available or when the maximum number of iterations, $N$ ,is reached.

# 3.3 Cross-modal Retriever

To accomplish ZS-CIR, we leverage the advanced capabilities of VLMs (e.g., CLIP) as a retrieval agent to facilitate zero-shot textto-image retrieval. VLMs are pre-trained on large-scale image-text pairs, aligning textual and visual features in a shared embedding space, which demonstrates significant zero-shot retrieval abilities. Given the refined caption $\mathcal { T } _ { t } ^ { r e }$ (or the initial caption $\mathcal { T } _ { t }$ ), AutoCIR employs the image encoder $\Psi _ { i } ( \cdot )$ in VLMs to encode an image $I \in \mathcal { D }$ from an image database $\mathcal { D }$ and uses the textual encoder $\Psi _ { t }$ to model $\mathcal { T } _ { t } ^ { r e }$ . Consequently, the target image $\mathcal { T } _ { t }$ is retrieved as follows:

$$
\mathcal { T } _ { t } = \underset { I \in \mathcal { D } } { \operatorname { a r g m a x } } \frac { \Psi _ { i } ( I ) ^ { \top } \Psi _ { t } \left( \mathcal { T } _ { t } ^ { r e } \right) } { \Vert \Psi _ { i } ( I ) \Vert \cdot \left. \Psi _ { t } \left( \mathcal { T } _ { t } ^ { r e } \right) \right. } .
$$

where the final retrieved target image $I _ { t }$ is the one most similar to the description $\mathcal { T } _ { t } ^ { r e }$ of the target image $\mathcal { T } _ { t }$ .

# 3.4 Chain-of-Thought Corrector

To further refine the caption $\mathcal { T } _ { t }$ for effective retrieval, we design a self-correcting agent that uses a chain-of-thought (COT) mechanism to thoroughly analyze retrieved images and make subsequent corrections. This LLM-based corrector evaluates the retrieved images, disentangles user modifications, and offers corrective suggestions through a step-by-step process. The corrector comprises three stages: see, think, and judge. These stages identify inconsistencies between the retrieved results and the composed query, ultimately generating a set of re-editing operations. Stage 1: See. To obtain the content of the retrieved Top $_ { \cdot n }$ images, denoted as $N = \{ { \cal T } _ { t } ^ { 1 } , \cdots , { \cal T } _ { t } ^ { n } \}$ , we employ the captioning model $\varphi$ to extract key object characteristics and generate natural language captions $C _ { t } ^ { i }$ for each image $\boldsymbol { \mathcal { I } } _ { t } ^ { i }$ .These visual concepts are then selected and further analyzed in the judge stage to assess whether they align with the user's intended modifications. Stage 2: Think. In this stage, the corrector primarily identifies the target entities in the reference image that require modification and generates detailed editing operations (i.e., addition, deletion, or attribute modification) by reasoning over the composed query of text and image. First, we feed the caption, $C _ { q }$ of the reference image and the edited text, $\mathcal { T } _ { t } ^ { r e }$ , into the corrector to extract key entities that are semantically associated with the modification text, $\mathcal { T } _ { q }$ Next, the selected entities are combined with the user's modification text to generate fine-grained editing operations, $\varepsilon _ { q }$ providing more informative and detailed descriptions of the characteristic changes within these entities. Stage 3: Judge. During the judgment stage, we integrate the descriptions of the retrieved images $N$ and the editing operations $\varepsilon _ { q }$ into a prompt template. This prompts the LLM to verify whether the retrieved images meet the requirements specified in the editing operations. If retrieved images do not align with these criteria, we generate new descriptions $\scriptstyle { S _ { q } }$ to refine the target caption $C _ { q } ^ { t }$ Otherwise, we terminate the self-correcting process and use the retrieved images $N$ as the final output for the target user. This self-correcting mechanism aims to gather rationales that contain sufficient information to further optimize the retrieval process, thereby enhancing the user experience.

# 4.1 Experimental Settings

- Datasets. For evaluation, we conduct comprehensive experiments to assess the performance of our AutoCIR on three ZS-CIR benchmarks: CIRCO [3], CIRR [32], and FashionIQ [43]. Detailed descriptions of the three datasets are summarized as follows:

• CIRCO [3] is an extension of the COCO image dataset [30] and includes multiple ground truth annotations per query. Specifically, CIRCO has an average of 4.53 ground truth images per query. As CIRCO is specifically designed for evaluating ZS-CIR methods, it does not include a training split. Instead, it provides a validation split (220 queries) and a test split (800 queries), with the test split evaluated via a remote evaluation server. CIRR [32] is collected from a broader range of domains and contains complex descriptions, consisting of 21,552 real-life images extended from NLVR2 [40]. Specifically, CIRR is divided into a training set, validation set, and test set, with the test split evaluated separately via a remote evaluation server. FashionIQ [43] primarily contains fashion-related images and includes three categories: Shirt, Dress, and Toptee. Specifically, FashionIQ is composed of 30,134 triplets from 77,684 fashionrelated images, which are split into 6:2:2 for training, validation, and evaluation. FashionIQ targets realistic online shopping scenarios, such as chat windows. Therefore, the captions in FashionIQ are collected using a visual chat-based interface, resulting in more realistic user text queries. - Metrics. Following previous works [24, 39], we use Recall@k (R@k) as the evaluation metric for the CIRR and FashionIQ datasets. For CIRCO, where each query is associated with multiple target images, we use mean average precision $( \mathrm { m A P  @ k } )$ as a more fine-grained metric to evaluate the ranking quality of the retrieval results. - Baselines. To evaluate the effectiveness of AutoCIR for ZS-CIR, we compare our AutoCIR with 11 competitive baselines, including Image-only, Text-only, Image $+ ^ { ' }$ Text, Pic2word [39], Slerp [21], Lin-CIR [16], PALAVRA [12], SEARLE [3], SEARLE-OTI [3], CIReVL [24], and LDRE [44]. Detailed descriptions of all baselines are summarized as follows: •Image-only directly inputs the reference images into the CLIP image encoder and then computes similarity for retrieval. •Text-only primarily focuses only on the modification text features, extracted by the CLIP text encoder, which are used as retrieval features to calculate similarity.   
Image $^ { + }$ Text combines visual and textual features by averaging them for the subsequent retrieval process.   
Pic2word follows a training-dependent strategy that utilizes a pre-trained textual inversion network optimized with contrastive loss to construct the pseudo-word token, which is then combined with the modification text for retrieval.   

•Slerp employs spherical linear interpolation to directly integrate visual and textual representations by identifying an intermediate embedding of both for subsequent retrieval. LinCIR designs a language-only training strategy that projects textual features into the token embedding space and constructs a new text by replacing the keyword tokens of the original text. PALAVRA is a two-stage training-dependent method based on textual inversion, involving a pre-trained mapping function followed by the optimization of the pseudo-word token.   
SEARLE follows a training-dependent mechanism in which pseudo-word tokens for unlabeled images are generated using an optimization-based textual inversion strategy and then distilled into a textual inversion network. SEARLE-OTI is a variant of SEARLE that removes the distillation network. CIReVL proposes a training-free method that utilizes a pretrained generative VLM to generate a caption for the reference image and then asks an LLM to recompose the caption based on the textual target modification for subsequent retrieval.   
LDRE designs a training-free LLM-based divergent reasoning and ensemble strategy to capture diverse possible semantics of the composed result for ZS-CIR.

Among them, Image-only, Text-only, and Image $+$ Text are simple techniques directly applied to a single modality; Pic2word, Slerp, LinCIR, PALAVRA, SEARLE, and SEARLE-OTI are trainingdependent methods based on the textual inversion strategy; CIReVL and LDRE are LLM-based training-free methods that utilize LLMs to recompose captions generated from the reference image based on the textual target modification. - Implementation Details. We use the pre-trained BLIP-2 [27] with the language model OPT-6.7b as our captioning model. For the retriever, we employ ViT-B/32 (ViT-B) and ViT-L/14 (ViT-L) CLIP models from OpenCLIP [20]. By default, we use GPT-4o-Mini [5] as our large language model (LLM), with additional ablations considering both Qwen-Turbo [2], GPT-3.5-Turbo [5] and GPT-4 [1]. The feature space dimensions for each model are as follows: 512 for B/32, and 768 for L/14. Our proposed model, AutoCIR, is implemented in Pytorch [35] and runs on two NVIDIA RTX4090 GPUs.

# 4.3 Ablation Studies (RQ2)

In this subsection, we provide an in-depth analysis of our design choices. Unless otherwise specified, comparisons are made on the FashionIQ test split.

•Impact of chain-of-thought corrector. To assess the impact of the corrector in AutoCIR, we designed two variant models: (1) w/o Corrector, where the chain-of-thought corrector is removed, and (2) w/o COT, where only the chain-of-thought mechanism within the corrector is removed. Results in Table 3 reveal a significant drop in performance when the corrector is removed $( w / o$ Corrector), highlighting its crucial role in analyzing the retrieved images and implementing subsequent corrections. This process enables the further refinement of the modified descriptions to align more closely with user intent. Additionally, the variant $( w / o C O T )$ also shows inferior performance, confirming the effectiveness of reasoning during the image evaluation process and underscoring the importance of our chain-of-thought self-correcting strategy. These findings provide substantial evidence for the effectiveness of our corrector in conducting an in-depth evaluation of retrieved results and generating appropriate self-correction actions. Impact of captioning quality. Given that our AutoCIR framework relies on generated captions of reference images for ZS-CIR, we conducted an ablation study to assess the importance of the captioning model choice. Table 3 summarizes these results on the Toptee test split. We experimented with alternative captioning models, including QwenVL [2], LLAVA-OV [31], and CoCa [45]. The results show that most public captioning models yield similar performance, with a slight improvement using CoCa. This indicates that the quality of reference image captions has a tangible impact on the retrieval process in ZS-CIR. •Impact of textual reasoning. Since our method relies heavily on the LLM for multi-agent collaboration in ZS-CIR, we conducted comparative experiments on the Toptee dataset using various configurations, summarized in Table 3: no LLM (with fixed templates), Qwen-Turbo, GPT-4o-Mini, GPT-3.5-Turbo, and GPT-4o. The results indicate that LLM-based methods significantly outperform the fixed template method, highlighting the substantial impact of dynamic textual reasoning in LLM for generating target captions. Furthermore, the GPT variants, GPT-3.5-Turbo and GPT-4, show relative improvements of $4 . 2 1 \%$ and $5 . 4 6 \%$ in Recall@10 compared to Qwen-Turbo, respectively.

# 4.4 Parameter Sensitivity (RQ3)

Figure 3 presents the sensitivity analysis of AutoCIR 's parameters. •Feedback Round Analysis. Since our model (AutoCIR) introduces a multi-agent collaboration mechanism to iteratively refine the target caption, we conducted a parameter analysis experiment to examine the effect of feedback rounds $N$ on the final performance. The results for the Toptee category of the FashionIQ validation set are reported in Figure 3. As shown, using re-edited suggestions through the self-correcting enhances model performance, providing strong evidence that interactions between multiple LLM-based agents facilitate a deeper understanding of user intent and lead to more accurate image retrieval. This highlights the potential of multi-agent collaboration in creating a user-friendly composed image retrieval system. Notably, the performance of AutoCIR peaks at $N = 2$ Beyond this point, with additional rounds $\left( N > 2 \right)$ , performance starts to decline, likely due to the introduction of noise (irrelevant suggestions) into the planner, possibly stemming from corrector-induced hallucinations. Interestingly, at $N = 5$ , we observe an increase in retrieval performance on Recall $@ 5 0$ but a decrease on Recall@10. This suggests that while the self-correcting strategy in AutoCIR increases diversity, it reduces accuracy with too many feedback rounds, aligning with the intuition that ZS-CIR is inherently a fuzzy retrieval task [44].

For example, it overlooks "purple" and "single-shouldered" in case one, as well as "long sleeves" and "monkey" in case two. This provides key evidence that methods using inflexible retrieval strategies fail to adapt to complex scenarios involving multiple objects and intricate modifications. In contrast, our AutoCIR effectively identifies all target modifications and accurately retrieves the target images. This can be attributed to the self-correcting ability in AutoCIR enabled by multi-agent collaboration. Following that, Figure 6 visualizes exemplary failure cases. Since our AutoCIR primarily performs composed image retrieval in the language domain, it is straightforward for users to understand these particular failure cases. Specifically, in the first case, we observe that AutoCIR generates a suitable retrieval instruction for the target image. However, the model retrieves incorrect results. This can be attributed to the fact that the training-free framework lacks customized knowledge associated with specific attributes, such as the "Volkswagen saying". Additionally, we also observe instances where the LLM generates an accurate description of the target image, as seen in case two. Despite the captioning and reasoning components functioning correctly, the CLIP model fails to retrieve an accurate image. This highlights potential limitations in the retrieval backbone and the challenges of aligning generated captions with visual representations. Reasoning Sanity Check. To assess whether the modified descriptions generated by the reflective planner align with the userintended modifications, we conducted a quantitative evaluation of text-image alignment using TIFA [18]. Unlike simpler metrics, such as the CLIP similarity score, TIFA provides a more comprehensive measure of cross-modal alignment by converting it into a series of question-answering tasks. The experimental results on the Dress and Shirt datasets are presented in Figure 5. These results demonstrate that the modified descriptions generated by our AutoCIR exhibit significantly higher and more consistent average alignment compared to plain image captions.

# 5 Conclusion

In this study, we introduced AutoCIR, a novel automatic multi-agent collaboration framework for zero-shot composed image retrieval. AutoCIR performs self-correction to ensure alignment between retrieved images and intended modifications, and it comprises three LLM-based agents: a planner, a retriever, and a corrector. The planner crafts a strategic plan to guide retrieval, refining the plan by eliminating semantic discrepancies. The retriever performs zeroshot text-to-image retrieval by aligning the planner's target with relevant images. The corrector uses chain-of-thought reasoning for an in-depth evaluation of the retrieved images, generating appropriate self-correction adjustments. The entire process operates in a closed-loop manner, iteratively identifying and rectifying errors through multi-agent collaboration. This design is user-friendly and operates without the need for additional training. Extensive experiments on three benchmark datasets demonstrated the superior performance of our framework compared to existing methods. In the future, we plan to design a lightweight multi-agent collaboration framework to reduce computational and cost overhead, making it more suitable for real-world retrieval systems. Additionally, we will explore extending AutoCIR to other domains, such as visual question answering and autonomous driving.

# A Prompt for AutoCIR

In this section, we illustrate all the prompts used in our paper. Part of our prompts are taken from CIReVL [24].

# Prompt for the planner in the recaption stage.

Task Instruction: I have an image. Given an instruction to edit the image, carefully generate a description of the edited image. I will put my image content beginning with "Image Content:". The instruction I provide will begin with "Instruction:". The edited description you generate should begin with "Edited Description:". The edited description you generate should be complete and can cover various semantic aspects, such as cardinality, addition, negation, direct addressing, compare & change, comparative, conjunction, spatial relations & background, viewpoint. The edited description needs to be as simple as possible. The instruction does not need to explicitly indicate which type it is. Avoid adding imaginary things. Each time generate one instruction and one edited description only. Keep the edited description as short as possible.

# Prompt for the planner in the refining stage.

Task Instruction: Assume you are an experienced composed image retrieval expert, skilled at precisely generating new image descriptions based on a reference image's description and the user's modification instructions. 1. {base-prompt} 2Image Content: {all-captions[i]} Instruction: {relative-captions[i]} However, the caption you generated is not entirely appropriate, as it does not fully integrate the content of the reference image and the user's modification instructions, resulting in inaccurate retrieval results. After comparing the retrieved images, the specific modification suggestions are as follows: {suggestions[i]}. Please think step by step and make appropriate adjustments to the caption you generated last time. Fully understand the reference image and the user's modification requests to create a caption that better aligns with the modifications, ensuring more accurate compositional retrieval.

# Prompt for the corrector.

Task Instruction: Assume you are an experienced composed image retrieval expert, skilled at precisely generating new image descriptions based on a reference image's description and the user's modification instructions. Your task is to help improve the effectiveness of compositional image retrieval by generating precise modification suggestions that will assist another LLM in producing a better image description. Moreover, a retrieval has been performed based on this modified description. Thus, your task is to analyze the last retrieval result and provide modification suggestions and please follow the below steps to finish this task. 1. Step 1: Identifying Modifications Your first task is to identify the modifications and generate corresponding modification phrases. Specifically, here is the description of the reference image: "{image-caption}." Here are the user's modification requests: "{relative-caption}" By deeply understanding the image description and the user's modifications, please generate the following two types of modification phrases: (1) If the modification involves changing the characteristics of an entity in the original reference image, please specify the changes, (2) If the modification involves adding or deleting an entity, please specify the additions or deletions. Step 2: Analyzing the Top-k Retrieved Image. Compare the modification phrases identified in Step 1 with the description of the top-k retrieved image : "{captions}". Note that this retrieval is performed with the modified description generated by another LLM, which has been mentioned above. Determine if the retrieved image meets the user's modification instructions. If it matches after excluding subjective modifications (e.g., "casual", "relaxed"), respond with: "Good retrieval, no more loops needed." If there are unmet modification phrases, proceed to Step 3. 3.Step 3: Providing Modification Suggestions. For any unmet modifications identified in Step 2, suggest targeted changes to help the LLM regenerate an improved modified description. Keep suggestions concise and specific to ensure they effectively guide the LLM.
