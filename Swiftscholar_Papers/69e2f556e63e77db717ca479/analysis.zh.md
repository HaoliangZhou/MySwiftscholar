## 1.5. 摘要
零样本组合图像检索 (ZS-CIR) 旨在在不依赖三元组数据集训练的情况下，根据由参考图像和描述组成的组合查询来检索相似图像。现有的 ZS-CIR 方法通常依赖于预定义的、固定的检索过程，通过手工制作的模板结合图像和修改后的文本。这些方法存在两个主要问题：检索查询的非自适应性和检索过程的用户不友好性。为了解决这些局限性，本文提出了一个新颖的框架——用于零样本组合图像检索的自动多智能体协作框架 (AutoCIR)。AutoCIR 由三个无需训练的智能体——规划者、检索器和校正器——组成，它们协同工作以迭代地识别和纠正不匹配。规划者通过为组合查询生成定制的目标描述来引导检索器，并根据反馈进一步细化该描述以解决语义差异。校正器配备了思维链 推理机制，对检索结果进行深入评估并生成适当的自我纠正操作。在三个基准数据集上的广泛实验表明，AutoCIR 始终优于以前具有竞争力的 ZS-CIR 方法。

# 2. 整体概括

## 2.1. 研究背景与动机
**核心问题：**
组合图像检索 (CIR) 的任务是根据一个组合查询（包含一张参考图像和一段修改文本）来检索目标图像。目标图像需要在视觉上与参考图像相似，同时包含文本中指定的修改。传统的监督学习方法需要大量人工标注的三元组（参考图、修改文本、目标图），成本高昂。虽然零样本组合图像检索 (ZS-CIR) 试图通过利用预训练的视觉语言模型 (VLM) 来解决数据依赖问题，但现有方法通常采用固定的、静态的模板（例如 "a photo of [image caption] that [modification]"）来生成检索查询。

**挑战与空白：**
1.  **非自适应的检索查询：** 现有的固定模板策略无法灵活适应复杂的场景，例如涉及多个对象的操作、对象移除或复杂的属性调整。这种僵化限制了模型理解和验证用户意图的能力。
2.  **用户不友好的检索过程：** 由于缺乏自适应调整，用户（尤其是新手）往往需要进行繁琐的试错循环，不断编辑描述文本以获得满意的检索结果，体验较差。

**切入点与创新思路：**
受人类解决问题行为的启发（即制定计划并根据当前状态不断调整），本文提出了 **AutoCIR** 框架。该框架引入了多智能体协作机制，通过三个专门的智能体（规划者、检索器、校正器）形成一个闭环系统。系统不再是一次性生成查询，而是通过自我检查和反馈机制，迭代地修正检索指令，从而在无需额外训练的情况下实现更准确、更符合用户意图的检索。

## 2.2. 核心贡献/主要发现
1.  **提出了 AutoCIR 框架：** 这是一个新颖的闭环框架，通过迭代识别和纠正 CIR 中的不匹配来工作。它开创了自动多智能体协作在提高检索过程组合性方面的应用，确保了在无需额外训练数据的情况下进行准确检索。
2.  **设计了三个即插即用的智能体：**
    *   **规划者：** 负责生成和细化目标图像描述。
    *   **检索器：** 利用 VLM 执行零样本文本到图像的检索。
    *   **校正器：** 利用思维链 机制评估检索结果并提供修正建议。
        这三个智能体以模型无关的方式协同工作，可以无缝集成到各种 LLM 架构和检索主干网络中。
3.  **实验验证了有效性：** 在 CIRCO、CIRR 和 FashionIQ 三个真实数据集上的广泛实验表明，AutoCIR 推进了最先进的 ZS-CIR 性能，特别是在 FashionIQ 数据集上，Recall@10 平均提升了 7.26%。

# 3. 预备知识与相关工作

## 3.1. 基础概念
为了理解本文，读者需要掌握以下核心概念：

- **组合图像检索：**
  这是一种检索任务，输入由两部分组成：一张参考图像和一段描述修改内容的文本。目标是找到一张既保留了参考图像主要视觉特征，又应用了文本描述中修改内容的目标图像。例如，参考图是一只“红狗”，修改文本是“变成黑色”，目标图应该是一只“黑狗”。

- **零样本学习：**
  指模型在没有见过特定任务训练数据的情况下，仅利用预训练知识或通用数据就能完成该任务的能力。在 ZS-CIR 中，意味着模型不需要在特定的三元组数据集上进行训练。

- **视觉语言模型：**
  如 CLIP、BLIP 等模型。它们在大规模图像-文本对上进行预训练，能够将图像和文本映射到同一个共享的嵌入空间。在这个空间中，语义相似的图像和文本距离更近，从而实现跨模态的检索。

- **文本反演：**
  一种技术，旨在学习将图像映射到文本嵌入空间中的特定“伪词元”。通过这种方式，图像可以像文本一样被处理，从而与修改文本结合进行检索。

- **思维链：**
  一种提示策略，通过引导大语言模型 (LLM) 逐步展示推理过程（即“思考”步骤），从而提高模型在复杂任务上的表现。

## 3.2. 前人工作
- <strong>组合图像检索 (CIR)：</strong>
  早期方法主要依赖监督学习，需要大量人工标注的三元组。例如，CIRR 和 FashionIQ 数据集上的早期工作通常训练专门的编码器来融合图像和文本特征。
- <strong>零样本组合图像检索 (ZS-CIR)：</strong>
  为了减少对标注数据的依赖，研究者开始利用 VLM。例如：
    - **Pic2word:** 设计映射网络将图像特征转换为伪词元，再与修改文本融合。
    - **SEARLE:** 利用 LLM 提取视觉增量信息。
    - **CIReVL:** 利用预训练生成式 VLM 为参考图像生成标题，然后使用 LLM 根据修改文本重新组合标题。
      这些方法虽然不需要训练，但大多依赖于静态的提示策略，即通过固定的模板将图像标题和修改文本拼接，缺乏对复杂修改的适应性。

## 3.3. 技术演进
该领域经历了从“监督学习（依赖大量标注）”到“零样本学习（依赖预训练 VLM 和静态模板）”的演变。本文的工作处于这一演进的前沿，进一步引入了“动态推理与多智能体协作”，利用 LLM 的强大推理能力，将静态的一次性检索转变为动态的、迭代的自我修正过程。

## 3.4. 差异化分析
与现有的 ZS-CIR 方法（如 CIReVL）相比，AutoCIR 的核心区别在于：
1.  **多智能体协作：** 不再是单一的“生成描述 -> 检索”流程，而是分解为规划、检索、校正三个角色，各司其职。
2.  **闭环反馈机制：** 引入了“校正器”对检索结果进行评估，并将反馈传递给“规划器”进行修正，形成闭环。这使得系统能够自我发现错误并改进，而不是像以前那样一旦生成了错误的查询就无法挽回。
3.  **用户友好性：** 由于过程在语言域进行，用户可以理解系统的推理步骤，甚至可以在中间环节介入，极大地提升了用户体验。

# 4. 方法论

## 4.1. 方法原理
AutoCIR 的核心思想是模拟人类专家的检索过程：首先根据参考图和修改文本制定一个初步计划（生成目标描述），然后去检索图片。拿到检索结果后，专家会检查这些图片是否符合要求。如果不符合，专家会分析原因（比如漏掉了某个属性），然后修改检索计划，再次检索，直到满意为止。
这一过程通过三个基于 LLM 的智能体实现：
- <strong>Reflective Planner (反思规划者)：</strong> 负责生成和优化检索用的文本描述。
- <strong>Cross-modal Retriever (跨模态检索器)：</strong> 负责根据文本描述在图像库中找到最匹配的图像。
- <strong>Chain-of-Thought Corrector (思维链校正器)：</strong> 负责审视检索结果，判断是否达标，并给出修改建议。

## 4.2. 核心方法详解

### 4.2.1. 预备知识
首先，我们定义组合查询为 $\{ \mathcal { I } _ { q } , \mathcal { T } _ { q } \}$，其中 $\mathcal { I } _ { q }$ 是参考图像，$\mathcal { T } _ { q }$ 是修改文本。目标是从图像语料库 $\mathcal { D }$ 中检索出目标图像 $\mathcal { I } _ { t }$。

在零样本设置中，主流方法设计一个映射模块 $\phi : \mathcal { I } _ { q } \to \mathcal { Z } _ { q }$，将图像 $\mathcal { I } _ { q }$ 投影为伪文本词元 $\mathcal { Z } _ { q }$。然后，通过精心制作的提示词（例如 "a photo of $\{ \mathcal { Z } _ { q } \}$ that $\{ \mathcal { T } _ { q } \}$"）将 $\mathcal { Z } _ { q }$ 和 $\mathcal { T } _ { q }$ 结合，生成目标描述 $\mathcal { T } _ { t }$。
最后，通过 VLM 的文本编码器 $\Psi _ { t }$ 编码 $\mathcal { T } _ { t }$，图像编码器 $\Psi _ { i }$ 编码候选图像 $\tau \in \mathcal { D }$，计算余弦相似度：
$$
S = \cos \_ \sin ( \Psi _ { i } ( \mathcal { I } ) , \Psi _ { t } ( \mathcal { T } _ { t } ) )
$$
这里，$\cos \_ \sin$ 表示余弦相似度函数，用于衡量文本特征和图像特征之间的匹配程度。

### 4.2.2. 反思规划者
规划者的目标是生成一个可靠的目标图像描述 $\mathcal { T } _ { t }$。它包含三个阶段：

**阶段 1：视觉感知**
为了将视觉信号转换为语言形式，规划者使用预训练的标题生成模型 $\varphi$（如 BLIP）为参考图像生成描述：
$$
C _ { q } = \varphi ( \mathcal { I } _ { q } )
$$
其中 `C _ { q }` 是参考图像的文本描述。这样做的好处是让后续的推理过程都在语言域进行，便于理解和干预。

**阶段 2：重新描述**
为了避免固定模板的局限性，规划者引入了一个基于 LLM 的修改器 $\Phi _ { p }$。它通过一个简单的提示模板 $\mathcal { P }$，将参考图像描述 `C _ { q }` 和修改文本 $\mathcal { T } _ { q }$ 结合，生成初始的目标图像描述：
$$
C _ { q } ^ { t } = \Phi _ { p } ( \mathcal { P } \circ C _ { q } \circ \mathcal { T } _ { q } )
$$
这里 $\circ$ 表示字符串的拼接操作。$C _ { q } ^ { t }$ 是经过 LLM 理解和整合后的初步目标描述。

**阶段 3：细化**
这是规划者智能化的关键。由于初次尝试可能存在语义偏差，规划者需要根据反馈来调整。它利用校正器提供的编辑建议 `S _ { q }`（详见 4.2.4 节），通过一个细化提示模板 $\mathcal { P } _ { r }$，对初始描述进行修正：
$$
C _ { q } ^ { r e } = \Phi _ { p } ( \mathcal { P } _ { r } \circ C _ { q } ^ { t } \circ S _ { q } )
$$
其中 $C _ { q } ^ { r e }$ 是细化后的描述。这个过程会迭代进行，直到没有反馈或达到最大迭代次数 $N$。

### 4.2.3. 跨模态检索器
检索器利用 VLM（如 CLIP）强大的零样本检索能力。给定规划器生成的（细化后的）目标描述 $\mathcal { T } _ { t } ^ { r e }$，检索器在图像库 $\mathcal { D }$ 中寻找最相似的图像。
具体来说，它计算图像库中每个图像 $I$ 与目标描述的相似度，并返回相似度最高的图像作为目标图像 $\mathcal { I } _ { t }$：
$$
\mathcal { I } _ { t } = \underset { I \in \mathcal { D } } { \operatorname { a r g m a x } } \frac { \Psi _ { i } ( I ) ^ { \top } \Psi _ { t } \left( \mathcal { T } _ { t } ^ { r e } \right) } { \Vert \Psi _ { i } ( I ) \Vert \cdot \left\Vert \Psi _ { t } \left( \mathcal { T } _ { t } ^ { r e } \right) \right\Vert } .
$$
这个公式的含义是：在所有候选图像 $I$ 中，找到那个使得图像特征 $\Psi _ { i } ( I )$ 和文本特征 $\Psi _ { t } \left( \mathcal { T } _ { t } ^ { r e } \right)$ 的点积（归一化后即余弦相似度）最大的图像。

### 4.2.4. 思维链校正器
校正器负责对检索结果进行“体检”，并开出“处方”。它也包含三个阶段：

**阶段 1：观察**
为了评估检索结果，校正器首先需要“看”到检索到了什么。它使用标题生成模型 $\varphi$ 为检索到的 Top-$n$ 张图像 $N = \{ \mathcal { I } _ { t } ^ { 1 } , \cdots , \mathcal { I } _ { t } ^ { n } \}$ 生成描述 $C _ { t } ^ { i }$。这些描述将用于后续的判断。

**阶段 2：思考**
校正器分析用户的修改意图。它将参考图像描述 `C _ { q }` 和编辑文本 $\mathcal { T } _ { q } ^ { r e }$ 输入 LLM，提取出需要修改的关键实体和具体的编辑操作（如增加、删除、修改属性），生成详细的编辑操作 $\varepsilon _ { q }$。这一步利用了 LLM 的推理能力，将模糊的修改文本转化为具体的操作指令。

**阶段 3：判断**
这是决策阶段。校正器将检索到的图像描述 $N$ 和编辑操作 $\varepsilon _ { q }$ 放在一起进行比对。如果检索到的图像满足编辑操作的要求，则输出“Good retrieval, no more loops needed.”，流程结束。如果不满足，校正器会生成新的修改建议 `S _ { q }`，告诉规划器哪里做错了，需要如何修改描述。这个建议 `S _ { q }` 将被反馈给规划器进行下一轮的细化。

# 5. 实验设置

## 5.1. 数据集
实验在三个标准的 ZS-CIR 基准数据集上进行：
1.  **CIRCO:** 基于 COCO 数据集的扩展。每个查询平均有 4.53 个真实标注图像。它专门用于评估 ZS-CIR，没有训练集，只有验证集和测试集。由于存在固有的语义模糊性（一个修改可能对应多个合理结果），该数据集适合测试模型的鲁棒性。
2.  **CIRR:** 包含 21,552 张真实生活图像，源自 NLVR2，涵盖更广泛的领域。描述较为复杂。分为训练集、验证集和测试集。
3.  **FashionIQ:** 主要包含时尚类图像（衬衫、连衣裙、上衣）。包含 30,134 个三元组。其描述通过视觉聊天界面收集，更接近真实的用户查询场景。

## 5.2. 评估指标
1.  <strong>Recall@k (召回率@k):</strong>
    - **概念定义:** 用于评估在检索结果的前 $k$ 个项目中，是否包含正确目标图像的概率。它是衡量检索系统准确性的核心指标，关注模型能否将正确答案排在前列。
    - **数学公式:**
      $$
        \text{Recall@k} = \frac{1}{|\mathcal{Q}|} \sum_{q \in \mathcal{Q}} \mathbb{I}(\text{rank}(q, \text{target}) \leq k)
        $$
    - **符号解释:** $\mathcal{Q}$ 是查询集合，$|\mathcal{Q}|$ 是查询总数。$\text{rank}(q, \text{target})$ 是目标图像在查询 $q$ 的结果列表中的排名。$\mathbb{I}(\cdot)$ 是指示函数，如果条件成立（排名 $\leq k$）则为 1，否则为 0。

2.  <strong>Mean Average Precision @k (平均精度均值@k, mAP@k):</strong>
    - **概念定义:** 主要用于 CIRCO 数据集，因为每个查询可能有多个正确答案。它计算每个查询在前 $k$ 个结果中的平均精度，然后对所有查询取平均。它比 Recall@k 更精细地考虑了排序质量。
    - **数学公式:**
      $$
        \text{mAP@k} = \frac{1}{|\mathcal{Q}|} \sum_{q \in \mathcal{Q}} \left( \frac{1}{\min(k, m_q)} \sum_{i=1}^{k} P(i) \times \text{rel}(i) \right)
        $$
    - **符号解释:** $m_q$ 是查询 $q$ 的相关图像总数。`P(i)` 是前 $i$ 个结果的精度。$\text{rel}(i)$ 是一个二值函数，表示第 $i$ 个结果是否相关。

# 7. 总结与思考

## 7.1. 结论总结
本文提出了 AutoCIR，一个新颖的自动多智能体协作框架，用于解决零样本组合图像检索中的非适应性和用户不友好问题。通过引入规划者、检索器和校正器三个智能体，AutoCIR 实现了检索过程的闭环和自我修正。实验表明，该方法在多个基准数据集上取得了最先进的性能，显著提升了检索的准确性和用户体验。

## 7.2. 局限性与未来工作
**局限性：**
1.  **计算成本：** 依赖多个 LLM 调用和迭代反馈，导致计算开销和 API 成本较高，实时性可能受限。
2.  **幻觉风险：** LLM 在生成描述或反馈时可能产生幻觉，引入噪声，影响检索效果。
3.  **检索器瓶颈：** 实验表明，即使生成的描述非常准确，CLIP 等检索主干网络可能仍无法检索到最完美的图像，说明视觉-语言对齐仍有提升空间。

**未来工作：**
1.  设计轻量级的多智能体协作框架，降低计算成本，使其更适合实时系统。
2.  将 AutoCIR 扩展到其他领域，如视觉问答 (VQA) 和自动驾驶。
3.  探索更强大的检索主干网络或微调策略，以更好地对齐生成的文本描述。

## 7.3. 个人启发与批判
**启发：**
1.  **多智能体协作的潜力：** 本文展示了将复杂任务拆解并分配给不同智能体的巨大潜力。这种“分而治之”并结合反馈的思路，可以广泛应用于其他需要多步推理和自我修正的 AI 任务中。
2.  **语言域的优势：** 将视觉检索问题转化为语言推理问题，不仅利用了 LLM 强大的语义理解能力，还带来了天然的可解释性和用户友好性。这为构建“以人为本”的 AI 系统提供了很好的范例。
3.  **自我修正机制的重要性：** 一次性的生成往往是不完美的，引入自我检查和修正机制是提升 AI 系统鲁棒性的关键。

**批判与思考：**
1.  **效率与性能的权衡：** 虽然 AutoCIR 性能优异，但其高昂的计算成本是实际部署的一大障碍。未来的研究应重点关注如何在保持性能的同时大幅减少 LLM 的调用次数或使用更小的模型。
2.  **对 VLM 的依赖：** AutoCIR 的性能上限在很大程度上受限于底层的 VLM（如 CLIP）。如果 CLIP 无法区分某些细微的视觉差异，无论 LLM 生成的描述多么精确，检索结果都不会理想。因此，结合视觉域的反馈（而不仅仅是基于标题的文本反馈）可能是一个改进方向。
3.  **反馈循环的稳定性：** 实验显示过多的反馈轮次会导致性能下降。如何设计更稳定的反馈机制，避免“纠正过头”或引入噪声，是需要进一步研究的问题。或许可以引入置信度评估机制，只在不确定时才进行迭代。
