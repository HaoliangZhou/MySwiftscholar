# 1. 论文基本信息
## 1.1. 标题
论文标题为《ReTrack: Evidence-Driven Dual-Stream Directional Anchor Calibration Network for Composed Video Retrieval》，核心主题是面向<strong>组合视频检索（Composed Video Retrieval, CVR）</strong>任务，提出一种证据驱动的双流方向锚校准网络，解决现有方法中组合特征方向偏向参考模态、检索精度不足的问题。

## 1.5. 摘要
- **研究目的**：解决组合视频检索任务中视频与文本模态信息密度差异大，传统融合方法生成的组合特征偏向参考视频，导致检索性能不佳的问题。
- **核心方法**：提出ReTrack框架，包含三个核心模块：①语义贡献解耦：拆分组合特征中参考视频和修改文本的语义贡献；②组合几何校准：生成双模态方向锚，校准组合特征的方向偏差；③可靠证据驱动对齐：基于证据理论计算双向匹配证据，提升相似性估计的可靠性。
- **主要结果**：在3个基准数据集（覆盖CVR和组合图像检索CIR任务）上均达到<strong>最先进的（state-of-the-art, SOTA）</strong>性能，相比现有最优CVR模型平均指标提升2.09%。
- **关键结论**：校准组合特征的方向偏差可有效提升多模态查询理解能力，证据学习可降低检索不确定性，且该框架具备跨任务泛化能力。

# 2. 整体概括
## 2.1. 研究背景与动机
### 2.1.1. 核心问题
组合视频检索（CVR）是当前多模态检索领域的新兴任务：输入为「参考视频+修改文本」的多模态查询，模型需要检索出符合修改文本描述的目标视频。例如参考视频是「海边的风景」，修改文本是「添加一个冲浪的人」，目标就是检索出「有人在海边冲浪」的视频。
现有CVR方法存在的核心问题是：视频模态包含丰富的时序、视觉信息，文本模态语义凝练，两者信息密度差异极大，导致融合生成的组合特征过度偏向参考视频，忽略修改文本的语义，最终检索结果和参考视频高度相似但不符合修改要求。
### 2.1.2. 领域重要性与现有挑战
随着短视频数据的爆发式增长，灵活的多模态视频检索需求日益迫切，CVR可广泛应用于智能交互、多模态推理等场景。但现有研究存在三个核心挑战：
1. **模态贡献纠缠**：组合特征中参考视频和修改文本的语义贡献相互耦合，无法显式区分，难以校正偏向性。
2. **组合特征显式优化难**：缺乏明确的监督信号衡量组合特征是否存在方向偏差，也无法直接对特征方向进行校准。
3. **检索不确定性**：训练三元组标注成本高，候选池中存在大量语义相似的负样本，仅依靠组合特征与候选的余弦相似度难以准确区分正负样本。
### 2.1.3. 创新切入点
现有CVR方法均未关注组合特征的方向偏差问题，本文首次从特征空间几何分布的角度切入：通过显式解耦双模态语义贡献、生成方向锚校准组合特征的方向，再结合证据深度学习量化匹配可靠性，解决上述三个挑战。
## 2.2. 核心贡献/主要发现
### 2.2.1. 主要贡献
1. **首个面向方向偏差的CVR框架**：ReTrack是第一个通过校准组合特征的方向偏差提升多模态查询理解的CVR框架，无需额外标注数据即可实现性能提升。
2. **三大核心模块设计**：
   - 语义贡献解耦模块：基于Transformer解码器从组合特征中拆分出参考视频、修改文本的独立语义贡献。
   - 组合几何校准模块：生成双模态方向锚，通过对比损失显式校准组合特征的空间方向，消除模态偏向。
   - 可靠证据驱动对齐模块：基于证据理论计算双模态与目标的匹配置信度，提升相似性估计的鲁棒性。
3. **跨任务泛化能力**：该框架不仅适配CVR任务，还可直接迁移到组合图像检索（CIR）任务，在3个基准数据集上均达到SOTA性能。
### 2.2.2. 关键结论
1. 组合特征的方向偏差是制约CVR性能的核心因素，通过显式校准方向可带来显著的性能提升。
2. 证据深度学习可有效量化多模态匹配的可靠性，降低相似负样本带来的检索不确定性。
3. 基于模态贡献解耦-方向校准-可靠性增强的技术范式，可同时适配视频、图像等不同视觉模态的组合检索任务，具备强泛化性。

# 3. 预备知识与相关工作
## 3.1. 基础概念
本小节对理解论文所需的核心术语进行面向初学者的解释：
1. <strong>组合视频检索（Composed Video Retrieval, CVR）</strong>：一种多模态检索任务，输入为参考视频+描述修改要求的文本，输出为满足修改要求的目标视频，区别于传统单模态的文本搜视频、图搜视频。
2. <strong>组合图像检索（Composed Image Retrieval, CIR）</strong>：CVR的同源任务，输入为参考图像+修改文本，输出符合要求的目标图像，是组合检索领域的成熟方向。
3. **方向偏差**：融合多模态特征生成的组合特征，在特征空间中过度靠近某一模态（如参考视频）的特征方向，远离另一模态（如修改文本）的特征方向，导致检索结果不符合修改要求。
4. <strong>证据深度学习（Evidential Deep Learning, EDL）</strong>：一种量化模型预测不确定性的方法，无需贝叶斯神经网络的高计算开销，直接通过网络输出建模「证据」来衡量预测的置信度，适合处理模糊、高噪声的多模态匹配场景。
5. **Q-Former**：BLIP-2模型中的核心组件，是一种基于可学习查询的Transformer结构，可实现冻结视觉大模型与文本大模型的对齐，广泛用于多模态特征提取。
6. <strong>召回率@k（Recall@k, R@k）</strong>：检索任务的核心评估指标，指返回的Top k个候选结果中包含正确目标的查询占总查询数的比例，数值越高性能越好。
7. **Transformer注意力机制**：Transformer模型的核心计算单元，用于建模序列内或跨序列的依赖关系，标准化计算公式为：
   $$
   \mathrm{Attention}(Q, K, V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
   $$
   符号解释：$Q$为查询矩阵，$K$为键矩阵，$V$为值矩阵，$d_k$为$K$的特征维度，除以$\sqrt{d_k}$是为了防止点积值过大导致softmax函数饱和。
## 3.2. 前人工作
### 3.2.1. 组合检索相关工作
CVR任务由Ventura等人于2024年首次正式提出，现有方法可分为两类：
1. **基础CVR方法**：直接迁移CIR的融合思路，使用BLIP、BLIP-2等预训练多模态模型提取特征，通过简单的交叉注意力融合参考视频与修改文本特征，代表工作为CoVR（Ventura et al. 2024）、CoVR-2（Ventura et al. 2024）。
2. **增强型CVR方法**：通过额外生成的视频字幕增强查询语义，代表工作为CoVR Enrich（Thawakar et al. 2024），但该方法需要额外的生成成本，且仍未解决组合特征的方向偏差问题。
### 3.2.2. 不确定性估计相关工作
现有不确定性估计方法分为两类：
1. <strong>贝叶斯神经网络（BNN）</strong>：通过建模网络参数的后验分布量化不确定性，但计算开销大、推理速度慢，难以适配大规模检索任务。
2. <strong>证据深度学习（EDL）</strong>：由Sensoy等人于2018年提出，将主观逻辑与深度学习结合，直接通过网络输出建模证据、信念质量与不确定性，计算效率高，已在多模态分类、匹配任务中验证有效性。
## 3.3. 技术演进
组合检索技术的演进路径如下：
1. 2019年起，组合图像检索（CIR）成为研究热点，先后提出了基于特征融合、语义解析、对比学习的各类方法，但均针对图像模态设计。
2. 2024年，CVR任务正式提出，早期方法直接迁移CIR的融合逻辑，但未考虑视频与文本的模态信息密度差异，性能存在明显瓶颈。
3. 本文是首个针对CVR任务中组合特征方向偏差问题的工作，填补了现有研究的空白，同时将证据学习引入组合检索领域解决不确定性问题。
## 3.4. 差异化分析
与现有CVR/CIR方法相比，本文的核心区别与创新点为：

| 现有方法的共性问题 | 本文的创新解决方案 |
|------------------|------------------|
| 直接融合双模态特征，未考虑组合特征的方向偏差 | 显式解耦双模态语义贡献，生成方向锚校准组合特征的空间方向 |
| 仅依靠余弦相似度衡量组合特征与目标的匹配度，对相似负样本鲁棒性差 | 基于证据理论计算双模态匹配的置信度，自适应加权高可靠样本 |
| 仅适配单一任务（CVR或CIR） | 技术范式通用，可同时适配视频、图像模态的组合检索任务 |

## 4.1. 方法原理
核心思想：从特征空间的几何分布角度出发，组合特征的理想方向应当同时对齐参考视频的基础语义与修改文本的增量语义，不能偏向任何一方。因此首先解耦组合特征中两个模态的语义贡献，生成对应方向锚作为校准依据，再通过对比损失将组合特征的方向拉向目标特征的真实方向，最后结合证据学习量化匹配的可靠性，降低相似负样本的干扰。
## 4.2. 核心方法详解
### 4.2.1. 问题定义
CVR任务的训练数据为三元组集合$\mathcal{T} = \{ ( x _ { r } , x _ { m } , x _ { t } ) _ { n } \} _ { n = 1 } ^ { N }$，其中$x_r$为参考视频，$x_m$为修改文本，$x_t$为目标视频。任务目标是学习嵌入函数$\mathcal{G}$，使得多模态查询的嵌入与对应目标视频的嵌入尽可能接近：
$$
\mathcal{G}(x_r, x_m) \approx \mathcal{G}(x_t)
$$
符号解释：$\mathcal{G}$为可学习的多模态嵌入函数，输入为单模态或多模态数据，输出为对应特征向量。
### 4.2.2. 语义贡献解耦模块
该模块的目标是从初始组合特征中拆分出参考视频、修改文本各自的语义贡献，为后续方向校准提供基础。
#### 双模态特征提取与初始融合
使用BLIP-2的Q-Former作为基础特征提取器，分别提取参考视频、修改文本的特征，以及两者交叉融合的初始组合特征：
$$
\left\{ \begin{array} { l l } { \mathbf { F } _ { r } = \operatorname { Q - F o r m e r } ( \varPhi _ { \mathbb { I } } ( x _ { r } ) ) , \mathbf { F } _ { m } = \operatorname { Q - F o r m e r } ( \varPhi _ { \mathbb { T } } ( x _ { m } ) ) , } \\ { \mathbf { F } _ { c } = \operatorname { Q - F o r m e r } ( \varPhi _ { \mathbb { I } } ( x _ { r } ) , \varPhi _ { \mathbb { T } } ( x _ { m } ) ) , } \end{array} \right.
$$
符号解释：
- $\Phi_{\mathbb{I}}$为视觉编码器，将参考视频转换为视觉Token序列；$\Phi_{\mathbb{T}}$为文本分词器，将修改文本转换为文本Token序列。
- $\mathbf{F}_r \in \mathbb{R}^{Q \times D}$为参考视频特征，$\mathbf{F}_m \in \mathbb{R}^{Q \times D}$为修改文本特征，$\mathbf{F}_c \in \mathbb{R}^{Q \times D}$为初始组合特征。
- $Q$为Q-Former中可学习查询的数量，等于$32 \times N_f$，$N_f$为视频采样帧数（默认取4），$D$为特征嵌入维度。
  目标视频的特征$\mathbf{F}_t$使用相同的视觉编码器+Q-Former提取得到。
#### 语义贡献解耦
直接用$\mathbf{F}_c - \mathbf{F}_m$的简单减法无法准确得到参考视频的语义贡献（修改语义与参考语义存在复杂交互），因此使用Transformer解码器进行交叉注意力计算，以参考视频贡献为例：
$$
\mathbf { P } _ { r } = \operatorname { D e c o d e r } ( Q = \mathbf { F } _ { r } , \{ K , V \} = \mathbf { F } _ { c } )
$$
符号解释：
- 解码器使用参考视频特征$\mathbf{F}_r$作为查询$Q$，初始组合特征$\mathbf{F}_c$作为键$K$和值$V$，通过交叉注意力从$\mathbf{F}_c$中筛选出属于参考视频的语义信息，得到参考语义贡献$\mathbf{P}_r \in \mathbb{R}^{Q \times D}$。
- 同理，使用$\mathbf{F}_m$作为查询$Q$，得到修改文本的语义贡献$\mathbf{P}_m \in \mathbb{R}^{Q \times D}$。
### 4.2.3. 组合几何校准模块
该模块的目标是生成双模态方向锚，显式校准初始组合特征的方向偏差，使其同时对齐参考与修改语义。
#### 方向锚生成
不同特征通道对组合方向的影响权重不同，因此首先计算通道级的点权重$\mathbf{W}_p$：
$$
\mathbf { W } _ { p } = \mathrm { M L P } ( \mathbf { F } _ { c } \cdot \mathbf { F } _ { r } ^ { \top } )
$$
符号解释：
- $\mathbf{F}_c \cdot \mathbf{F}_r^\top$为初始组合特征与参考特征的点积相似度，衡量每个通道中参考语义的贡献大小。
- MLP为多层感知机，输出的$\mathbf{W}_p \in \mathbb{R}^{Q \times D}$为通道级权重，与组合语义相关性越高的通道权重越大。
  基于点权重生成参考方向锚：
$$
\mathbf { A } _ { r } = \mathbf { F } _ { c } + \mathbf { W } _ { p } \odot \mathbf { P } _ { r }
$$
符号解释：$\odot$为哈达玛积（对应元素相乘），将参考语义贡献按通道权重加到初始组合特征上，得到指向参考语义方向的参考锚$\mathbf{A}_r \in \mathbb{R}^{Q \times D}$。同理生成指向修改语义方向的修改锚$\mathbf{A}_m$。
#### 距离导向对齐损失
为了将初始组合特征拉到靠近目标特征的位置，为后续方向校准提供基础，使用批次内对比损失：
$$
\mathcal { L } _ { d i s } = \frac { 1 } { B } \sum _ { i = 1 } ^ { B } - \log \left\{ \frac { \exp \left\{ S \left( \mathbf { F } _ { c i } , \mathbf { F } _ { t i } \right) / \tau \right\} } { \sum _ { j = 1 } ^ { B } \exp \left\{ S \left( \mathbf { F } _ { c i } , \mathbf { F } _ { t j } \right) / \tau \right\} } \right\}
$$
符号解释：
- $B$为训练批次大小，$S(\cdot, \cdot)$为余弦相似度函数，$\tau$为温度系数（默认取0.1）。
- $\mathbf{F}_{ci}$为批次内第$i$个样本的初始组合特征，$\mathbf{F}_{ti}$为对应第$i$个样本的目标特征，$\mathbf{F}_{tj}$为批次内第$j$个样本的目标特征。
  该损失的目标是让正样本对（同一三元组的组合特征与目标特征）的相似度远高于负样本对（不同三元组的组合特征与目标特征）。
#### 方向导向校准损失
基于平行四边形法则生成组合方向锚，即两个方向锚相对于初始组合特征的方向向量之和：
$$
\mathbf { A } _ { c } = ( \mathbf { A } _ { r } - \mathbf { F } _ { c } ) + \left( \mathbf { A } _ { m } - \mathbf { F } _ { c } \right)
$$
符号解释：$\mathbf{A}_r - \mathbf{F}_c$为参考锚的方向向量，$\mathbf{A}_m - \mathbf{F}_c$为修改锚的方向向量，相加得到组合特征应该调整的总方向向量，即组合方向锚$\mathbf{A}_c$。
同时计算从初始组合特征指向目标特征的真实方向向量$\mathbf{A}_t = \mathbf{F}_t - \mathbf{F}_c$，使用对比损失引导组合方向锚对齐真实方向：
$$
\mathcal { L } _ { d i r } = \frac { 1 } { B } \sum _ { i = 1 } ^ { B } - \log \left\{ \frac { \exp \left\{ S \left( \mathbf { A } _ { c i } , \mathbf { A } _ { t i } \right) / \tau \right\} } { \sum _ { j = 1 } ^ { B } \exp \left\{ S \left( \mathbf { A } _ { c i } , \mathbf { A } _ { t j } \right) / \tau \right\} } \right\}
$$
符号解释：$\mathbf{A}_{ci}$为第$i$个样本的组合方向锚，$\mathbf{A}_{ti}$为第$i$个样本的真实方向向量，$\mathbf{A}_{tj}$为第$j$个样本的真实方向向量。该损失可显式校准组合特征的方向，消除模态偏向。
### 4.2.4. 可靠证据驱动对齐模块
该模块基于Dempster-Shafer证据理论（DST）计算双模态与目标的匹配置信度，提升相似性估计的可靠性，降低相似负样本的干扰。
#### 证据建模
以参考锚与目标的匹配证据计算为例：
1. 首先计算参考锚每个通道与目标特征的匹配证据：
   $$
   e _ { q } = \exp ( \underset { \hat { q } = 1 } { \overset { Q } { \operatorname* { m a x } } } \left( \mathbf A _ { r ( q ) } \cdot \mathbf F _ { t } ^ { \top } \right) _ { \hat { q } } / \tau )
   $$
   符号解释：$\mathbf{A}_{r(q)}$为参考锚的第$q$个通道特征，与目标特征$\mathbf{F}_t$做点积后取最大值，经指数化得到第$q$个通道的匹配证据$e_q$，值越大表示匹配程度越高。
2. 计算每个通道的信念质量，即该通道证据的归一化置信度：
   $$
   b _ { q } = \frac { e _ { q } } { \sum _ { \hat { q } = 1 } ^ { Q } \left( e _ { \hat { q } } + 1 \right) }
   $$
   符号解释：分母中的$+1$为EDL中的不确定质量项，衡量整体匹配的不确定性。
3. 计算参考锚与目标的整体匹配可靠性：
   $$
   \mathbb { E } _ { r } = \sum _ { q = 1 } ^ { Q } b _ { q } = 1 - \frac { Q } { \sum _ { \hat { q } = 1 } ^ { Q } \left( e _ { \hat { q } } + 1 \right) }
   $$
   符号解释：$\mathbb{E}_r$取值范围为`[0,1]`，值越大表示参考锚与目标的匹配置信度越高。同理计算修改锚与目标的匹配可靠性$\mathbb{E}_m$。
#### 证据正则化损失
为了让匹配可靠性与组合特征-目标的余弦相似度保持一致，设计均方误差损失：
$$
\mathcal { L } _ { e v i } = \frac { 1 } { B } \sum _ { b = 1 } ^ { B } { ( \mathbb { E } _ { r b } - S \left( \mathbf { F } _ { c b } , \mathbf { F } _ { t b } \right) ) ^ { 2 } } + \left( \mathbb { E } _ { m b } - S \left( \mathbf { F } _ { c b } , \mathbf { F } _ { t b } \right) \right) ^ { 2 }
$$
符号解释：$\mathbb{E}_{rb}$、$\mathbb{E}_{mb}$为第$b$个样本的参考、修改匹配可靠性，$S(\mathbf{F}_{cb}, \mathbf{F}_{tb})$为第$b$个样本的组合特征与目标的余弦相似度。该损失可让相似性高的样本同时具备高匹配证据，提升相似性估计的可靠性。
#### 总损失函数
$$
\Theta ^ { * } = \underset { \Theta } { \arg \operatorname* { m i n } } \left( \mathcal { L } _ { d i s } + \kappa \mathcal { L } _ { d i r } + \lambda \mathcal { L } _ { e v i } \right)
$$
符号解释：$\Theta$为模型所有可学习参数，$\kappa$、$\lambda$为超参数，分别控制方向损失、证据损失的权重，实验中取$\kappa=0.5$，$\lambda=1.0$。

# 5. 实验设置
## 5.1. 数据集
为验证方法的有效性与泛化性，实验覆盖CVR和CIR两类任务，共使用3个基准数据集：
1. **WebVid-CoVR**：CVR任务的大规模开放域数据集，从WebVid视频库构建，包含数十万条「参考视频-修改文本-目标视频」三元组，覆盖各类通用场景，是CVR领域的标准测试集。
2. **FashionIQ**：CIR任务的时尚领域数据集，包含连衣裙、衬衫、上衣三个类别的「参考图-修改文本-目标图」三元组，用于验证方法在垂直领域的泛化能力。
3. **CIRR**：CIR任务的开放域数据集，基于真实场景图像构建，包含复杂的修改文本描述，用于验证方法在通用图像检索场景的性能。
## 5.2. 评估指标
论文使用的核心评估指标为<strong>召回率@k（Recall@k, R@k）</strong>，详细说明如下：
### 概念定义
R@k衡量检索结果的准确性：对于所有测试查询，返回的Top k个最相似的候选结果中，包含正确目标的查询占总查询数的比例，数值越高表示检索性能越好。
### 数学公式
$$
R@k = \frac{\text{Number of queries where target is in top } k \text{ results}}{\text{Total number of queries}} \times 100\%
$$
### 符号解释
- 分子：Top k检索结果中包含正确目标的查询数量
- 分母：测试集的总查询数量
- 最终结果乘以100%转换为百分比形式。
  此外，CIRR数据集还使用子集召回率$\mathbf{R}_{sub}@k$，即仅在固定的小规模候选子集中计算R@k，衡量模型在低干扰场景下的检索精度。
## 5.3. 对比基线
实验对比三类基线模型，确保结果的公允性：
1. **通用预训练基线**：CLIP、BLIP，为通用多模态预训练模型，用于验证CVR专用方法的性能增益。
2. **CVR专用基线**：CoVR、CoVR Enrich、CoVR-2、FDCA，为现有最新的CVR模型，其中CoVR Enrich使用额外生成的字幕增强语义，是当前CVR领域的最优基线。
3. **CIR专用基线**：TG-CIR、SSN、SADN、SPRC、LIMN等，为现有最新的CIR模型，用于验证ReTrack的跨任务泛化能力。

# 7. 总结与思考
## 7.1. 结论总结
本文针对组合视频检索任务中组合特征方向偏差的核心问题，提出了ReTrack框架：通过语义贡献解耦模块拆分双模态语义贡献，通过组合几何校准模块显式校准特征方向，通过可靠证据驱动对齐模块提升相似性估计的鲁棒性。实验证明该框架在CVR和CIR两个任务、3个基准数据集上均达到SOTA性能，验证了方向校准范式的有效性与跨任务泛化能力，为组合检索领域提供了新的研究思路。
## 7.2. 局限性与未来工作
### 7.2.1. 论文指出的局限性与未来方向
- 局限性：当前框架仅支持单轮组合查询，未适配多轮交互的检索场景；仅使用BLIP-2作为主干网络，未验证更大规模多模态大模型的适配性。
- 未来工作：计划将方法扩展到多轮交互式组合多模态检索任务，支持用户连续的修改查询。
### 7.2.2. 潜在的其他局限性
1. 方法依赖对比损失进行训练，需要较大的批次大小才能达到较好的效果，对GPU显存要求较高。
2. 证据计算仅基于通道级的最大点积，未考虑多通道的上下文关联，仍有优化空间。
3. 未验证零样本/少样本场景下的性能，当前实验均基于全监督微调设置。
## 7.3. 个人启发与批判
1. **研究思路启发**：本文从特征空间几何分布的直观问题切入，没有引入过于复杂的结构，仅通过解耦-校准-增强三个逻辑连贯的模块就实现了显著的性能提升，且具备强泛化性，这种「问题驱动、简洁有效」的研究思路值得借鉴。
2. **可迁移性**：该方法的技术范式可直接迁移到其他组合检索任务，例如组合音频检索、组合3D模型检索、跨模态组合检索等，只要存在双模态融合的方向偏差问题，均可通过类似的方向校准+证据增强思路优化。
3. **可改进方向**：
   - 可结合大语言模型的语义解析能力，先对修改文本进行结构化解析，更精准地计算修改方向向量，进一步提升校准效率。
   - 可将证据学习扩展到候选集排序阶段，对候选结果的匹配置信度进行重排序，进一步降低相似负样本的干扰。
   - 可引入适配器（Adapter）结构实现低参数微调，降低对大规模训练数据的依赖，适配零样本/少样本场景。
