# 1. 论文基本信息
## 1.1. 标题
论文全称为《Embed-RL: Reinforcement Learning for Reasoning-Driven Multimodal Embeddings》，核心主题是提出一种基于强化学习的推理驱动多模态嵌入框架，解决现有生成式多模态嵌入中思维链与检索目标不对齐、梯度冲突的问题。

## 1.5. 摘要
现有基于多模态大语言模型（MLLM）的通用多模态嵌入（UME）方法中，生成式思维链（CoT）仅局限于查询的文本分析，与检索目标无关，且生成目标与嵌入目标联合优化存在梯度冲突问题。为解决上述缺陷，论文提出Embed-RL框架，通过<strong>嵌入器引导的强化学习（Embedder-Guided Reinforcement Learning, EG-RL）</strong> 优化推理器，生成带多模态证据的可追溯思维链（Traceability CoT, T-CoT）。
核心贡献包括三点：
1.  设计EG-RL解耦框架，用预训练好的嵌入器为推理器提供明确监督，保证生成的CoT与嵌入任务对齐；
2.  提出T-CoT，提取关键多模态线索（文本关键词、图像边界框、视频关键帧），聚焦检索相关元素，为嵌入器提供多模态输入；
3.  在有限计算资源下，在MMEB-V2和UVRB两个 benchmark 上均超越现有SOTA嵌入模型。
    实验证明该框架有效增强了跨模态语义一致性，提升了细粒度匹配能力和复杂场景泛化性，为推理驱动的通用多模态嵌入提供了高效实用的解决方案。

# 2. 整体概括
## 2.1. 研究背景与动机
### 2.1.1. 核心问题
通用多模态嵌入（UME）是跨模态检索、视频片段定位、视觉文档理解等任务的核心支撑技术。现有方法存在两大核心痛点：
1.  **梯度冲突问题**：早期将生成式CoT融入嵌入的方法（如UME-R1）同时优化对比损失和下一词预测损失，两类目标的梯度方向存在冲突，导致性能次优；
2.  **CoT不对齐问题**：后续解耦架构（如TTE）用预训练MLLM离线生成CoT，再训练嵌入器，但生成的CoT没有针对嵌入任务优化，与检索目标无关，甚至会引入幻觉和噪声，且仅支持文本CoT，无法充分利用多模态信息。
### 2.1.2. 问题重要性
随着多模态内容（图文、短视频、视觉文档）的爆发，通用多模态嵌入的质量直接决定了跨模态检索、推荐、RAG等下游任务的性能上限。现有方法的上述缺陷导致细粒度匹配能力弱、复杂场景泛化性差，无法满足工业级应用的需求。
### 2.1.3. 创新切入点
论文采用**解耦+强化学习对齐**的思路：首先独立训练嵌入器获得稳定的嵌入能力，然后将嵌入器作为奖励模型，用强化学习指导推理器生成和检索任务对齐的、带多模态证据的T-CoT，既避免了梯度冲突，又解决了CoT与检索目标不对齐的问题。
## 2.2. 核心贡献与主要发现
### 2.2.1. 核心贡献
1.  **EG-RL解耦强化学习框架**：首次提出用预训练冻结的嵌入器为推理器的CoT生成提供奖励信号，实现推理目标与嵌入目标的精准对齐，解决了传统生成式嵌入的梯度冲突和CoT不对齐问题；
2.  <strong>可追溯思维链（T-CoT）</strong>：将CoT从纯文本扩展到多模态场景，显式融入图像边界框、视频关键帧、文本关键词等可追溯的多模态证据，引导嵌入器聚焦检索相关信息，过滤冗余噪声；
3.  **低资源下的SOTA性能**：在计算资源远低于竞品（训练 batch 仅为TTE的1/32）的情况下，在MMEB-V2（覆盖78个多模态任务）和UVRB（视频检索基准）上均超越现有SOTA模型，尤其在细粒度定位、长视频检索、跨域泛化任务上优势显著。
### 2.2.2. 主要发现
1.  用已训练好的嵌入器作为奖励模型指导CoT生成，能有效让推理过程服务于嵌入目标，比直接用人工标注的CoT效果更好；
2.  多模态可追溯证据的加入，能大幅提升细粒度匹配和跨模态对齐能力，尤其对视频、视觉文档等复杂模态的任务增益最大；
3.  解耦架构的训练效率更高，不需要超大规模的计算资源，具备更高的工业落地可行性。

# 3. 预备知识与相关工作
## 3.1. 基础概念
为方便初学者理解，本部分对论文涉及的核心基础概念做详细解释：
### 3.1.1. 通用多模态嵌入（Universal Multimodal Embedding, UME）
是指将文本、图像、视频等不同模态的信息映射到同一个向量空间，语义相似的内容会得到距离相近的向量表示，从而支持跨模态检索、匹配等任务，是所有跨模态应用的核心基础。
### 3.1.2. 多模态大语言模型（Multimodal Large Language Models, MLLMs）
是融合了文本、图像、视频等多模态理解能力的大语言模型，既能处理文本输入，也能理解图像、视频等视觉内容，具备强大的多模态推理和指令遵循能力。
### 3.1.3. 思维链（Chain-of-Thought, CoT）
是大模型提升任务表现的核心技术之一，通过让模型输出中间推理步骤，而不是直接给出最终答案，来提升结果的准确性和可解释性，最初用于文本推理任务，后续扩展到多模态场景。
### 3.1.4. 对比学习与InfoNCE损失
对比学习是表示学习的主流范式，核心思想是让正样本对（语义相似的内容）的向量距离尽可能近，负样本对（语义不相似的内容）的向量距离尽可能远。InfoNCE损失是对比学习最常用的损失函数，通过噪声对比估计的方式优化上述目标。
### 3.1.5. 强化学习（Reinforcement Learning, RL）
是机器学习的一个分支，智能体（Agent）通过与环境交互，根据获得的奖励信号调整自身的行为策略，最终最大化累计奖励。本文中智能体是生成CoT的推理器，环境是多模态输入和冻结的嵌入器，奖励由嵌入效果和CoT对齐程度决定。
### 3.1.6. 分组相对策略优化（Group Relative Policy Optimization, GRPO）
是DeepSeek-R1提出的专为大模型设计的强化学习算法，通过将样本分组计算相对优势，不需要额外的价值网络，训练更稳定，内存开销更低，是当前大模型RL微调的主流算法。
## 3.2. 前人工作
### 3.2.1. 通用多模态嵌入技术演进
1.  **早期双编码器架构**：以CLIP、BLIP、SigLIP为代表，采用图像编码器和文本编码器分别提取特征，通过对比学习对齐跨模态表示，但跨模态理解能力弱，无法处理复杂的多模态输入和指令；
2.  **MLLM驱动的判别式嵌入**：以VLM2Vec、GME、LamRA为代表，直接从MLLM的最终隐藏层提取嵌入特征，利用MLLM的强理解能力提升嵌入质量，但没有利用MLLM的生成和推理能力；
3.  **生成式嵌入+CoT**：以UME-R1、TTE为代表，尝试将CoT融入嵌入过程，UME-R1联合优化生成和嵌入目标，存在梯度冲突；TTE采用解耦架构，先离线生成CoT再训练嵌入器，但CoT没有针对嵌入任务优化，与检索目标不对齐。
### 3.2.2. 多模态推理与强化学习结合
近期大量工作探索用强化学习提升MLLM的多模态推理能力：
- GRIT将边界框坐标融入文本推理链，基于GRPO设计双奖励机制，不需要额外标注数据即可提升视觉推理能力；
- Ground-R1提出RL框架实现接地视觉推理，通过双奖励引导响应生成，提升推理可靠性和可解释性；
- DeepEyes采用端到端RL引导模型学会“用图像思考”，提升多模态推理任务表现。
  上述工作主要聚焦推理任务本身，没有将强化学习用于对齐推理过程和嵌入目标，本文是首次尝试用RL让CoT服务于多模态嵌入任务。
## 3.3. 技术脉络定位
该领域的技术演进路径为：
`双编码器对比学习` → `MLLM判别式嵌入` → `生成式嵌入+纯文本CoT` → <strong>`RL对齐的多模态CoT生成式嵌入`（本文工作）</strong>
本文的核心突破是将多模态推理过程和嵌入目标通过强化学习做端到端对齐，同时保留了解耦架构的训练稳定性优势。
## 3.4. 差异化分析
与现有生成式嵌入方法的核心区别如下：

| 方法 | 架构 | CoT类型 | 优化方式 | 核心问题 | 本文优势 |
|------|------|---------|----------|----------|----------|
| UME-R1 | 联合优化 | 纯文本CoT | 同时优化对比损失和生成损失 | 梯度冲突 | 解耦架构无梯度冲突 |
| TTE | 解耦架构 | 纯文本CoT | 先离线生成CoT再训练嵌入器 | CoT与检索目标不对齐 | 用RL让CoT和嵌入目标对齐，加入多模态证据 |
| Embed-RL | 解耦RL架构 | 多模态T-CoT | 先训练嵌入器，再用嵌入器作为奖励模型优化推理器 | - | 无梯度冲突，CoT与检索对齐，多模态证据增强细粒度匹配 |

# 4. 方法论
本部分为论文核心技术方案，严格按照原文内容拆解，所有公式100%与原文一致。
## 4.1. 方法原理
核心直觉可以概括为两点：
1.  **解耦避免梯度冲突**：将推理（生成CoT）和嵌入两个过程完全解耦，先独立训练嵌入器获得稳定的嵌入能力，再单独优化推理器，避免两类目标的梯度冲突；
2.  **RL实现目标对齐**：将训练好的嵌入器作为“裁判”，给推理器生成的CoT打分（奖励），引导推理器生成能让嵌入效果最好的CoT，从根本上解决CoT与检索目标不对齐的问题；
3.  **多模态证据增强细粒度匹配**：在CoT中显式加入可追溯的多模态线索（图像边界框、视频关键帧、文本关键词），让嵌入器聚焦检索相关的核心信息，过滤冗余噪声，提升细粒度匹配能力。

## 4.2. 核心方法详解
### 4.2.1. 数据集构建
为支撑推理驱动的多模态嵌入训练，论文首先构建了高质量的多模态数据集，流程为“采样-标注-过滤-拆分”：
#### 4.2.1.1. 数据来源
采用分层采样策略，从三个核心数据源采样，总初始样本量222万：
1.  图像类任务：来自MMEB-train，覆盖图像分类、QA、检索、定位等任务；
2.  视频类任务：来自LLaVA-Hound，覆盖视频字幕、QA、检索等任务；
3.  视觉文档类任务：来自ViDoRe和VisRAG，覆盖视觉文档检索任务。
#### 4.2.1.2. T-CoT标注
为所有查询-正样本对标注结构化的T-CoT，固定为三段格式：
1.  $<thinking>$：提取模态特定线索，文本用`text_keywords`存储关键词，图像用$bbox_2d$存储二维边界框坐标，视频用`key_frames`存储关键帧索引；
2.  $<rethink>$：优化推理逻辑，聚焦检索相关的核心要素；
3.  $<answer>$：总结检索相关的核心信息。
#### 4.2.1.3. 相关性过滤
通过自定义的判断提示，过滤掉T-CoT与任务明显无关或冲突的样本，最终保留183万样本，留存率约80%。过滤掉的20%样本作为难例用于RL阶段的训练，提升模型的鲁棒性。
### 4.2.2. 嵌入器预训练
首先用对比学习训练嵌入器，采用InfoNCE损失，原文公式如下：
$$
\begin{array} { r } { \mathcal { L } _ { \mathrm { I n f o N C E } } = - \frac { 1 } { N } \sum _ { i = 1 } ^ { N } \log \frac { \exp \Bigl ( \cos ( h _ { q _ { i } } , h _ { t _ { i } ^ { + } } ) / \tau \Bigr ) } { \exp \Bigl ( \cos ( h _ { q _ { i } } , h _ { t _ { i } ^ { + } } ) / \tau \Bigr ) + \sum _ { t ^ { - } \in \mathcal { T } ^ { - } } \exp \bigl ( \cos ( h _ { q _ { i } } , h _ { t ^ { - } } ) / \tau \bigr ) } \ , }
\end{array}
$$
符号解释：
- $N$：批次大小；
- $h_{q_i}$：查询$q_i$的嵌入向量，从MLLM最后一个特殊$<emb>$ token的最后一层隐藏状态提取；
- $h_{t_i^+}$：查询$q_i$对应的正样本$t_i^+$的嵌入向量；
- $\mathcal{T}^-$：批次内的负样本集合；
- $\cos(\cdot, \cdot)$：两个向量的余弦相似度函数；
- $\tau$：温度超参数，用于缩放相似度值，控制概率分布的尖锐程度。
  训练完成后，嵌入器被完全冻结，作为RL阶段的奖励模型，为推理器生成的T-CoT提供稳定的奖励信号。
### 4.2.3. EG-RL强化学习框架
RL阶段的目标是优化推理器，使其生成的T-CoT能最大化嵌入的检索效果，框架采用完全解耦设计：推理器负责生成T-CoT，嵌入器冻结作为奖励模型。
推理器的输入是多模态查询，输出结构化的T-CoT，将T-CoT与原始输入拼接后输入到冻结的嵌入器，得到最终嵌入，嵌入的质量直接决定奖励的高低。
#### 4.2.3.1. 奖励函数设计
奖励函数由三个部分加权组成，分别从格式合规性、过程对齐性、结果有效性三个维度评估T-CoT的质量：
##### （1）格式奖励 $\mathcal{R}_{format}$
用于保证T-CoT严格遵循 predefined 的三段格式，包含所有 required 的多模态线索。规则为：完全符合格式奖励1，否则奖励0，确保T-CoT的可解释性和与嵌入器的兼容性。
##### （2）嵌入引导的结果奖励 $\mathcal{R}_{outcome}$
用于衡量T-CoT对嵌入对齐效果的提升，联合评估正样本的排序准确率和正负样本的相似度间隔，原文公式如下：
$$
\begin{array} { r } { \mathcal { R } _ { \mathrm { o u t c o m e } } ( o _ { i } ^ { q } ) = \operatorname { A c c } _ { k } ( e _ { q _ { i } } , t _ { i } ^ { + } ) \cdot \Big ( \sin ( e _ { q } , e _ { t _ { i } ^ { + } } ) - \mathbb { E } _ { \tau } \big [ \sin ( e _ { q _ { i } } , e _ { t _ { j } ^ { - } } ) \big ] \Big ) , }
\end{array}
$$
> 注：原文公式中$\sin$为笔误，结合上下文实际指代余弦相似度函数$\mathrm{sim}(\cdot, \cdot)$，后文明确说明该值为归一化嵌入的余弦相似度。
符号解释：
- $o_i^q$：查询$q_i$对应的T-CoT输出；
- $\mathrm{Acc}_k(e_{q_i}, t_i^+)$：top-k检索准确率，即正样本$t_i^+$是否在与查询嵌入$e_{q_i}$相似度最高的前k个结果中，本文$k=8$；
- $\mathrm{sim}(e_q, e_{t_i^+})$：查询嵌入与正样本嵌入的余弦相似度；
- $\mathbb{E}_\tau[\mathrm{sim}(e_{q_i}, e_{t_j^-})]$：查询嵌入与批次内负样本嵌入余弦相似度的软max加权平均值，$\tau$为温度超参数，本文设为0.5，用于突出难负样本的权重。
  该奖励同时从排序正确性和正负样本间隔两个维度优化嵌入效果，并且会对称计算正样本作为锚点时的奖励，保证双向对齐。
##### （3）T-CoT过程奖励 $\mathcal{R}_{process}$
用于衡量查询和正样本的T-CoT之间的对齐程度，采用独立预训练的视觉语言判别器$\mathcal{D}$做列表式比较，原文公式如下：
$$
\begin{array} { r } { \mathcal { R } _ { \mathrm { p r o c e s s } } ( o _ { i } ) = \left\{ \begin{array} { l l } { 1 , } & { \mathrm { i f } \ \mathcal { D } \big ( q _ { \mathrm { c o t } } , \{ c _ { \mathrm { c o t } } ^ { j } \} _ { j = 1 } ^ { m } \big ) \in \mathcal { P } , } \\ { 0 , } & { \mathrm { o t h e r w i s e } , } \end{array} \right. }
\end{array}
$$
符号解释：
- $o_i$：第i个样本的T-CoT生成结果；
- $q_{cot}$：查询的T-CoT输出；
- $\{c_{cot}^j\}_{j=1}^m$：m个候选T-CoT输出，包含正样本和负样本的T-CoT；
- $\mathcal{P}$：真实正样本的索引集合；
- $\mathcal{D}(\cdot, \cdot)$：预训练的视觉语言判别器，输出与查询T-CoT最匹配的候选T-CoT的索引。
  如果判别器正确选出来自正样本的T-CoT则奖励1，否则奖励0，该奖励同样会对称计算，保证双向对齐。
##### （4）总奖励
总奖励为三个部分的加权和：
$$
\mathcal { R } _ { \mathrm { t o t a l } } = \alpha \mathcal { R } _ { \mathrm { f o r m a t } } + \beta \mathcal { R } _ { \mathrm { p r o c e s s } } + \gamma \mathcal { R } _ { \mathrm { o u t c o m e } },
$$
其中权重系数本文设置为$\alpha=0.05$，$\beta=0.8$，$\gamma=0.2$，平衡三类奖励的贡献。
#### 4.2.3.2. GRPO策略优化
采用GRPO算法优化推理器的策略，通过分组奖励稳定训练过程，原文的优化目标公式如下：
$$
\begin{array} { r l } { \mathcal { L } _ { \mathrm { g r p o } } = \mathbb { E } \underset { \{ o _ { i } \} \sim \pi _ { \theta _ { \mathrm { o l d } } } } { \mathbb { E } } \Bigg [ \frac { 1 } { G } \sum _ { i = 1 } ^ { G } \bigg ( } & { \operatorname* { m i n } ( r _ { \theta } ( o _ { i } ) A _ { i } , \mathrm { c l i p } ( r _ { \theta } ( o _ { i } ) , 1 - \epsilon , 1 + \epsilon ) A _ { i } ) } \\ & { - \beta \mathbb { D } _ { \mathrm { K L } } ( \pi _ { \theta } \| \pi _ { \mathrm { r e f } } ) \bigg ) \Bigg ] , }
\end{array}
$$
符号解释：
- $\{o_i\}_{i=1}^G$：从旧策略$\pi_{\theta_{old}}$为每个查询采样的G个T-CoT序列，本文$G=8$；
- $r_\theta(o_i) = \frac{\pi_\theta(o_i|q)}{\pi_{\theta_{old}}(o_i|q)}$：重要性权重，即新策略和旧策略生成该T-CoT序列的概率比值；
- $A_i$：优势函数，$A_i = \frac{r_i - \mu_r}{\sigma_r}$，其中$r_i$是该序列的总奖励，$\mu_r$是组内奖励的均值，$\sigma_r$是组内奖励的标准差，用于归一化奖励，稳定训练；
- $\epsilon$：裁剪阈值，本文设为0.2，限制重要性权重的范围，防止策略更新步长过大；
- $\mathbb{D}_{KL}(\pi_\theta \| \pi_{ref})$：新策略与参考策略（优化前的旧策略）的KL散度，防止策略偏离初始状态过远，$\beta$为KL散度的权重，本文设为0.01。

## 5.2. 评估指标
本部分对所有评估指标做完整说明，包含定义、公式、符号解释：
### 5.2.1. Hit@1
#### 概念定义
衡量检索任务中，正样本排在检索结果第一位的比例，值越高说明检索准确性越好。
#### 数学公式
$$
Hit@1 = \frac{1}{N} \sum_{i=1}^N \mathbb{I}(rank_i = 1)
$$
#### 符号解释
- $N$：查询总数量；
- $\mathbb{I}(\cdot)$：指示函数，条件成立时取1，否则取0；
- $rank_i$：第i个查询对应的正样本在检索结果中的排名。
### 5.2.2. NDCG@5（Normalized Discounted Cumulative Gain @5）
#### 概念定义
衡量排序结果的质量，考虑了结果的相关性程度和位置权重，对排在前面的结果赋予更高的权重，值越高说明排序质量越好，多用于文档检索任务。
#### 数学公式
首先计算DCG（折现累积增益）：
$$
DCG@k = \sum_{i=1}^k \frac{2^{rel_i} - 1}{\log_2(i+1)}
$$
然后计算IDCG（理想折现累积增益），即所有相关结果按相关性从高到低排序后的DCG值，最终NDCG@k为：
$NDCG@k = \frac{DCG@k}{IDCG@k}$
本文$k=5$，即评估前5个检索结果的排序质量。
#### 符号解释
- $rel_i$：第i个位置的检索结果与查询的相关性得分；
- $k$：考虑的检索结果数量，本文为5。
### 5.2.3. mAP（mean Average Precision）
#### 概念定义
平均精度的均值，衡量所有查询的平均检索精度，同时考虑了召回率和准确率，是检索任务最常用的综合指标，值越高说明检索性能越好。
#### 数学公式
首先计算单个查询的平均精度（AP）：
$$
AP = \sum_{k=1}^M P(k) \cdot \mathbb{I}(k是正样本)
$$
其中`P(k)`是前k个检索结果的准确率，$M$是检索结果总数。然后对所有查询的AP取平均得到mAP：
$$
mAP = \frac{1}{N} \sum_{i=1}^N AP_i
$$
#### 符号解释
- $N$：查询总数量；
- $AP_i$：第i个查询的平均精度。

# 7. 总结与思考
## 7.1. 结论总结
本文针对现有生成式多模态嵌入的两大核心痛点：CoT与检索目标不对齐、生成与嵌入目标联合优化存在梯度冲突，提出了Embed-RL推理驱动多模态嵌入框架，通过EG-RL解耦强化学习框架，用预训练的嵌入器引导推理器生成带多模态证据的T-CoT，实现了推理过程与嵌入目标的精准对齐。
实验证明，在有限计算资源下，Embed-RL在MMEB-V2和UVRB两个基准上均超越现有SOTA模型，尤其在细粒度定位、长视频检索、跨域泛化任务上优势显著，为推理驱动的通用多模态嵌入提供了高效、可落地的解决方案。
## 7.2. 局限性与未来工作
论文作者明确指出了三点局限性和对应的未来方向：
1.  **奖励权重固定**：当前多组件奖励的权重系数是手动设置的，缺乏针对不同任务的自适应调整机制，未来可探索自适应权重调整策略，针对不同模态、不同任务动态调整奖励权重，进一步提升性能；
2.  **分类任务表现弱**：训练集排除了部分分类任务，避免大批次对比学习中的假负样本问题，导致分类任务表现相对较弱，未来可设计针对分类任务的损失函数和负采样策略，在避免假负样本的同时提升分类任务表现；
3.  **缺乏难负样本和课程学习**：当前未引入难负样本挖掘和课程学习策略，未来加入上述技术可进一步提升模型的判别能力和训练稳定性。
## 7.3. 个人启发与批判
### 7.3.1. 核心启发
1.  **解耦RL对齐的思路可迁移性强**：本文用下游任务的判别模型作为奖励模型，引导上游生成过程的思路，不仅可用于多模态嵌入，还可推广到其他生成+判别结合的任务，比如RAG的查询改写、推荐系统的用户特征生成等，用下游任务效果直接指导上游生成过程，避免目标错位；
2.  **多模态可追溯证据的工业价值**：T-CoT显式融入多模态证据的设计，不仅提升了嵌入性能，还让嵌入过程具备可解释性，适合对可解释性有要求的工业场景，比如内容审核、电商检索等；
3.  **低资源训练的落地优势**：解耦架构的训练效率远高于联合优化架构，不需要超大规模的计算资源，可快速复现和落地，对中小团队非常友好。
### 7.3.2. 潜在改进方向
1.  **在线推理延迟优化**：当前T-CoT生成需要额外的推理开销，虽然可以离线处理库中的文档和视频，但在线用户查询的T-CoT生成会增加延迟，未来可探索轻量化的T-CoT生成模型，或者支持查询端无CoT的推理模式，平衡性能和延迟；
2.  **T-CoT结构自适应**：当前T-CoT采用固定的三段格式，未来可根据任务类型自适应调整CoT的结构和长度，比如分类任务不需要太多的空间定位证据，检索任务需要更多的多模态线索，进一步提升效率和性能；
3.  **多任务统一奖励**：当前奖励主要针对检索任务设计，未来可扩展奖励函数，支持分类、QA等多任务的统一优化，进一步提升通用多模态嵌入的能力。
### 7.3.3. 应用价值
该工作非常适合工业级多模态检索场景，尤其是短视频平台、电商平台、文档管理系统等，可直接用于提升跨模态检索的准确率和泛化性，同时T-CoT的可追溯性也支持后续的bad case分析和优化，具备很高的落地价值。
